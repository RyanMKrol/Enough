# T036 worklog

## 2026-07-16

**Done:** Assembled the "Your plan" screen — seeded bundle/pack recommendation from the

### What was built

1. **PlanViewModel.swift** (`@Observable`)
   - `init(country:draft:)` seeds `draft.selectedBundleId` only when unset: `.weekend` picks
     the country's `popular` bundle (fixture: "Weekend"), `.week` picks the bundle covering
     every deck (max `deckIds.count`, fixture: "The whole week"). Seeding is skipped entirely
     when a bundle id is already present, so back-navigation never re-seeds.
   - `selectedBundle` / `packRows` are always derived live from the catalog + draft (never
     cached) — bundle members first (catalog order), then the rest; state is `.inBundle`,
     `.extra`, or `.available` per membership/`extraDeckIds`, price label always
     `PricingCalculator.price(deck.priceGBP)`.
   - `bundleSubtitle(for:)` composes `"<subtitle> · save <PricingCalculator.price(savings)>"`
     via `PricingCalculator.bundleSavings`.
   - `selectBundle(_:)` sets the draft's bundle and drops any now-redundant `extraDeckIds`;
     `togglePack(_:)` is a no-op for bundle members, otherwise toggles `extraDeckIds`.
   - `total`/`totalLabel`/`summary`/`ctaTitle` all delegate to `PricingCalculator` (T025) —
     no arithmetic on `priceGBP` happens in the view model or view.
   - `purchaseSelection(purchase:)` purchases the bundle (`kind: .bundle`) then each extra
     deck (`kind: .deck`); stops immediately and returns `.cancelled` on the first cancelled
     purchase, tracks `.pending`, and only returns `.success` when nothing was
     cancelled/pending.

2. **PlanView.swift**
   - Header ("Your plan" / subtitle), `BundleRowView` per country bundle (price sourced from
     `services.purchase.displayPrice(productId:)` falling back to `PricingCalculator.price`),
     "or choose packs" hairline divider, a white `PackChecklistRow` card with 49pt-inset
     hairline dividers between rows (`onToggle` nil for `.inBundle` rows), and a
     non-interactive "Browse all `<country>` decks" link row (`// Browse tab arrives in T055`).
   - Sticky footer (`safeAreaInset(.bottom)`) over a white gradient fade: summary + total on
     the left, `PrimaryButtonStyle` CTA on the right (spinner while purchasing), microcopy
     below. CTA drives `purchaseSelection`; `.success` calls the flow's `finish()` (T031),
     `.cancelled` stays silent, `.pending` shows an inline note, a thrown error shows an
     inline "Purchase failed — try again" — all without leaving the screen.
   - Wired into `OnboardingFlowView`'s `.plan` case in place of the `Text("Plan")` placeholder.

3. **Tests/PlanViewModelTests.swift** — loads the real Japan catalog via `ContentStore`
   (same pattern as T012's `ContentStoreTests`): weekend/week seeding, seeding-respects-set-id,
   pack row states + toggle + total/totalLabel, no-op toggle on a bundle member,
   summary/ctaTitle, and `purchaseSelection` success against a small in-memory
   `PurchaseProviding` stub (`InMemoryPurchaseService`, local to the test file — the shipping
   `StubPurchaseService` is DB-backed persistence-stack scaffolding, not a fit for a pure
   value-level view-model test, so a lightweight protocol conformance was used instead of
   duplicating persistence setup).

### Follow-up fix (same session)

Interactive visual verification (via `axe tap` against `./build_run.sh`'s simulator, working
around the lack of accessibility identifiers pre-T065 with hand-computed point coordinates)
surfaced a real layout bug: `BundleRowView`'s title + "POPULAR" badge wrapped onto multiple
lines ("Weeke"/"nd", "POPULA"/"R") instead of staying on one line. Fixed by adding
`.fixedSize()` to the title Text, the POPULAR badge Text, and the price Text, plus
`.fixedSize(horizontal: false, vertical: true)` on the subtitle, in
`Sources/Features/Onboarding/BundleRowView.swift` (in scope: this file lives under
`Sources/Features/Onboarding/**`). Re-ran the full DoD after the fix — format, lint, and
`xcodebuild test` (152/152 tests green) all still pass — and re-verified visually: the
"Weekend" bundle row now renders "Weekend" + "POPULAR" cleanly on one line with the correct
subtitle ("3 essential packs · save £0.51") and price (£2.49).

One further visual observation, not fixed: in this sandboxed simulator session the onboarding
content renders inside a visibly small viewport (the same ~300pt-tall white content box
appears identically on the pre-existing, unmodified Welcome and "Shape your trip" screens),
and within that constrained box the Plan screen's sticky footer visually overlaps the second
bundle row / pack checklist below the fold. Given the identical artifact on screens this task
never touched, and that `PlanView`'s footer uses the exact same `ScrollView` +
`.safeAreaInset(edge: .bottom)` pattern already shipped and working in T034's `TripShapeView`,
this reads as a sandbox/simulator display-scaling artifact rather than a real `PlanView` bug —
but it's noted here in case it turns out otherwise on a real device or a non-sandboxed
simulator.
