# T030 worklog

## 2026-07-16 — Build liquid-glass tab bar and main shell

### What I built
- **EnoughTab.swift** (`Sources/DesignSystem/`): Enum with 4 cases (learn, reviews, progress, browse), each with `title` and `symbol` properties. Used SF Symbols as specified: rectangle.stack.fill, arrow.trianglehead.2.clockwise.rotate.90 (available in iOS 18), chart.bar.fill, bag.fill.
- **LiquidGlassTabBar.swift** (`Sources/DesignSystem/`): Floating tab bar component with:
  - 4 equal-width tab items (symbol above 10pt semibold label)
  - Capsule background with `.ultraThinMaterial` + `Color.white.opacity(0.55)` overlay (not 0.82, to preserve blur visibility)
  - Shadow + inset hairline stroke per spec
  - Active tab styled with `accentTheme.accent`, inactive with `EnoughColor.inactiveTab` (#b0b0b6)
  - Light impact haptic on tap via `.sensoryFeedback(.impact(weight: .light), trigger: selection)`
- **Environment.swift** (`Sources/DesignSystem/`): `@Entry` Bool `isTabActive` (default true) for pausing infinite animations on hidden tabs
- **MainShellView.swift** (`Sources/App/`): Container with:
  - `@State private var selection: EnoughTab = .learn`
  - ZStack with all 4 tab screens alive simultaneously (opacity + allowsHitTesting approach to preserve `@State` across switches)
  - Each tab receives `\.isTabActive` environment value
  - Floating `LiquidGlassTabBar` at bottom with ~44 horizontal pad, 18 bottom pad
  - Placeholder screens (canvas background + large-title tab name) via `@ViewBuilder private func screen(for:)`
- **RootView.swift**: Replaced `.main` placeholder with `MainShellView()`
- **Moved EnoughTab** from App to DesignSystem to avoid circular dependency (DesignSystem component needs the enum)
