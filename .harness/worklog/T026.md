# T026 Worklog

## 2026-07-16

**Task:** PurchaseService protocol + stub implementation (buy, restore, owned stream)

**What was done:**

1. **PurchaseService.swift**: Created the purchase service protocol with:
   - `EntitlementKind` enum with `.deck` and `.bundle` raw values
   - `PurchaseOutcome` enum with `.success`, `.cancelled`, `.pending` cases
   - `PurchaseProviding` protocol defining the purchase interface
   - `Notification.Name.entitlementsChanged` extension for multicast entitlement change notifications

2. **StubPurchaseService.swift**: Implemented the v1 stub that:
   - Simulates store latency with a 300ms sleep
   - Grants entitlements via `EntitlementStore.grant()` (relying on idempotency)
   - Posts `.entitlementsChanged` notifications on the main actor after grant/restore
   - Implements `ownedProductIds` as a live read from the entitlement store
   - Implements `restorePurchases()` as a no-op with notification (real StoreKit deferred to T062)
   - Implements `displayPrice()` using `PricingCalculator.price()` for decks/bundles from the catalog

3. **StubPurchaseServiceTests.swift**: Comprehensive test suite with 6 async tests:
   - testPurchaseSingleDeck: verifies purchase returns .success, notification posted, product owned
   - testPurchaseSameDeckTwiceIsIdempotent: verifies idempotency (single EntitlementRecord)
   - testPurchaseBundle: verifies bundle purchase grants all member decks
   - testRestorePostsNotification: verifies restore doesn't alter owned products but posts notification
   - testDisplayPrice: verifies prices formatted correctly (£1.00 for decks, £2.49 for bundle)
   - testDisplayPriceWithNoCatalog: verifies nil return when catalog is absent
