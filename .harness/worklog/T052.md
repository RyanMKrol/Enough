# T052 — Progress components: streak card + week-dot row

## Summary

Built two pure presentational components for the Progress tab:
- **WeekDotsRow**: Renders a 7-dot week row with Monday-first day letters, supporting four dot states (done/amber-filled, today/accent-ring, upcoming/missed/faint-filled)
- **StreakCard**: Displays a streak card with a flickering flame animation, streak count text, motivational copy, and the week-dot row on an amber→white gradient

## Work done

1. Created `Sources/Features/Progress/WeekDotsRow.swift`
   - Pure presentational view receiving `dots: [DayDot]`
   - Renders HStack with VStack per day: 24pt circle + day letter (11pt Medium, tertiary text color)
   - Day letters: Monday-first `["M", "T", "W", "T", "F", "S", "S"]`
   - Circle styles per spec: done (amber fill + white checkmark), today (white fill + accent stroke), upcoming/missed (faint fill)
   - Preview covering all four dot styles

2. Created `Sources/Features/Progress/StreakCard.swift`
   - Pure presentational view receiving `streak: Int, dots: [DayDot]`
   - Card with 20pt radius, 18pt interior padding, resting shadow `0 1px 3px rgba(0,0,0,.05)`
   - Background: LinearGradient from `streakAmberTint` (top-leading) to white (bottom-trailing)
   - Top row: 26pt animated flame (flicker: 2.2s ease-in-out scale/rotate) + streak text (22 Bold)
   - Middle: "Keep it alive — 3 min today does it" (13 Regular, secondary text color)
   - Bottom: WeekDotsRow displaying passed-in dots
   - Preview with 6-day streak mock state (5 done, 1 today, 1 upcoming)

3. Verified all compilation & runtime requirements
   - Both views use injected state only; no service access or @Environment(\.services)
   - DayDot enum consumed from prerequisite T028 (StatsService)
   - Previews render without running app
