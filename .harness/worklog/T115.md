# T115: Group Reviews deck list by country/language

**Date:** 2026-07-20

## What was done

Restructured the flat deck strength list in ReviewsTabView to group decks by their parent country, making same-named decks from different packs visibly distinct.

### Changes made in `Sources/Features/Reviews/ReviewsTabView.swift`:

1. **Added state tracking** for country groupings:
   - `@State private var deckToCountry: [String: CountryInfo]` — maps each deckId to its parent country
   - `@State private var countryGroups: [(country: CountryInfo, deckIds: [String])]` — ordered groups per country

2. **Extended `reload()` function** (moved to extension):
   - While iterating `catalog.countries[].decks`, built `deckToCountryMap` alongside `deckInfoById`
   - After loading owned decks and progresses, calls `rebuildCountryGroups()` to organize decks by country

3. **Implemented `rebuildCountryGroups()` function** (in extension):
   - Prioritizes active trip's country first
   - Then adds other owned countries sorted alphabetically by name
   - Filters to only countries that have at least one owned deck
   - Returns groups as `[(country: CountryInfo, deckIds: [String])]`

4. **Rewrote `deckStrengthList()` function**:
   - Replaced flat `ForEach(ownedDeckIds)` with `ForEach(countryGroups)`
   - For each group, renders a section header ONLY if `countryGroups.count > 1` (suppresses redundant single-header)
   - Header shows `flagEmoji` + `nativeLanguageName` from CountryInfo
   - Each section's decks rendered via reused `deckStrengthRow()` unchanged
   - Dividers maintained between rows, smart logic prevents final divider

5. **Kept existing features unchanged**:
   - Global Due/Learning/Mastered counters (statTiles) unchanged
   - Hero card (with or without due items) unchanged
   - deckStrengthRow rendering unchanged
   - Accessibility identifiers for decks unchanged
   - Added new `country-section-<countryId>` identifier for section headers
