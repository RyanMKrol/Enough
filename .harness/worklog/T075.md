### Triage table

### Fix applied

- `Sources/Persistence/CardSRSStore.swift` — added `deleteAll(forDeck:)`, reusing the
  existing `records(forDeck:)` fetch + the store's own canonical context.
- `Sources/Features/DeckDetail/DeckDetailView.swift` — `resetProgress()` now calls
  `services.cardSRSStore.deleteAll(forDeck:)` instead of creating a second
  `ModelContext` via `services.container.mainContext`. No behavior change to what gets
  deleted, only which context performs the delete.
