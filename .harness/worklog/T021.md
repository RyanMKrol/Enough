# T021 — SRS queue builder

## 2026-07-16

Added `Sources/SRSKit/SRSQueueBuilder.swift` with `QueueEntry` and `SRSQueueBuilder`:
- `reviewQueue(from:now:limit:)` filters to due entries (via `SRSEngine.isDue`), sorts by
  `dueAt` ascending (stable — ties keep input order), then applies an optional `limit`.
- `learnBatch(newCardIds:size:)` returns the first `size` ids in input order.
- `requeuePosition(queueLength:)` returns `min(queueLength, 3)`.

Fixed a latent isolation bug uncovered while building this: `extension SRSEngine` in
`SRSQueries.swift` (added in T020) wasn't marked `nonisolated`, so under this project's
`-default-isolation=MainActor` build setting its members were implicitly MainActor-isolated
despite `SRSEngine` itself being a `nonisolated enum`. Calling `SRSEngine.isDue` from the new
nonisolated `SRSQueueBuilder` failed to compile until the extension was marked
`nonisolated extension SRSEngine`.

Added `Tests/SRSQueueBuilderTests.swift` (XCTest, fixed epoch dates) covering: due-only
filtering + soonest-first ordering (now−2h, now−1h, now exactly; future and nil-due
excluded), `limit` behavior, `learnBatch` truncation/pass-through, and the pinned
`requeuePosition` table (0→0, 1→1, 2→2, 3→3, 10→3).
