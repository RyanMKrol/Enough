# T049 — Flashcard review session screen (grades, live interval previews)

## 2026-07-16

Built the review session (REVIEW mode): flashcard + grade row with live SRS interval
previews, and wired Home's reviews banner to launch it.

### What I did

- `Sources/Features/Session/ReviewSessionViewModel.swift` — `@Observable
  ReviewSessionViewModel`: `isRevealed` (bindable, resets on advance), `previews:
  [GradeChoice: String]` built per-card from `SRSEngine.previewLabel(_:for:)` (pinned
  `.again→.again`, `.hard→.hard`, `.good→.good`, `.easy→.easy` — no hardcoded labels),
  `grade(_:)` (haptic → `engine.submitGrade` → advance), `advance()` (resets reveal,
  commits + routes to `.complete` when the engine finishes), `closeAndCommit()`. Progress
  text/value mirror the T047 MC pattern; audio auto-plays once per card via
  `services.audio.autoPlayOnce`.
- `Sources/Features/Session/ReviewSessionView.swift` — grey canvas (`EnoughColor.canvas`,
  not white), `SessionChrome` header, centered `FlashcardView` bound to
  `vm.isRevealed`, "How well did you know it?" + `GradeButtonRow` shown ONLY once
  revealed, pinned near the bottom. Same card-to-card transition as T047
  (`.id(currentCardId)` + asymmetric move transition). `navigationDestination` reuses the
  same T047 placeholder for `.complete` (T050 replaces it once).
- `Sources/Features/Home/HomeView.swift` — replaced the T044 `// TODO(T049)`: reviews
  banner now presents `ReviewSessionView(engine: try services.study.makeReviewSession())`
  as a `fullScreenCover`, mirroring the existing learn-session wiring.
- `Tests/ReviewSessionViewModelTests.swift` — seeds 3 due `CardSRSRecord`s (status
  `.learning`, `intervalDays = 1`, `dueAt` in the past) via `CardSRSStore.upsert`, builds
  a real `SessionEngine` through `StudyService.makeReviewSession()`. Asserts: (a)
  `previews` for the four grades match `SRSEngine.previewLabel` output directly (and
  equal the exact `["<1 min", "1 day", "3 days", "6 days"]` the spec calls for on a
  1-day-interval learning card); (b) `grade(.good)` records a result, clears
  `isRevealed`, and advances to a different card; (c) grading all three cards commits
  (SRS rows flip to the engine-computed post-Good state, activity row is written via
  `commit`) and sets `route == .complete`; (d) `closeAndCommit()` after grading 1 of 3
  persists exactly that one record (checked via `lastReviewedAt` timestamp, since the
  other two keep their seeded past `lastReviewedAt`).
