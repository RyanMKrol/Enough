# T065 — Accessibility identifiers (AXID)

## Summary

Added `Sources/App/AXID.swift`, a stable-string enum of accessibility identifiers, and wired
`.accessibilityIdentifier(...)` onto interactive elements and screen roots across the SwiftUI app
so UI-driving tools can navigate by id. Purely additive — no behavior/appearance changes.

## Work done

1. `Sources/App/AXID.swift` — `enum AXID` with camelCase `static let` constants for the tab bar
   (`tab-learn`, `tab-reviews`, `tab-progress`, `tab-browse`), screen roots (welcome, country,
   trip-shape, plan, home, deck-detail, session-mc, session-review, session-complete, reviews,
   progress, browse, debug), onboarding (`onboarding-continue`, `restore-purchases`,
   `countryCard(_:)`), home (`home-continue-card`, `home-reviews-banner`, `streak-pill`,
   `deckRow(_:)`), deck detail (`deck-continue`, `deck-practice`), sessions (`audio-button`,
   `flashcard`, four `grade-*`, `session-close`, `session-next`, `answerRow(_:)`), session
   complete (`complete-done`, `complete-learn-more`), reviews (`start-review`), and browse
   (`browse-restore`, `buy(_:)`).
2. Attached ids on the tappable element itself (never a wrapping stack) and screen ids on each
   screen's outermost view:
   - `Sources/DesignSystem/EnoughTab.swift` — added an `axID` computed property mapping each tab
     case to its `AXID` constant; `Sources/DesignSystem/LiquidGlassTabBar.swift` applies
     `tab.axID` to each tab button.
   - Onboarding: `WelcomeView` (screen root, Get started, Restore purchases),
     `CountryPickerView` (screen root, Continue), `CountryCardView` (`countryCard(country.id)`),
     `TripShapeView` (screen root, Build my plan), `PlanView` (screen root, purchase CTA reuses
     `onboardingContinue`).
   - Home: `HomeView` (screen root on the `NavigationStack`, `ContinueCard`, `ReviewsBanner`,
     each deck row via `deckRow(row.deck.id)`, `StreakPill`).
   - Deck detail: `DeckDetailView` (screen root, Continue learning, Practice all).
   - Sessions: `Sources/DesignSystem/GradeButtons.swift` — added a `GradeChoice.axID` computed
     property, applied per grade button; `Sources/DesignSystem/PulsingAudioButton.swift` —
     `audio-button` on its `Button`; `SessionChrome` — `session-close` on the close button;
     `FlashcardView` — `flashcard` on the card face container; `AnswerRow` usage in
     `MCSessionView` — `answerRow(index)` per row, plus `screen-session-mc` on the session
     root and `screen-session-complete` on the (still-placeholder, TODO(T050)) complete
     destination `Text`; `ReviewSessionView` — `screen-session-review` on the session root and
     the same `screen-session-complete` placeholder handling.
   - Reviews: `ReviewsTabView` — screen root and `start-review` on both the due and empty-state
     "Start review" buttons.
   - Browse: `BrowseView` — screen root, `browse-restore`, and `buy(productId)` on the buy
     button.
3. `Tests/AccessibilityIDsTests.swift` — XCTest asserting every `AXID` constant/helper equals its
   exact expected string (matches the `Tests/DesignTokensTests.swift` style: `XCTestCase`,
   `@testable import Enough`).
