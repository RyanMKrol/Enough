# T043 — Home components: continue card, reviews banner, deck list row

## Summary

Implemented three presentational components for the Home (Learn) screen as specified in the task:

1. **ContinueCard.swift** — accent-tinted gradient hero card with progress ring, deck name, detail line, and tappable play button
2. **ReviewsBanner.swift** — white surface card with circular arrow icon, review count, and review pill CTA
3. **DeckListRow.swift** — deck list row with icon tile, title, progress bar, and status indicator (learned/progress/new)

Also defined `DeckRowStatus` enum with three cases: `.learned`, `.progress(Int, Int)`, and `.new`.

## Implementation details

- Used existing DesignSystem types: `EnoughColor`, `EnoughFont`, `AccentTheme`, `ProgressRing`, `AnimatedProgressBar`, `EyebrowLabel`
- All three components accept only init parameters and read `@Environment(\.accentTheme)` — no services, persistence, or navigation
- Exact specifications:
  - ContinueCard: gradient from accent to accent@55%, radius 24, HStack(14) with 64pt ring, 44pt play button, deck name/detail
  - ReviewsBanner: white surface, radius 20, 34pt circular arrow on easyTint, "cards ready to review" + "Spaced repetition · keeps them stuck", Review pill
  - DeckListRow: 34pt icon tile, title, 4pt progress bar, status view (learned/progress/new variants)
- All color values match the design spec exactly (hex values via `EnoughColor`, secondaryText `#6b6b70`)
- Added #Preview blocks for all three components covering every state
