## 2026-07-15 — T017: CardSRSStore

Added `Sources/Persistence/CardSRSStore.swift`, the persistence gateway for per-card SRS rows
on top of the existing `CardSRSRecord` model (T014, unchanged): `record(deckId:cardId:)`,
`upsert(...)` (insert-or-mutate keyed by (deckId, cardId), exactly one row per pair), `records(forDeck:)`,
`dueRecords(now:ownedDeckIds:)`, and `reset()`.

`dueRecords` follows the decreed strategy: a `#Predicate` filters status != "new" and
`(dueAt ?? .distantFuture) <= now` with `sortBy: [SortDescriptor(\.dueAt)]` in the
`FetchDescriptor`, then the owned-deck filter is applied in memory (commented — `Set.contains`
inside `#Predicate` is a runtime-crash risk).

Added `Tests/CardSRSStoreTests.swift` (6 tests) covering every "Done when" acceptance item:
nil for untracked pair, insert via upsert, mutate-in-place on repeat upsert (row count stays 1),
`records(forDeck:)` deck isolation, `dueRecords` due/status/owned-deck filtering with the exact
ascending-order fixture from the spec, and `reset()` clearing both decks.
