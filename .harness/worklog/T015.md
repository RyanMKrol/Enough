# T015 worklog

## 2026-07-15

Built `TripStore`, the persistence gateway for the user's active trip on top of the T014
`TripProfileRecord` model:

- `Sources/Persistence/TripStore.swift` — `TripStore(context:)` with:
  - `activeTrip()` — fetches the single `isActive == true` row via a predicate + `fetchLimit = 1`.
  - `saveNewTrip(...)` — deactivates any existing active row(s), inserts + saves a new active
    `TripProfileRecord`, returns it (`@discardableResult`). Old trips stay as inactive history rows.
  - `dayNumber(now:)` — 1-based day counter from whole calendar days between
    `Calendar.current.startOfDay` of the trip's `startDate` and of `now`, clamped to a minimum of 1;
    `nil` when there's no active trip.
  - `reset()` — deletes all `TripProfileRecord` rows and saves.
- `Tests/TripStoreTests.swift` — hermetic tests (always `PersistenceStack.container(inMemory:
  true)` + a fresh `ModelContext`, fixed `DateComponents` dates, never the wall clock): empty-store
  nil, save + round-trip of the active trip, a second save deactivating the first (2 rows, exactly 1
  active), day-counter behaviour (day 1 on start, day 2 the next calendar day including late in that
  day, clamped to 1 for an earlier `now`, `nil` with no active trip), and `reset()` clearing all rows.
