# T077 — Axiom audit: SwiftUI architecture

## 2026-07-17

Ran the `axiom:swiftui-architecture-auditor` agent over `Sources/` and `Tests/`. It reported 13
findings (3 CRITICAL, 4 HIGH, 3 MEDIUM, 1 LOW) plus a health summary (view/VM ratio ~6:1,
"TANGLED" overall). Triaged every finding below; fixed the ones that were real, localized
defects with a small diff, deferred the ones that are either false positives or systemic
patterns whose "fix" would be a disproportionately large refactor for an audit sweep.

### Fixes applied

1. `Sources/Services/NotificationsService.swift` — added `rescheduleAfterActivity(services: AppServices) async`, the single source of truth for the review-due forecast + reschedule pipeline.
2. `Sources/App/RootView.swift` — `onChange(of: scenePhase)` now calls the shared method; removed the duplicated private `rescheduleReviewNotification()`.
3. `Sources/Features/Session/SessionCompleteView.swift` — `.onAppear` now calls the shared method; removed the duplicated private `rescheduleReviewNotification()` and the inline `getStrongestDeckTitle` selection algorithm (now delegates to `DeckProgressService.strongestDeckId(among:)`).
4. `Sources/Services/DeckProgressService.swift` — added `strongestDeckId(among:)`.
5. `Sources/ContentKit/ContentStore.swift` — added `countryId(forDeck:)`.
6. `Sources/Features/DeckDetail/DeckDetailView.swift` — uses `ContentStore.countryId(forDeck:)` instead of its own private `findCountryForDeck`.

### Tests added

- `Tests/NotificationsServiceTests.swift`: `testRescheduleAfterActivityForecastsFromLiveSRSState`.
- `Tests/DeckProgressServiceStrongestDeckTests.swift` (new file, to keep `DeckProgressServiceTests` under swiftlint's `type_body_length`): `testStrongestDeckIdPicksHighestStrengthThenMostLearned`, `testStrongestDeckIdSkipsUnreadableDecksAndReturnsNilWhenNoneMatch`.
- `Tests/ContentStoreTests.swift`: `testCountryIdForDeck`, `testCountryIdForDeckNotFound`.
