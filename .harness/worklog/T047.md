# T047 — Multiple-choice learn session screen worklog

## 2026-07-16 — Implementation complete

Built the full MC learn/practice session on top of `SessionEngine`/`StudyService`, T046's
`MCQuestionHeader`/`AnswerRow`, and `BottomResultSheet`, and wired its three launch points.

### New files
- `Sources/Features/Session/SessionChrome.swift` — shared top bar (X button, thin
  `AnimatedProgressBar`, "n/N" counter).
- `Sources/Features/Session/SessionRoute.swift` — `enum SessionRoute { case complete }`
  placeholder route for the `.navigationDestination`; T050 replaces the destination view.
- `Sources/Features/Session/MCSessionViewModel.swift` — `@Observable` view model:
  deterministic 4-option distractor selection (FNV-1a hash of the card id seeds a
  `SplitMix64` generator — no `hashValue` in the seeding path, so the same card always
  produces the same options across processes/launches), same-deck distractors with a
  same-country fallback when a deck has < 4 cards, answer/haptic/audio flow, progress
  text/value tied to engine progress (advances only on grade), advance/close-and-commit
  hooked into `StudyService.commit`.
- `Sources/Features/Session/MCSessionView.swift` — the session screen: `SessionChrome` +
  centered `MCQuestionHeader` + 4 `AnswerRow`s, green/red `BottomResultSheet` result copy
  (exact strings from the spec, bolded english via `AttributedString`), tap-anywhere/swipe-down
  to advance, card-to-card asymmetric slide transition keyed by card id, sensory feedback for
  tap + outcome, `.navigationDestination(for: SessionRoute.self)` placeholder for T050.
- `Tests/MCSessionViewModelTests.swift` — 5 tests against the real bundled content fixture
  (plus one scratch on-disk fixture bundle for the < 4 cards case, since every real deck has
  ≥ 18 cards): option determinism across two separately-constructed VMs, the small-deck
  fallback, progress text only advancing after a submitted answer, wrong-answer requeue
  within the session, and `closeAndCommit()` persisting only the graded-so-far SRS rows.

### Launch wiring
- `HomeView`: `ContinueCard.onPlay` now builds `services.study.makeLearnSession(deckId:)` and
  presents `MCSessionView` via `fullScreenCover`.
- `DeckDetailView`: "Continue learning" → the same learn session; "Practice all N" (N is now
  `deckInfo.cardCount`, the true deck size, rather than the 5-card sample cap) →
  `makePracticeSession(deckId:)`. Both present as `fullScreenCover`.
- `SessionEngine` isn't `Identifiable` and is out of this task's scope (`Sources/SRSKit/**`),
  so both call sites use a `Bool` + optional-engine pair with `.fullScreenCover(isPresented:)`
  rather than `.fullScreenCover(item:)`.
