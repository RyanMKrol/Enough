# T055 — Browse tab store (country sections, prices, buy & restore)

## Summary

Built the Browse tab store: `BrowseViewModel` (catalog/entitlement state, buy/restore flows)
and `BrowseView` (country sections, bundle cards, deck rows, restore + coming-soon footer),
wired into `MainShellView` in place of the `.browse` placeholder.

## Work done

1. `Sources/Features/Browse/BrowseViewModel.swift` — `@Observable final class BrowseViewModel`,
   `init(services:)`. State: `countries`, `comingSoon`, `ownedDeckIds`, `ownedProductIds`,
   `inFlightProductId`, `errorMessage`, `pendingProductId`, `showRestoredToast`.
   - `refresh()` reloads catalog + owned sets, active trip's country ordered first.
   - `buy(productId:kind:purchase:)` guards in-flight, handles `.success`/`.cancelled`/`.pending`,
     routes throws to `errorMessage`. `.cancelled` is a no-op (no alert, no state change).
   - `restore(purchase:)` sets `showRestoredToast` only on success; throw routes to `errorMessage`.
   - `price(for:fallbackGBP:)` uses `services.purchase.displayPrice` with `PricingCalculator`
     fallback.
   - Observes `.entitlementsChanged` (T026) via `NotificationCenter`, hopping to the main actor
     to call `refresh()`; removes the observer in `deinit`.
   - `buy`/`restore` accept an optional `purchase: PurchaseProviding?` override (same DI pattern
     as `PlanViewModel.purchaseSelection(purchase:)`) so tests can inject a fake without touching
     `AppServices`.
2. `Sources/Features/Browse/BrowseView.swift` — canvas background, large title "Browse", per
   country: header (flag + name + native-language `EyebrowLabel`), bundle cards (title, subtitle
   + savings via `PricingCalculator.bundleSavings`, Owned check or accent-tinted price pill), a
   grouped deck-row card (icon tile, title, card count, Owned check / price pill / in-flight
   spinner / pending note). Bottom: "Restore purchases" (`TextLinkButtonStyle`) + "Purchases
   restored" toast (auto-dismiss ~2s) + "More soon: …" footer joined from `comingSoon`. Errors
   surface via `.alert`.
3. `Sources/App/MainShellView.swift` — `.browse` case now renders `BrowseView()`.
4. `Tests/BrowseViewModelTests.swift` — catalog load, `.cancelled` → no state change, `.pending`
   → inline note + no grant, buy-while-in-flight is blocked, restore success shows toast, restore
   throw routes to `errorMessage`.
