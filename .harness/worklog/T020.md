# T020 — SRS queries worklog

## 2026-07-16

Implemented SRSQueries.swift with all required query functions for the SRS system:

1. **previewInterval**: Returns the TimeInterval in seconds that a given grade would produce when applied to a card state. Implemented by calling SRSEngine.apply() with a fixed reference date and extracting the resulting intervalDays.

2. **previewLabel**: Formats the interval into a human-readable label following spec rules:
   - < 1 hour → "<1 min"
   - 1 hour to < 1 day → "<1 day"
   - 1 day exactly → "1 day"
   - < 30 days → "\(n) days"
   - ≥ 30 days → rounds to months ("1 month" or "\(m) months")

3. **isDue**: Checks if a card is due by verifying dueAt is not nil and <= now. New cards with nil dueAt are not due.

4. **isMastered**: Checks if a card is in review status with intervalDays >= 21.

5. **nextDueLabel**: Formats a future due date for display:
   - Past dates → "now"
   - < 1 hour away → "in \(minutes) minutes" (singular "in 1 minute")
   - < 24 hours away same day → "in \(hours) hours" (singular "in 1 hour")
   - Next calendar day → "tomorrow"
   - Further away → "in \(days) days"
