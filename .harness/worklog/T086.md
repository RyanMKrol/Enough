# T086 — Axiom audit: Codable safety (codable-auditor)

**Date:** 2026-07-17

## What I did

Ran the `axiom:codable-auditor` agent over the whole repo (`Sources/`, `Tests/`). It mapped the
entire Codable surface: 6 types, all in `Sources/ContentKit/Models.swift`
(`ContentCatalog`, `ScenarioInfo`, `CountryInfo`, `DeckInfo`, `BundleInfo`, `DeckCards`,
`CardContent`), decoded exclusively via two `JSONDecoder()` call sites in
`Sources/ContentKit/ContentStore.swift` (`catalog()`, `cards(forDeck:)`). The `Persistence/`
layer (SwiftData) doesn't use Codable at all, so it's out of scope. No network boundary, no
encode path, no round-trip pairs.

## Triage table

| Finding | Verdict | Reason |
|---|---|---|
| No `JSONSerialization` in `Sources/` | false-positive / non-issue | Auditor confirmed zero production `JSONSerialization` use; the only hits are in `Tests/` fixture-writing helpers, not production code. |
| No `try?` decode sites swallowing errors in `ContentStore.catalog()`/`cards(forDeck:)` | false-positive / non-issue | Both decode call sites `throw`, they don't swallow. The widespread `try? services.contentStore.catalog()` at call sites (HomeViewModel, ProgressViewModel, BrowseViewModel, etc.) is a pre-existing, intentional convention for a static bundled read-only content pipeline — out of this task's domain focus per the spec ("known-intentional choices: try? decoding that swallows errors"). |
| No `dateDecodingStrategy`/`keyDecodingStrategy` set | false-positive / non-issue | No `Date` fields exist on any Codable type and all JSON keys already match camelCase properties 1:1 — nothing to configure, no drift risk. |
| No Codable-conforming `String` enums, so no future-case-crash risk | false-positive / non-issue | `EntitlementKind`, `SRSGrade`, `DayDot`, `AccentTheme`, `EnoughTab` are String-backed but none conform to `Codable` and none are JSON-decoded. |
| **[LOW] Missing-resource error path untested for malformed fixture JSON** (`ContentStore.swift:33,68`) | **fixed** | No test asserted what happens when a `Content/*.json` fixture is missing a required key (e.g. a future content-authoring mistake). No CRITICAL, HIGH, or MEDIUM findings. Health score: **SAFE**.
