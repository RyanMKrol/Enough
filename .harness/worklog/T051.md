# T051 — Reviews tab worklog

## 2026-07-16

**Built ReviewsTabView and StrengthBars components.**

### Implementation

1. Created `Sources/Features/Reviews/StrengthBars.swift`: A simple view that displays 0–3 filled green bars with grey remainders. Includes a preview showing all strength levels.

2. Created `Sources/Features/Reviews/ReviewsTabView.swift`: The main Reviews tab screen with:
   - Large title "Reviews" with subtitle "Keeping <destination> fresh" (destination computed from active trip's country: japan→Tokyo, france→Paris, germany→Berlin)
   - Hero card showing:
     - ProgressRing (120pt) displaying progress as due/(due+learning) with centered text overlay showing due count + "due now"
     - When due > 0: subtext "A quick <m>-minute round keeps these from slipping" (m = ceil(due * 20 / 60))
     - "Start review" button that creates a ReviewSessionEngine and presents ReviewSessionView full-screen
   - When due == 0: Empty state showing "Nothing due right now" with optional next-due date using relative formatting, disabled button
   - Three stat tiles (Due/Learning/Mastered) with counts in red/amber/green
   - Per-deck strength list showing StrengthBars, deck title, and due count per owned deck
   - Data wired from services: deckProgress.totals(), deckProgress.progress(forDeck:), entitlementStore.ownedDeckIds()

3. Updated `Sources/App/MainShellView.swift` to wire ReviewsTabView into the `.reviews` tab case.

4. Fixed linting violations:
   - Added `// swiftlint:disable:next large_tuple` for the 3-tuple dueCounts state
   - Added `// swiftlint:disable:next inclusive_language` for masteredCount variable (term inherited from DeckProgressService)
