# T011 — Content schema models (catalog, country, deck, bundle, card)

## 2026-07-15

Added `Sources/ContentKit/Models.swift` with the Codable value types specified in the task
(`ContentCatalog`, `ScenarioInfo`, `CountryInfo`, `DeckInfo`, `BundleInfo`, `DeckCards`,
`CardContent`) — property names/types match the JSON contract verbatim, no custom CodingKeys.
Each struct is marked `nonisolated` because the app target defaults new declarations to
`@MainActor` (`SWIFT_DEFAULT_ACTOR_ISOLATION: MainActor` in `project.yml`), which would otherwise
make `JSONDecoder().decode(...)` unusable from the nonisolated test target.

Added `Tests/ContentModelsTests.swift` (XCTest) decoding inline JSON literals:
- `testContentCatalogDecodes` — full catalog sample (2 scenarios, 1 country with 1 deck + 1
  bundle, 1 comingSoon entry), asserts version/ids/priceGBP/deckIds/popular.
- `testDeckCardsDecodesOptionalKeys` — two cards, one with `notes`/`category` present (asserted
  non-nil), one with both keys omitted entirely (asserted nil) — proves the optionals tolerate
  absent keys rather than requiring explicit `null`.
- `testUnicodeSurvivesDecoding` — a card with `target: "おはようございます。"` round-trips through
  `JSONDecoder`.
