# T058 — New-trip flow: start/switch trips, accent reskin, owned packs persist

## 2026-07-17

**What I did**

- `TripStore.deactivateActiveTrip()` — deactivates the active trip but KEEPS the record
  (trip history), unlike `reset()` which deletes.
- `AppState.startNewTrip()` — deactivates the active trip, resets `cardSRSStore` +
  `activityStore`, cancels pending review notifications, sets `isReonboarding = true`, and
  flips `phase = .onboarding`. Never touches entitlements/purchases.
  `completeOnboarding()` clears `isReonboarding`.
- `RootView` now injects `appState` into the environment (`.environment(appState)`) and
  passes `startAtCountryPicker: appState.isReonboarding` to `OnboardingFlowView`.
- `OnboardingFlowView.init(appState:startAtCountryPicker:)` — when `true`, the flow's root
  is `CountryPickerView` directly (skipping `WelcomeView`); `CountryPickerView` gained a
  `showsBackButton` flag (default `true`) that hides/disables the back button in this mode.
- `ProgressTabView` — added a "New trip" text button (13pt semibold, `linkBlue`) top-right
  of the title row, wired to a `.confirmationDialog` ("Start a new trip?" /
  "Your packs stay yours. Progress and streaks reset." / destructive "Start new trip" +
  "Cancel"). Reads `AppState` via `@Environment(AppState.self)`.
- Added `AXID.progressNewTrip` and a new `AppStateTests` case:
  `testStartNewTripDeactivatesTripWipesProgressKeepsEntitlements`.
