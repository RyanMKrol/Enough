## Fix applied

`Sources/Features/Reviews/ReviewsTabView.swift`:
- Added `@State private var deckInfoById: [String: DeckInfo] = [:]`.
- `reload()` now builds `deckInfoById` once from `catalog.countries` alongside the existing
  `deckProgresses` build.
- `deckStrengthRow` reads `deckInfoById[deckId]` instead of calling the removed
  `getDeckInfo(_:)`, which previously did a nested linear scan over `catalog.countries` /
  `country.decks` on every row render.

No other findings required a code change (all verified false positives, reasoning above).
