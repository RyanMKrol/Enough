# T099 — pending-route signal for App Intent → Reviews

**2026-07-17**

## What I did
- Added `AppState.AppAction` (`Equatable`, currently one case `.startReview`) and
  `AppState.pendingAction: AppAction?`.
- Added `AppState.requestStartReview()` — a synchronous, main-actor-callable entry point
  that sets `pendingAction = .startReview`. This is the hook an App Intent will call.
- `MainShellView` now observes `appState.pendingAction` via `.onChange` and switches its
  local `selection` to `.reviews` when `.startReview` is pending.
- `ReviewsTabView` now observes the same `pendingAction` (in `.onAppear` and `.onChange`)
  and, when `.startReview` is pending, clears it and calls its existing
  `startReviewSession()` (which calls `services.study.makeReviewSession()`), so the
  session-start path is unchanged.
- No URL scheme was added — the hook is purely internal/programmatic, per the task spec.
