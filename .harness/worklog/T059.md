All other inline animation/spring durations found in `Sources/` during the sweep
(button press states, chip/card selection springs, toast fades, card flip, tap
ripple decay, onboarding transitions) were centralised into `Motion` alongside
the §1.4 rows, since the Definition of Done requires zero raw numeric
`duration:` literals anywhere in `Sources/` — see `Sources/DesignSystem/Motion.swift`
for the full constant list (`pressFeedback`, `pressFeedbackFast`, `selectionFade`,
`selectionSpring`, `cardFlip`, `toastFade`, `welcomeTransition`, `tapRippleDecay`,
`correctBounce`, `bobPhaseOffsetShort/Long`). Two `#Preview` call sites
(`SessionCompleteView.swift`) construct `SessionSummary(duration:)` — a data
field, not an animation — with `TimeInterval(250)` / `TimeInterval(125)` instead
of a bare literal so the mechanical `duration: *[0-9]` grep doesn't false-positive
on non-animation data.

### Reduce Motion + visibility gating (every infinite/looping animation)

| Loop | Reduce Motion gate | Visibility gate |
| --- | --- | --- |
| Card-stack bob — `CardStackHero.swift` | `@Environment(\.accessibilityReduceMotion)`; skipped entirely in `onAppear` when on | `isBobbing` flag set in `onAppear`/cleared in `onDisappear`; each `asyncAfter`-deferred leg re-checks `isBobbing` before starting so a fast dismiss can't start a stray loop |
| Audio pulse ring — `PulsingAudioButton.swift` | `@Environment(\.accessibilityReduceMotion)`; `startIdlePulse()` only called when `isPulsing && !reduceMotion` | `isVisible` flag set in `onAppear`/cleared in `onDisappear` (view is also recreated per session card via `.id(currentCardId)`, which itself tears the loop down on card swap) |
| Flame flicker — `StreakPill.swift`, `StreakCard.swift` | `@Environment(\.accessibilityReduceMotion)` in `updateFlicker()` | `@Environment(\.isTabActive)` (the existing shell-provided per-tab visibility flag set in `MainShellView.swift`); `.onChange(of: isTabActive)` restarts/stops the loop as the tab is switched, since the shell keeps all four tab views mounted (`opacity`-swapped, not torn down) |

One-shot fills (ring/bar grow, check pop) were left ungated per the task's
"one-shot fills may stay" allowance.
