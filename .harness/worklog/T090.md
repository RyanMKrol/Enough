# T090: Session complete screen (ring race, check pop, stats, learn-5-more)

## 2026-07-16 — Initial Implementation

### What was done

Implemented SessionCompleteView with the following features:

1. **SessionCompleteView.swift** — New view with:
   - Hero section: 120pt ProgressRing (fills to 100%) with CheckPopView (44pt) overlaid, check pops at 0.3s delay
   - Title: "Round complete" (30pt bold)
   - Subline: Review variant shows "You cleared today's reviews. {strongestDeck} is holding strong."; learn variant shows "That's {deck} under way. {n} cards down."
   - Stat tiles (3 equal-width cards, 14pt padding, hairline border, 18pt radius): cards completed, duration formatted as m:ss, current day streak
   - Actions: Primary "Done" button (accent color) that calls onDone to dismiss the full-screen cover; TextLinkButtonStyle "Learn 5 more" shown only when new cards exist (via makeLearnMoreSession check)
   - Accessibility: screenSessionComplete identifier
   - Motion: ProgressRing animates to full on appear (Motion.ringFill), CheckPopView pops at 0.3s (Motion.popSpring), no loops

2. **MCSessionView.swift** — Replaced placeholder `Text("Complete")` with SessionCompleteView call:
   - Passes `engine.summary()` as summary
   - Passes `engine.current?.deckId` as deckId (for learn/practice sessions)
   - Passes `{ dismiss() }` as onDone to use the view's inherited @Environment(\.dismiss)
   - Removed TODO(T050) and swiftlint:disable:next todo

3. **ReviewSessionView.swift** — Replaced placeholder with SessionCompleteView call:
   - Passes `engine.summary()` as summary
   - Passes `nil` for deckId (review sessions review multiple decks)
   - Passes `{ dismiss() }` as onDone

### How it works

- When MCSessionViewModel or ReviewSessionViewModel detects engine.isComplete, it sets route = .complete
- NavigationStack's .navigationDestination(for: SessionRoute.self) catches this and renders SessionCompleteView
- SessionCompleteView reads the strongest deck (via DeckProgressService) for review mode or the current deckId for learn mode
- Streak is fetched via StatsService.currentStreak() at appear time
- The hero ring animates on appear, check pops after 0.3s delay
- "Learn 5 more" attempts to create a 5-card learn session via services.study.makeLearnMoreSession(size: 5)
