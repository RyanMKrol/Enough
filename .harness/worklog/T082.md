# T082 — Axiom audit: energy & battery (energy-auditor)

Date: 2026-07-17

## What I did

Ran the `axiom:energy-auditor` agent over the whole repo (`Sources/` + `Tests/`), covering
timers, polling, location, background modes, network efficiency, GPU/animation lifecycle,
disk I/O, and audio session lifecycle.

## Triage table

| Finding | Verdict | Reason |
|---|---|---|
| Audio session activated (`setActive(true)`) in `AudioService.resetAutoPlay()` but never deactivated in production code — only in tests (`AudioService.swift:83-96`, called from `MCSessionViewModel.init`/`ReviewSessionViewModel.init`) | **Fixed** | Confirmed asymmetric start/stop lifecycle: every review/MC session activates the shared `AVAudioSession` but nothing in production called `sessionEnded()` to deactivate it. Session stayed active indefinitely after session completion or early exit. |
| No timers/polling anywhere in `Sources/` | False positive / not applicable | Zero `Timer.scheduledTimer`/`CADisplayLink`/interval-polling usage exists in the codebase. |
| No location services | Not applicable | App has no location feature. |
| No background modes registered, no `BGTaskScheduler` usage | Not applicable | No background execution anywhere. |
| No networking layer (`URLSession`) | Not applicable | App is fully offline (SwiftData + bundled content + stubbed StoreKit). |
| 4 `repeatForever` continuous animations (`StreakCard`, `StreakPill`, `PulsingAudioButton`, `CardStackHero`) | False positive | All 4 are correctly gated by `onAppear`/`onDisappear` and/or the `isTabActive` environment key plus `accessibilityReduceMotion` — no leak. |
| `NotificationsService` reschedule pattern | False positive | Event-driven (fires on session completion/foregrounding), not polling. |

## Fix

Wired `services.audio.sessionEnded()` into the actual end-of-session lifecycle in both
`ReviewSessionViewModel` and `MCSessionViewModel`:
- In `advance()`, right after `engine.isComplete` triggers `route = .complete` (natural session
  end).
- In `closeAndCommit()` (early exit / user backs out of the session mid-way).

This mirrors the `resetAutoPlay()`/`sessionEnded()` pairing already exercised by
`Tests/AudioServiceTests.swift`.
