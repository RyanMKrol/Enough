# T076 — Axiom audit: Swift 6 concurrency (concurrency-auditor)

## 2026-07-17

Ran the `axiom:concurrency-auditor` agent over the whole repo (Sources/ + Tests/) for Swift 6
strict-concurrency violations, per `.harness/tasks/T076.md`. Cross-checked its findings by hand,
plus a manual grep sweep for `@unchecked Sendable` / `nonisolated(unsafe)` / `@preconcurrency` /
`Task.detached` / `DispatchQueue` across `Sources/`.

### Audit summary (agent report)

- Isolation architecture: `project.yml` sets `SWIFT_DEFAULT_ACTOR_ISOLATION: MainActor` for the
  `Enough` target (Swift 6.0) — every type in `Sources/` is implicitly `@MainActor` unless marked
  `nonisolated`. No `actor` types exist; `nonisolated` is used deliberately for pure value
  types/algorithms (`SessionEngine`, `SRSEngine`, `ContentKit/Models.swift`, etc.) — a coherent,
  intentional split, not a gap.
- No `StoreKitPurchaseService` exists yet (StoreKit not built), so the spec's "sanctioned detached
  listener" carve-out doesn't apply — no `Task.detached` usage found anywhere in `Sources/`.
- 2 confirmed escape hatches: `AudioService: @unchecked Sendable` (+ `@preconcurrency import
  AVFoundation`) and `BrowseViewModel.entitlementsToken: nonisolated(unsafe)`.
- 5 `DispatchQueue.main.asyncAfter` call sites (animation-timing) inconsistent with the rest of the
  codebase's `Task { try? await Task.sleep(...) }` pattern.
- Verdict: NEEDS WORK (no CRITICAL findings; 1 HIGH, 1 HIGH/MEDIUM, 2 MEDIUM, 2 LOW).

### Fixes applied

- `Sources/Services/AudioService.swift` — removed `@unchecked Sendable` conformance and
  `@preconcurrency import AVFoundation`; restructured the interruption-notification closure to
  extract the `Sendable` interruption value before entering `MainActor.assumeIsolated` for the
  `currentPlayer` mutation; added a `// SAFE:` comment on the (still-necessary)
  `nonisolated(unsafe)` `interruptionObserverToken` used for deinit teardown.
- `Sources/Features/Browse/BrowseViewModel.swift` — added a `// SAFE:` comment justifying
  `entitlementsToken`'s `nonisolated(unsafe)` (confirmed required for `deinit` access; the
  escape hatch stays, now documented per the audit's own recommendation).
- `Sources/DesignSystem/CardStackHero.swift`, `Sources/DesignSystem/PulsingAudioButton.swift`,
  `Sources/Features/Session/AnswerRow.swift`, `Sources/Features/Browse/BrowseView.swift` — replaced
  5 `DispatchQueue.main.asyncAfter` call sites with `Task { try? await Task.sleep(for: .seconds(...)) }`,
  matching the codebase's established delayed-UI-update pattern. No behavior change (both land back
  on the main actor before mutating state).
