# T069 — Accessibility conformance sweep (VoiceOver, Dynamic Type, Reduce Motion, hit targets)

## 2026-07-17 — built cold from spec

### What I did

An accessibility sweep across the design system + every screen, adding VoiceOver
labels/values/traits to custom controls, hiding decorative art, enlarging small hit
targets to ≥44pt, and confirming Reduce Motion gating. Reduce Motion was already
handled by T059 (every infinite animation gates on `accessibilityReduceMotion`) — I
verified and left it.

**VoiceOver (labels / values / traits):**
- `PulsingAudioButton` → `.accessibilityLabel("Play audio")` (+ 44pt tap frame).
- `StreakPill` → combined element, label `"\(count)-day streak"` (flame ignored).
- `ProgressRing` / `AnimatedProgressBar` → `.accessibilityValue("\(percent) percent")`.
- `GradeButtonRow` → each button: label = title, value = interval preview.
- Tab bar items → label = tab name, `.isSelected` trait when active.
- `ScenarioChip`, `CountryCardView`, `BundleRowView`, `PackChecklistRow` → `.isSelected`
  trait when selected; decorative radio/checkmark/flag marked `.accessibilityHidden`.
- `DeckListRow` + deck-detail sample rows → `.accessibilityElement(children: .combine)`
  so each row reads as one sentence; decorative icon/checkmark hidden.
- `CardStackHero` (welcome art) → `.accessibilityHidden(true)`.
- `StreakCard` flame hidden; `WeekDotsRow` collapsed to one summary label
  ("This week: N of 7 days practiced") instead of reading five raw checkmarks.
- Icon-only buttons labeled: SessionChrome close → "Close", DeckDetail back → "Back",
  DeckDetail menu → "More options", OnboardingBackButton → "Back", BottomResultSheet
  chevron → "Continue". (ContinueCard play already had "Continue learning".)

**Hit targets (≥44×44):** PulsingAudioButton (34pt variant), SessionChrome close
(30pt), DeckDetail back/menu (36pt), OnboardingBackButton (34pt), ProgressTabView
"New trip" — all given `.frame(min… 44)` + `.contentShape(Rectangle())`. The 22pt
checks/radios sit inside full-width row buttons (row itself ≥44) so they need no change.

**Dynamic Type:** `EnoughFont` styles (largeTitle/caption used for screen titles &
eyebrows) DO scale; the fixed `.system(size:)` body text does not, by design (the
design-spec pins exact point sizes). Defensive fixes: GradeButtonRow height →
`minHeight`. Walked every screen at accessibility-extra-large — titles/eyebrows grow
and reflow cleanly with no clipping or overlap.
