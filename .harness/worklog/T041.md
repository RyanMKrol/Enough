# T041 Worklog

## 2026-07-16 — Initial implementation

### What was done

Implemented the debug menu action to reset all app data (factory reset). This includes:

1. **Created `Sources/Services/AppReset.swift`**
   - `AppReset.wipeAll(context:userDefaults:)` method that:
     - Deletes all rows from TripProfileRecord, EntitlementRecord, CardSRSRecord, DailyActivityRecord using `context.delete(model:)`
     - Saves the context
     - Removes the "debug.dayOffset" key from UserDefaults to reset time-travel offset

2. **Created `Sources/Features/Debug/DebugReset.swift`**
   - Defines `DebugReset` enum with a static `section` property
   - Section has id="reset", title="Danger zone"
   - Single `.action` row that calls `AppReset.wipeAll()` and returns "Data wiped — relaunch the app" toast message
   - Row has id="reset-all", title="Reset all data", subtitle="Wipes trip, purchases, SRS progress and history"

3. **Updated `Sources/Features/Debug/DebugRegistry.swift`**
   - Added `DebugReset.section` to the `DebugProviders.all` array (last in list so "Danger zone" renders at bottom)

4. **Created `Tests/AppResetTests.swift`**
   - `AppResetTests` class with one comprehensive test: `testWipeAllDeletesAllRecords()`
   - Test inserts one record of each type (TripProfileRecord, EntitlementRecord, CardSRSRecord, DailyActivityRecord)
   - Sets debug.dayOffset in a throwaway UserDefaults instance
   - Calls `AppReset.wipeAll()`
   - Verifies all four record types are empty after reset
   - Verifies the debug.dayOffset key was removed from UserDefaults
