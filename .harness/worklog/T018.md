## 2026-07-15

Built `ActivityStore`, the persistence gateway for the per-day study activity log
(`DailyActivityRecord` from T014), following the `CardSRSStore` / `EntitlementStore` pattern
already in `Sources/Persistence/`.

What I did:
- Added `Sources/Persistence/ActivityStore.swift` with `record(for:)`, `addActivity(...)`
  (accumulates into today's row, creating it if absent), `all()` (ascending by day), and
  `reset()`. Day normalization uses `Calendar.current.startOfDay(for:)` on both the write path
  (`addActivity` stores `startOfDay(now)`) and the read path (`record(for:)` compares against
  `startOfDay(day)`).
- Added `Tests/ActivityStoreTests.swift` (XCTest) covering: empty store, single insert with
  normalized day, same-day accumulation (1 row, summed counters), next-day creates a second row
  with ascending `all()` order, `record(for:)` finds the row at 23:59 on day1, and `reset()`
  empties the store. Fixed dates built from `DateComponents` so results don't depend on when
  the suite runs.
