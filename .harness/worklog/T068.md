# T068: Store URL isolation seam

## Date: 2026-07-16

### What was done

Added `storeURL(appGroupID:)` static function to `PersistenceStack` to isolate on-disk store path configuration behind a single function. This enables future widget support which requires the store in an App Group container.

**Implementation:**
- Added `nonisolated static func storeURL(appGroupID: String? = nil) -> URL` to PersistenceStack
  - Default path (nil): `URL.applicationSupportDirectory` + `"Enough/Enough.store"`, creates intermediate directory
  - App group path: Uses `FileManager.default.containerURL(forSecurityApplicationGroupIdentifier:)` + `"Enough.store"`, fatal-errors if entitlement missing
- Updated `container(inMemory:)` to use `storeURL()` for on-disk branch (line 37)
- In-memory container unchanged
- Added widget adoption comment per spec
- Created `Tests/StoreURLTests.swift` with 4 tests:
  - URL ends with `Enough/Enough.store` in Application Support
  - Calling twice returns same URL
  - Directory exists after call
  - Signature accepts appGroupID parameter
