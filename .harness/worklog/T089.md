**The swift-format vs swiftlint trailing-comma conflict (why this re-attempt exists)**

The prior T089/T037 attempt hit swift-format --strict (requires a trailing comma on the
last element of a *multiline* collection literal) directly conflicting with swiftlint
--strict's `trailing_comma` rule (flags that same comma) for the two-element `rows: [...]`
array in `DebugAbout.swift`. Resolution used here, matching the existing convention in
`GradeButtons.swift` / `ContinueCard.swift` / `StreakCard.swift`: keep the trailing comma
but suppress the lint rule with `// swiftlint:disable:this trailing_comma` **on the exact
line bearing the comma** (disable:this only applies to that line — placing the comment on
the closing `]` line instead does not suppress it, which is what tripped up the middle
attempt). For the single-element `DebugProviders.all` array, swift-format instead wants to
collapse it to one line and then flags a trailing comma on a *single-line* literal — so for
single-element arrays the fix is to just write it on one line
(`static let all: [DebugSection] = [DebugAbout.section]`), which has no comma at all.
Added a LIMITATIONS row recording this as the standing resolution.
