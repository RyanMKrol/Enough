# T005 — Progress indicators: animated progress bar + progress ring

## Overview

Add the two progress components used across Home, deck detail, sessions, and the Reviews tab: a
thin bar that grows on appear and a trim-based ring that fills on appear. Motion values from
design-spec.md §1.4.

## Do

1. Create `Sources/DesignSystem/ProgressViews.swift` with exactly these two views:
   - `struct AnimatedProgressBar: View` with `init(progress: Double, tint: Color,
     trackColor: Color = Color.black.opacity(0.08))`. `progress` is 0…1 (clamp internally).
     A capsule track 5pt tall; the fill is a leading-aligned capsule whose width =
     `progress × available width` (GeometryReader). On appear, animate an internal
     `displayedProgress` from 0 to `progress` with `.easeOut(duration: Motion.barGrow)` (1.0s) —
     the "grows scaleX 0→1 anchored left" behavior. If `progress` later changes, animate to the
     new value with the same easing.
   - `struct ProgressRing: View` with `init(progress: Double, size: CGFloat,
     lineWidth: CGFloat = 6, tint: Color, showsPercent: Bool = true)`. `progress` is 0…1
     (clamp). A `Circle` track stroked in `tint.opacity(0.15)` at `lineWidth`, plus a
     `Circle().trim(from: 0, to: displayedProgress)` stroked in `tint` with round line caps,
     rotated −90° so fill starts at 12 o'clock. On appear, animate `displayedProgress` 0 →
     `progress` with `.easeOut(duration: Motion.ringFill)` (1.2s); animate subsequent changes
     too. When `showsPercent`, center a label `"\(Int((progress * 100).rounded()))%"` in
     `.system(size: size * 0.28, weight: .bold)`, color `tint`. Overall frame `size × size`.
2. Add a `#Preview` showing: a bar at 0.27 in `AccentTheme.japan.accent` on a white card, a
   64pt ring at 0.4, and a 120pt ring at 1.0 without percent label.

Conventions: single app target, SwiftUI only, 2-space indent, SwiftLint strict (≤130 cols) +
swift-format strict. No business logic; values in, pixels out. Touch only
`Sources/DesignSystem/**` (and `Tests/**` if adding optional tests).

## Done when

- `Sources/DesignSystem/ProgressViews.swift` compiles with the exact two view names and
  initializer signatures above (later tasks call them verbatim — e.g.
  `ProgressRing(progress: 0.4, size: 64, tint: accentTheme.accent)`).
- Both components animate from empty to their value on first appearance using
  `Motion.barGrow` / `Motion.ringFill` durations, and clamp out-of-range progress values
  (−0.1 → 0, 1.3 → 1) rather than crashing or overdrawing.
- The ring's percent label shows an integer percent (0.27 → "27%") and can be disabled via
  `showsPercent: false`.
- The `#Preview` block exists with the three variants described.
