## Overview

Assemble the "Your plan" screen: seed a recommended bundle + packs from the interview answers,
let the user freely edit (bundle choice, à-la-carte packs), show computed totals/savings, and
purchase everything through `PurchaseService` before finishing onboarding into the main shell.

## Do

1. Read design-spec.md §2.4 fully and architecture.md. Use the T035 rows
   (`BundleRowView`, `PackChecklistRow`) and T025 `PricingCalculator` EXACTLY as they exist —
   all money strings/savings come from `PricingCalculator`; this screen never does arithmetic
   on `priceGBP` itself.
2. Create `Sources/Features/Onboarding/PlanViewModel.swift` — `@Observable final class
   PlanViewModel`:
   - `init(country: CountryInfo, draft: OnboardingDraft)`.
   - SEEDING (on init, only if `draft.selectedBundleId == nil`): `draft.duration == .weekend`
     ⇒ select the country's bundle whose `deckIds.count < ` the all-packs bundle's (in the
     fixture: "Weekend", `popular == true`); `.week` ⇒ the bundle containing every deck
     ("The whole week"). Store into `draft.selectedBundleId`.
   - Derived, always computed from the catalog + draft (never cached):
     `selectedBundle: BundleInfo?`, `packRows: [(deck: DeckInfo, state: PackRowState)]` —
     ordering: decks in the selected bundle first (catalog order), then the rest; state:
     member ⇒ `.inBundle(bundleName: selectedBundle.title)`, in `draft.extraDeckIds` ⇒
     `.extra(priceLabel: "+ " + PricingCalculator.price(deck.priceGBP))`, else
     `.available(...)` with the same label.
   - `bundleSubtitle(for:)` — "<bundle.subtitle> · save <PricingCalculator.price(savings)>"
     using `PricingCalculator.bundleSavings(_:in:)` (e.g. "3 essential packs · save £0.51").
   - `func selectBundle(_ id: String)` — sets draft.selectedBundleId; removes any
     `extraDeckIds` now covered by the bundle. `func togglePack(_ deckId: String)` — no-op for
     bundle members; otherwise toggles in `draft.extraDeckIds`.
   - `total: Double` via `PricingCalculator.planTotal(...)`; `totalLabel: String`;
     `summary: String` — "<bundle title> · <n> packs" counting bundle members + extras (e.g.
     "Weekend · 3 packs"), or "<n> packs" when no bundle.
   - `ctaTitle: String` — "Get the bundle" when a bundle is selected, else "Get <n> packs".
   - `func purchaseSelection(purchase: PurchaseProviding) async throws` — purchases the
     selected bundle (`kind: .bundle`) then each extra deck (`kind: .deck`).
3. Create `Sources/Features/Onboarding/PlanView.swift` — `struct PlanView: View` (reads
   `\.services`, the shared `OnboardingDraft`, the flow's finish hook from T031):
   - Toolbar: shared `OnboardingBackButton`. Header: title "Your plan" (30–32 bold), subtitle
     exactly "Built from your answers — pick a bundle or add packs à la carte." (15 `#6b6b70`).
   - Bundle rows (spacing 10): one `BundleRowView` per country bundle — title, subtitle via
     `bundleSubtitle(for:)`, price `PricingCalculator.price(bundle.priceGBP)`,
     `isPopular: bundle.popular`, selected binding via `selectBundle`.
   - Divider row: hairlines either side of "or choose packs" (13 `#8e8e93`).
   - Pack checklist inside one white card (radius 18): `PackChecklistRow` per `packRows` entry
     with hairline dividers (`Color.black.opacity(0.08)`, height 0.5) inset 49pt leading
     between rows; `onToggle` nil for `.inBundle` rows.
   - Below the card: non-interactive link row — `magnifyingglass` icon + "Browse all
     <country.name> decks" 15 medium linkBlue `#3d7ede` (rendered as text, no action — the
     Browse tab ships separately; leave a `// Browse tab arrives in T055` comment).

     (13 semibold `#1c1c1e`) + `vm.totalLabel` (17 bold); primary CTA `vm.ctaTitle` with
     `PrimaryButtonStyle(background: draft.accent.accent)`; microcopy centered beneath:
     "One-time purchase · yours forever" (12 `#8e8e93`).
   - CTA action (async): `try await vm.purchaseSelection(purchase: services.purchase)` with an
     in-flight disabled/spinner state, then call the flow's `finish()` (T031) — which writes
     the trip and flips to the main shell. On error: inline "Purchase failed — try again"
