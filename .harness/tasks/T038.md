# T038 — Debug setting: unlock all packs

## Overview

Add a debug-menu action that grants every deck and bundle in the content catalog, so any screen
that gates on ownership can be exercised without purchase flows.

## Do

1. Create `Sources/Services/DebugEntitlementTools.swift`:

   ```swift
   enum DebugEntitlementTools {
     /// Grants every deck (kind .deck) and bundle (kind .bundle) of every country in the
     /// catalog via EntitlementStore.grant(productId:kind:). Idempotent (granting an
     /// already-owned product must not duplicate or throw). Returns the number of DECKS owned
     /// after the sweep.
     static func grantAll(catalog: ContentCatalog, entitlements: EntitlementStore) throws -> Int
   }
   ```

2. Create `Sources/Features/Debug/DebugUnlock.swift`: `enum DebugUnlock` with
   `static let section = DebugSection(id: "unlock", title: "Entitlements", rows: [...])`
   containing one `.action` row (id "unlock-all", title "Unlock everything", subtitle
   "Grant every pack & bundle in the catalog") whose closure loads the catalog from
   `services` (ContentStore), calls `DebugEntitlementTools.grantAll`, and returns
   `"Unlocked N packs"` (N = the returned deck count).
3. Append `DebugUnlock.section` to the `DebugProviders.all` array literal in
   `Sources/Features/Debug/DebugRegistry.swift` (one-line edit; keep existing entries).
4. Add `Tests/DebugEntitlementToolsTests.swift` (in-memory ModelContainer): load the real bundled
   catalog via `ContentStore`, run `grantAll` twice, and assert
   `EntitlementStore.ownedDeckIds(catalog:)` equals the set of ALL deck ids across ALL countries
   in the catalog, and that the second run neither throws nor changes the count (idempotence).

## Done when

- The Debug sheet shows an "Entitlements" section with "Unlock everything"; tapping it on a fresh
  install toasts "Unlocked 12 packs" (the fixture catalog has 4 decks × 3 countries), and
  afterwards every deck in the catalog reports as owned via
  `EntitlementStore.isOwned(deckId:catalog:)`.
- `Tests/DebugEntitlementToolsTests.swift` passes with the assertions in step 4.
