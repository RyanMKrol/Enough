## Overview

Create the Codable model layer for the bundled content catalog (countries → decks/bundles →
cards) so every later feature reads typed content instead of raw JSON. Pure value types only —
no bundle loading in this task (that is T012).

## Do

1. Create `Sources/ContentKit/Models.swift` containing exactly these public-by-default (internal)
   Codable structs — names, properties, and types must match verbatim (they are the project-wide
   contract other tasks build against):

   ```swift
   struct ContentCatalog: Codable {
     let version: Int
     let scenarios: [ScenarioInfo]
     let countries: [CountryInfo]
     let comingSoon: [String]
   }
   struct ScenarioInfo: Codable, Identifiable, Equatable {
     let id: String
     let title: String
   }
   struct CountryInfo: Codable, Identifiable, Equatable {
     let id: String
     let name: String
     let languageName: String
     let nativeLanguageName: String
     let accent: String
     let flagEmoji: String
     let decks: [DeckInfo]
     let bundles: [BundleInfo]
   }
   struct DeckInfo: Codable, Identifiable, Equatable {
     let id: String
     let title: String
     let subtitle: String
     let scenario: String
     let icon: String
     let cardsFile: String
     let cardCount: Int
     let priceGBP: Double
   }
   struct BundleInfo: Codable, Identifiable, Equatable {
     let id: String
     let title: String
     let subtitle: String
     let deckIds: [String]
     let priceGBP: Double
     let popular: Bool
   }
   struct DeckCards: Codable {
     let deckId: String
     let cards: [CardContent]
   }
   struct CardContent: Codable, Identifiable, Equatable {
     let id: String
     let english: String
     let target: String
     let pronunciation: String
     let audio: String
     let notes: String?
     let category: String?
   }
   ```

   No custom CodingKeys are needed — property names equal the JSON keys exactly. Do not add
   extra fields, defaults, or computed properties in this task.

2. Create `Tests/ContentModelsTests.swift` (XCTest, class `ContentModelsTests`) that decodes
   INLINE JSON string literals (do NOT read any file or bundle — bundle wiring is a later task).

Conventions: single app target, so no `import` of any project module is needed in tests beyond
`@testable import Enough`. Swift 5 / iOS 18, 2-space indent, SwiftLint strict (lines ≤ 130).

## Done when

- `Tests/ContentModelsTests.swift` asserts all of the following with `JSONDecoder`:
  - A full `ContentCatalog` sample (2 scenarios, 1 country with 1 deck + 1 bundle, one
    `comingSoon` entry) decodes; assert `catalog.version == 1`, scenario/country/deck/bundle ids
    and `deck.priceGBP == 1.0`, `bundle.deckIds` contents, `bundle.popular == true`.
  - A `DeckCards` sample with two cards decodes: card 1 includes `"notes"` and `"category"`
    (assert both non-nil values), card 2 OMITS those two keys entirely (assert both are `nil`) —
    proving the optionals tolerate absent keys.
  - Unicode survives decoding: a card with `"target": "おはようございます。"` round-trips
    (`XCTAssertEqual` on the decoded string).
- All new code lives only in `Sources/ContentKit/` and `Tests/`.
