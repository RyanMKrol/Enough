# T053 — ReadinessCalculator: per-trip survival readiness % + copy line

## Overview

Implement the pure calculator behind the Progress tab's "survival readiness" card: a percent of
learned coverage across the trip's chosen scenarios, plus the confident-minimal copy line that
names what's handled and what's next. Design reference: .harness/docs/designs/design-spec.md
§7.2; architecture: .harness/docs/designs/architecture.md (Services).

## Do

1. Create `Sources/Services/ReadinessCalculator.swift`:

   ```swift
   enum ReadinessCalculator {
     static func readiness(
       tripScenarios: [ScenarioInfo],          // the scenarios picked on the trip, in picked order
       decks: [DeckInfo],                      // all decks of the trip's country
       progress: (String) -> DeckProgressService.DeckProgress?  // by deck id; nil = no rows yet
     ) -> (percent: Int, line: String)
   }
   ```

   Rules (pin these exactly):
   - Only decks whose `scenario` is in `tripScenarios` participate. A trip scenario with zero
     matching decks is ignored entirely (percent AND line).
   - `percent` = Σ `learned` across participating decks ÷ Σ their `cardCount` × 100, rounded
     half-away-from-zero (`.rounded()`), clamped 0...100. Treat a `nil` progress as
     `learned = 0`. Empty denominator → `percent = 0`.
   - Per-scenario coverage = Σ learned ÷ Σ cardCount over that scenario's decks. A scenario is
     **handled** when coverage ≥ 0.6.
   - Casual-noun table for scenario ids (used in the line): `eating-out` → "food",
     `greetings` → "greetings", `getting-around` → "directions", `nightlife` → "nightlife",
     `shopping` → "shopping", `emergencies` → "emergencies" (unknown id → its `title`
     lowercased).
   - Line composition:
     - No handled scenarios (including nothing learned at all) →
       **"Start with <first trip scenario's noun>."** (noun capitalised at sentence start,
       e.g. "Start with food.").
     - All participating scenarios handled → **"You're covered. Keep it fresh."**
     - Otherwise → **"You can handle <list>. <Next>'s next."** where `<list>` joins handled
       nouns in trip order — one item: "food"; two: "food & greetings"; three or more:
       "food, greetings & directions" — and `<Next>` is the LOWEST-coverage un-handled
       scenario's noun with its first letter capitalised (ties → first in trip order). Example
       target line: "You can handle food, greetings & directions. Nightlife's next."
2. Add `Tests/ReadinessCalculatorTests.swift` (XCTest) with a hand-built fixture (construct
   `ScenarioInfo`/`DeckInfo` values directly; return `DeckProgress` from a dictionary closure).

## Done when

- `xcodebuild test` passes with at least these pinned cases:
  - **Nothing learned**: scenarios `[eating-out, greetings]`, decks 30+24 cards, all progress
    nil/zero → `(0, "Start with food.")`.
  - **Partial (the design's mock)**: scenarios `[eating-out, greetings, getting-around,
    nightlife]`; decks (cardCount/learned): 30/21, 24/18, 34/24, 32/0 →
    `percent == 53` (63÷120 = 52.5 rounds to 53) and
    `line == "You can handle food, greetings & directions. Nightlife's next."`.
  - **All handled**: two scenarios fully learned → `(100, "You're covered. Keep it fresh.")`.
  - **Ignored scenario**: a trip scenario with no decks changes neither percent nor line.
  - **Two handled** formats as "food & greetings" (no comma).
- The calculator is pure (no SwiftData, no SwiftUI imports) and the UI-facing tuple matches the
  signature above so the Progress tab task can consume it unchanged.
