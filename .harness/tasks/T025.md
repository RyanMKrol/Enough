# T025 — PricingCalculator: pack/bundle totals, savings & summary lines

## Overview

Pure pricing maths + GBP formatting for the plan screen and Browse store (design-spec.md §2.4):
bundle savings ("save £0.51"), the running plan total, and the sticky-footer summary line
("Weekend · 3 packs"). All computed from catalog data — never hardcode prices.

## Do

1. Create `Sources/Services/PricingCalculator.swift` (`import Foundation` only):

   ```swift
   enum PricingCalculator {
     static func price(_ gbp: Double) -> String
     static func bundleSavings(_ bundle: BundleInfo, in country: CountryInfo) -> Double
     static func planTotal(selectedBundle: BundleInfo?, extraDeckIds: Set<String>,
                           country: CountryInfo) -> Double
     static func summaryLine(selectedBundle: BundleInfo?, extraDeckIds: Set<String>) -> String
   }
   ```

2. `price(_:)`: `NumberFormatter` with `numberStyle = .currency`, `locale =
   Locale(identifier: "en_GB")` → `"£2.49"`, `"£1.00"`. Create the formatter once
   (`static let`), not per call.

3. `bundleSavings`: Σ of `priceGBP` over the bundle's member decks (resolve `bundle.deckIds`
   against `country.decks`; ignore unknown ids) minus `bundle.priceGBP`. Round to 2 dp
   (`(x * 100).rounded() / 100`) to kill float noise.

4. `planTotal`: `(selectedBundle?.priceGBP ?? 0)` + Σ `priceGBP` of `extraDeckIds` that are NOT
   already members of the selected bundle (à-la-carte packs on top of a bundle, per
   design-spec.md §2.4), rounded to 2 dp. With no bundle, extras are simply summed.

5. `summaryLine`: pack count = (bundle member count, counting only ids that resolve — for the
   summary the raw `deckIds.count` is fine) + count of extra ids not in the bundle. With a
   bundle: `"\(bundle.title) · \(n) packs"` (`"1 pack"` singular). Without: `"\(n) packs"` /
   `"1 pack"`; zero → `"No packs"`.

6. Create `Tests/PricingCalculatorTests.swift` (XCTest) using the REAL bundled catalog via
   `ContentStore().catalog()` (the hosted test bundle's `Bundle.main` contains the committed
   `Content/` fixture) — country `"japan"` has 4 decks at £1.00 each, bundle `"jp-weekend"`
   (3 decks, £2.49, title "Weekend") and `"jp-week"` (all 4 decks, £3.49).

## Done when

Tests pin against the real fixture:
- `price(2.49) == "£2.49"`, `price(1) == "£1.00"`, `price(8.99) == "£8.99"`.
- `bundleSavings(jp-weekend) == 0.51` and `bundleSavings(jp-week) == 0.51`.
- `planTotal(selectedBundle: jp-weekend, extraDeckIds: ["jp-getting-around"]) == 3.49`;
  adding a deck ALREADY in the bundle (e.g. `"jp-greetings"`) changes nothing (still 2.49 with
  no extras beyond it); `planTotal(nil, ["jp-greetings", "jp-at-the-bar"]) == 2.00`;
  `planTotal(nil, []) == 0`.
- `summaryLine(jp-weekend, [])` == `"Weekend · 3 packs"`;
  `summaryLine(jp-weekend, ["jp-getting-around"])` == `"Weekend · 4 packs"`;
  `summaryLine(nil, ["jp-greetings"])` == `"1 pack"`; `summaryLine(nil, [])` == `"No packs"`.
