## Overview
Flip the app's on-disk SwiftData store to live in the shared App Group container so a widget extension can read the same data, migrating any existing store from the old Application Support location exactly once so no user data is lost.

## Do
Using the storeURL(appGroupID:) seam added by T068 and the App Group id chosen by reviews-surface-appgroup (read it from .harness/custom/docs/LIMITATIONS.md), make PersistenceStack.container() build its on-disk ModelConfiguration from storeURL(appGroupID: <group>). Before opening it, perform a one-time move: if a store exists at the old storeURL() (nil app-group) location and none exists in the app-group container, move the .store file(s) (including -wal/-shm sidecars) into the app-group container. In-memory behavior unchanged.

## Done when
On-disk container creation resolves its URL from the app-group container; a unit test proves the one-time move relocates an existing store file to the app-group path and is a no-op when already migrated (run against a temp/scratch directory, never the real store); existing persistence tests stay green; xcodegen generate + xcodebuild test passes.
