# T039 — Debug setting: time travel (shift the app clock by N days)

## Overview

Add a debug-menu stepper that shifts the app-wide clock forward/backward by whole days, so
SRS due-dates, streaks, and the trip day counter can be exercised without waiting. It drives the
existing `AdjustableDateProvider` (CoreKit, T013) — per the T013/T029 contract, `AppServices.live()`
already uses `AdjustableDateProvider` as the app-wide `DateProvider`, so no wiring outside this
task's scope is needed.

## Do

1. Create `Sources/Services/TimeTravel.swift`:

   ```swift
   enum TimeTravel {
     /// Current day offset, or nil when services.dateProvider is not an AdjustableDateProvider.
     static func offset(in services: AppServices) -> Int?
     /// Sets the day offset (no-op when the provider is not adjustable).
     static func setOffset(_ days: Int, in services: AppServices)
     /// "today" for 0, "+3 days" / "-2 days" otherwise.
     static func label(for days: Int) -> String
   }
   ```

   Both accessors work by casting `services.dateProvider as? AdjustableDateProvider`.
2. Create `Sources/Features/Debug/DebugTimeTravel.swift`: `enum DebugTimeTravel` with
   `static let section = DebugSection(id: "time", title: "Time", rows: [...])` containing one
   `.stepper` row (id "time-travel", title "Time travel", subtitle "Shifts every date the app
   sees"): `get` = `TimeTravel.offset(in:) ?? 0`, `set` = `TimeTravel.setOffset(_:in:)`,
   `label` = `TimeTravel.label(for:)`. If `TimeTravel.offset(in:)` is nil, use subtitle
   "unavailable — provider not adjustable" instead.
3. Append `DebugTimeTravel.section` to the `DebugProviders.all` array literal in
   `Sources/Features/Debug/DebugRegistry.swift` (one-line edit; keep existing entries).
4. Add `Tests/TimeTravelTests.swift`: with an `AdjustableDateProvider` backed by a throwaway
   `UserDefaults(suiteName:)`, assert that setting `dayOffset = 3` moves `now` forward by
   3 × 86_400 seconds (±5s tolerance vs the system clock), that a negative offset moves it
   backwards, and that `TimeTravel.label(for:)` returns "today", "+3 days", "-2 days" for
   0 / 3 / -2.

## Done when

- The Debug sheet shows a "Time" section with a Time-travel stepper; stepping it up on the
  simulator immediately changes what the app treats as "now" (e.g. the About section's
  active-trip day, once trips exist, reflects the shifted date after reopening the sheet), and
  the value survives relaunch (it persists via the provider's UserDefaults key
  `debug.dayOffset`).
- `Tests/TimeTravelTests.swift` passes with the assertions in step 4.
