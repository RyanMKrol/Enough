# T048 — Flashcard component: front, 3D flip, revealed layout

## Overview

Build the tap-to-reveal flashcard used by review sessions: a white card that shows only the
target phrase + audio on the front, 3D-flips on tap, and reveals a tinted header band over the
English answer. Pure component — reveal state is owned by the parent via a Binding.

## Do

Design reference: .harness/docs/designs/design-spec.md (read-only reference) §6b. Uses `PulsingAudioButton`,
`EnoughColor`, `Motion`; accent via `\.accentTheme`. One file:
`Sources/Features/Session/FlashcardView.swift`.

1. `struct FlashcardView: View` with
   `init(card: CardContent, isRevealed: Binding<Bool>, onPlayAudio: @escaping () -> Void)`.
2. **Card container**: RoundedRectangle radius 28, white fill, shadow
   `0 20 42 black@14%`, aspect/size driven by the parent (use
   `.frame(maxWidth: .infinity)` + a sensible min height ~380; interior padding 22). Parent
   screens place it on a `#f2f2f7` canvas — do not paint a canvas here.
3. **Front face**: centered VStack(spacing 18):
   `PulsingAudioButton(size: 60, isPulsing: true, action: onPlayAudio)`; `card.target` —
   36 Bold, `EnoughColor.label`, multiline center. NO pronunciation on the front
   (design: hidden until reveal).
4. **Hint** (rendered by this component BELOW the card, outside the white shape):
   "Tap the card to reveal" — 13 Regular `#8e8e93`, centered, top padding 14. Hidden once
   revealed.
5. **Back face**: VStack(spacing 0):
   - Tinted header band full-width across the card top (background `accentTheme.tint`,
     padding 14×18, top corners matching the card radius): `card.target` 20 Semibold +
     `card.pronunciation` 13 Regular, both `accentTheme.deep`, centered.
   - Below (fills the rest, centered, spacing 14): `card.english` — 22 Semibold
     `EnoughColor.label`, multiline center; `PulsingAudioButton(size: 34, isPulsing: false,
     action: onPlayAudio)`.
6. **Flip**: tapping the card toggles `isRevealed` (only front → back; tapping a revealed
   card does nothing). Animate with `rotation3DEffect` around the y-axis, total 0.4s: the
   content swaps at 90° so text is never mirrored — standard two-face technique: front face
   opacity 1 while angle < 90°, back face (pre-rotated 180°) after; drive angle from
   `isRevealed` with `withAnimation(.easeInOut(duration: 0.4))`. The component must also
   render correctly when the parent sets `isRevealed` programmatically (e.g. resets to false
   for the next card — no animation glitch; use `.animation(nil)` on identity resets if
   needed).
7. `#Preview`s: front and revealed, using a hardcoded preview `CardContent`
   (ありがとうございます / arigatō gozaimasu / "Thank you very much") on a `#f2f2f7`
   canvas.

Copy string EXACT: "Tap the card to reveal".

## Done when

- `FlashcardView(card:isRevealed:onPlayAudio:)` exists with exactly that signature — T049
  constructs it with a Binding it owns and re-uses the component across consecutive cards.
- Front shows ONLY audio button + 36pt target (no pronunciation) with the hint line below;
  revealed shows the tinted band (target + pronunciation) over the 22pt english + a 34pt
  audio button; hint disappears when revealed.
- The flip is a y-axis 3D rotation (~0.4s) with no mirrored text at any point; tapping a
  revealed card does not flip back; a programmatic reset to `isRevealed = false` renders the
  front cleanly for the next card.
- Both `#Preview`s compile and match design-spec §6b values (radius 28, shadow 0/20/42@14%,
  60pt front audio button).
