## Overview

Give every interactive element and screen root a stable `.accessibilityIdentifier` so the
harness's simulator-driving tools (xcui/AXe) can navigate the app by id, and VoiceOver work
(T069) has stable anchors. Identifiers are invisible to users and change no behavior.

## Do

1. Add identifiers using EXACTLY these names (they become the harness's navigation contract —
   list them in a new AXID constants file inside the Sources/App directory, declared as
   `enum AXID` with static string constants, and use the constants everywhere):
   - Tab bar buttons: `tab-learn`, `tab-reviews`, `tab-progress`, `tab-browse`.
   - Screen roots (outermost container of each screen): `screen-welcome`, `screen-country`,
     `screen-trip-shape`, `screen-plan`, `screen-home`, `screen-deck-detail`,
     `screen-session-mc`, `screen-session-review`, `screen-session-complete`,
     `screen-reviews`, `screen-progress`, `screen-browse`, `screen-debug`.
   - Onboarding: `onboarding-continue` (the primary CTA on every step), `country-card-<id>`
     (e.g. `country-card-japan`), `restore-purchases`.
   - Home: `home-continue-card`, `home-reviews-banner`, `deck-row-<deckId>`, `streak-pill`.
   - Deck detail: `deck-continue`, `deck-practice`.
   - Sessions: `audio-button`, `answer-row-0`…`answer-row-3`, `flashcard`, `grade-again`,
     `grade-hard`, `grade-good`, `grade-easy`, `session-close`, `session-next`.
   - Session complete: `complete-done`, `complete-learn-more`.
   - Reviews tab: `start-review`. Browse: `buy-<productId>`, `browse-restore`.
2. Attach ids on the tappable element itself (the `Button`, not a wrapping stack), and screen
   ids on the screen's outermost view.
3. Add `Tests/AccessibilityIDsTests.swift`: asserts the `AXID` constants match the strings
   above verbatim (guards against silent renames breaking the harness recipes).

## Done when

- Every element listed above carries its identifier via the shared `AXID` constants; a grep for
  `accessibilityIdentifier("` (string literals inline) finds nothing outside the AXID constants file.
- `Tests/AccessibilityIDsTests.swift` pins the exact strings.
- App behavior and appearance are unchanged (identifiers only).
