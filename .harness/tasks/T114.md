## Overview
The review flow shows a permanent blank page when the user taps "Start Review" in the Reviews tab, so the spaced-repetition review session is unreachable on device. This is the review-flow twin of the learn-session blank-screen bug fixed in T112 (now merged/done): the SAME two-part SwiftUI defect, in the review files. Fix it by mirroring T112's proven approach applied to the review presentation path.

## Do
Make the review session render instead of a blank page, fixing the SwiftUI view-state/presentation bug across BOTH the presented view and its call site.

Two interacting problems to resolve:

1) In Sources/Features/Session/ReviewSessionView.swift, the view holds `@State private var viewModel: ReviewSessionViewModel?` (nil at first) and only builds it inside `.onAppear` attached to the `else`-branch `EnoughColor.canvas` placeholder (lines ~101-104) — mutating that state from the branch that is about to be swapped out is fragile and can leave `content` stuck on the blank placeholder. Replace this with a state-safe pattern, mirroring the fix T112 applied to MCSessionView.swift: prefer a NON-optional view-model built once in `init`, OR build the view-model in a `.task`/`.onAppear` attached to a STABLE container (the NavigationStack root), never the branch being conditionally rendered. Note `services` comes from `@Environment` so it is not available in `init`; if you need `services` at construction time, inject it (pass `AppServices` into `ReviewSessionView.init` from the call site, or build the view-model in a `.task` where `@Environment` IS available) rather than reintroducing the onAppear-on-else-branch pattern. Do not leave any code path where `viewModel` can stay nil after the view has appeared. Keep the card UI intact (the reveal FlashcardView + `GradeButtonRow` grading, which differs from the learn path's multiple-choice rows).

2) At the call site — `ReviewsTabView.startReviewSession()` / `ReviewsTabView.body`'s `.fullScreenCover(isPresented: $isShowingReviewSession)` (line ~74) — the cover content closure does `if let reviewSessionEngine { ReviewSessionView(engine: reviewSessionEngine) }` against an optional `@State` engine (`reviewSessionEngine`, line 8) that is set immediately before the presenting flag is flipped (lines 231-232), the same nil-engine-capture shape T112 fixed for the learn path.
