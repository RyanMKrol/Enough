# T041 — Debug setting: reset all app data

## Overview

Add a destructive debug-menu action that wipes every persisted record (trip, entitlements, SRS
rows, activity) plus the time-travel offset, returning the app to a factory-fresh state after a
relaunch.

## Do

1. Create `Sources/Services/AppReset.swift`:

   ```swift
   enum AppReset {
     /// Deletes ALL rows of all four SwiftData record types (TripProfileRecord,
     /// EntitlementRecord, CardSRSRecord, DailyActivityRecord), saves the context, and
     /// removes the "debug.dayOffset" key from the given UserDefaults.
     static func wipeAll(context: ModelContext, userDefaults: UserDefaults = .standard) throws
   }
   ```

   Use `context.delete(model:)` (the batch form) or fetch-and-delete per type; either is fine as
   long as all four types end empty and the context is saved. Deliberately no in-place UI
   transition: the app is left running on wiped data and the user relaunches (RootView re-reads
   the trip on launch and lands on onboarding).
2. Create `Sources/Features/Debug/DebugReset.swift`: `enum DebugReset` with
   `static let section = DebugSection(id: "reset", title: "Danger zone", rows: [...])`
   containing one `.action` row (id "reset-all", title "Reset all data", subtitle
   "Wipes trip, purchases, SRS progress and history") whose closure calls
   `AppReset.wipeAll(context:userDefaults:)` using the services' model context and standard
   defaults, and returns `"Data wiped — relaunch the app"`.
3. Append `DebugReset.section` to the `DebugProviders.all` array literal in
   `Sources/Features/Debug/DebugRegistry.swift` (one-line edit; keep existing entries — this
   section should be LAST in the array so "Danger zone" renders at the bottom).
4. Add `Tests/AppResetTests.swift` (in-memory ModelContainer): insert at least one row of each of
   the four record types and set `debug.dayOffset` in a throwaway `UserDefaults(suiteName:)`;
   run `wipeAll`; assert a fetch of each record type returns empty and the defaults key is nil.

## Done when

- Tapping "Reset all data" toasts "Data wiped — relaunch the app"; after relaunching the
  simulator app, onboarding is shown again (no active trip, nothing owned, no SRS state, no
  streak), and the time-travel offset reads "today".
- `Tests/AppResetTests.swift` passes with the assertions in step 4.
