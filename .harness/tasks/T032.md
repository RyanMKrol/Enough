## Overview

Build the real Welcome ("cold open") screen — card-stack hero, wordmark, tagline, "Get started",
and a working "Restore purchases" link — replacing the placeholder root inside
`OnboardingFlowView`.

## Do

1. Read design-spec.md §2.1 fully (every value below comes from it) and
   architecture.md. The `CardStackHero` component already exists (T009) — use it as-is.
2. Create `Sources/Features/Onboarding/WelcomeView.swift` — `struct WelcomeView: View`:
   - Full-bleed white (`EnoughColor.surface`) background; layout = vertically centered hero
     block, CTAs pinned to the bottom; overall padding approx top 96 / horizontal 28 /
     bottom 40.
   - Hero (top→bottom, centered):
     a. `CardStackHero()` in its ~255×292 stage.
     b. Wordmark: "Enough." — system 44 bold, tracking −0.03em (kerning ≈ −1.3), label color
        `#1c1c1e`, with ONLY the trailing period colored japanAccent `#e24947` (two-Text
        concatenation or AttributedString). Position it so it grazes/overlaps the bottom edge
        of the front card — pin: apply a negative top padding/offset of about −26 so the
        wordmark's cap height overlaps the card stack's bottom blank edge, matching the design.
     c. Tagline, 14–18pt below the wordmark: exactly
        "Learn just enough of a language to get through the trip. A weekend. A week. No more
        than you need." — 18 regular, `#6b6b70`, multiline centered, `frame(maxWidth: 280)`.
   - Bottom CTA stack (spacing 12):
     a. `Button("Get started")` with `PrimaryButtonStyle()` (graphite `#1c1c1e` default, white
        17 semibold, height 54, radius 16, full width) → appends `.country` to the flow's
        navigation path (pin: the flow passes a closure `onGetStarted: () -> Void` or the view
        appends via a `Binding<[OnboardingStep]>` — match how OnboardingFlowView (T031) wired
        its placeholder root and keep that mechanism).
     b. `Button("Restore purchases")` with `TextLinkButtonStyle()` (linkBlue `#3d7ede`, 15pt).
        Action: async task calling `try await services.purchase.restorePurchases()`; while
        running show a small inline `ProgressView` in place of the label; on completion show a
        momentary confirmation line "Purchases restored" (13pt, `#8e8e93`) under the link for
        ~2s (`.transition(.opacity)`); on error show "Couldn't restore purchases" the same way.
        No alerts, no sheets — quiet inline feedback (confident-minimal voice).
3. Replace the placeholder root of `OnboardingFlowView` (T031) with `WelcomeView`, preserving
   the push-to-`.country` behavior.
4. `#Preview` with `AppServices.preview()` environment.
