# T019 — SRS core: card scheduling state + SM-2 grading function

## Overview

Create the pure spaced-repetition core: the `SRSState` value type that describes one card's
scheduling state, and the `SRSEngine.apply` grading function implementing this app's SM-2-style
ladder. This is the contract the whole app schedules against — the numbers below are pinned by
tests and must not drift.

## Do

1. Create `Sources/SRSKit/SRSState.swift`. SRSKit is PURE — `import Foundation` only (no
   SwiftUI/SwiftData/UIKit anywhere in `Sources/SRSKit/`). Define exactly:

   ```swift
   enum SRSGrade: String, CaseIterable { case again, hard, good, easy }
   enum CardStatus: String { case new, learning, review }
   struct SRSState: Equatable {
     var status: CardStatus
     var easeFactor: Double   // starts 2.5, floor 1.3
     var intervalDays: Double // 0 for a new card; 1/1440 = one minute
     var repetitions: Int
     var lapses: Int
     var dueAt: Date?         // nil for a new card
   }
   extension SRSState {
     static let newCard = SRSState(
       status: .new, easeFactor: 2.5, intervalDays: 0, repetitions: 0, lapses: 0, dueAt: nil)
   }
   ```

2. Create `Sources/SRSKit/SRSEngine.swift` with `enum SRSEngine` and
   `static func apply(_ grade: SRSGrade, to state: SRSState, now: Date) -> SRSState`.
   Never call `Date()` inside SRSKit — time always arrives via the `now` parameter.
   Rules (apply in this order; ease floor 1.3 applied after every ease change):
   - **again** (any status): `lapses += 1` only if incoming status was `.review`;
     `repetitions = 0`; `status = .learning`; `intervalDays = 1.0 / 1440.0` (one minute);
     `easeFactor -= 0.2`.
   - **hard**: `easeFactor -= 0.15`; `repetitions` unchanged; status unchanged (a `.new` card
     becomes `.learning`); interval: if incoming `intervalDays < 1` → `1.0`; else if status is
     `.learning` → interval HOLDS (unchanged — this keeps the designed "1 day" preview for a
     young card); else (`.review`) → `(intervalDays * 1.2).rounded(.up)` whole days.
   - **good**: `repetitions += 1`; ease unchanged. Ladder: `.new` card → `.learning`,
     `intervalDays = 1`; `.learning` with `intervalDays <= 1` → stays `.learning`,
     `intervalDays = 3`; `.learning` with `intervalDays > 1` → `.review`,
     `intervalDays = (3 * easeFactor).rounded()`; `.review` →
     `intervalDays = (intervalDays * easeFactor).rounded()`.
   - **easy**: `repetitions += 1`; `easeFactor += 0.15`. From `.new`/`.learning` with
     `intervalDays <= 1` → `.review`, `intervalDays = 6`; from `.learning` with
     `intervalDays > 1` or `.review` → `.review`,
     `intervalDays = (intervalDays * easeFactor * 1.3).rounded()` (ease used is the value BEFORE
     this grade's +0.15).
   - Always: `dueAt = now.addingTimeInterval(intervalDays * 86_400)` using the NEW interval.

   Worked ladder a fresh card walks when graded good four times (ease stays 2.5):

   | grade # | status after | intervalDays | repetitions |
   |---|---|---|---|
   | 1 | learning | 1 | 1 |
   | 2 | learning | 3 | 2 |
   | 3 | review | 8  (3 × 2.5 = 7.5 rounded) | 3 |
   | 4 | review | 20 (8 × 2.5) | 4 |

3. Create `Tests/SRSEngineTests.swift` (XCTest) pinning the rules with a fixed
   `let now = Date(timeIntervalSince1970: 1_800_000_000)`.

## Done when

- `Tests/SRSEngineTests.swift` asserts, at minimum:
  - The good/good/good/good ladder from `SRSState.newCard` produces exactly intervals
    `[1, 3, 8, 20]` days, statuses `[.learning, .learning, .review, .review]`, repetitions
    `[1, 2, 3, 4]`, and each `dueAt == now + intervalDays * 86_400`.
  - `again` on a `.review` card (interval 8, ease 2.5): lapses 0→1, repetitions→0,
    status→`.learning`, `intervalDays == 1.0/1440.0`, ease → 2.3.
  - `again` on a `.learning` card does NOT increment lapses.
  - `hard` on a `.learning` card at interval `1.0/1440.0` (sub-day) → interval 1 day, ease → 2.35.
  - `hard` on a `.learning` card at interval exactly 1 → interval STAYS 1 day (ease → 2.35).
  - `hard` on a `.review` card at interval 8 → interval `ceil(8 × 1.2) = 10` days.
  - `easy` on a `.learning` card at interval 1 → `.review`, interval 6, ease → 2.65.
  - Ease never drops below 1.3 (grade `again` five times from new; assert `easeFactor == 1.3`).
- `Sources/SRSKit/` contains no `import` other than `Foundation` and no use of `Date()`
  (verifiable by grep).
