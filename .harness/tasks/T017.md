## Overview

Add `CardSRSStore`, the persistence gateway for per-card spaced-repetition rows: upsert keyed by
(deckId, cardId), per-deck fetches, and the "what's due now" query filtered to decks the user
owns. Builds on `CardSRSRecord` from T014 — do NOT change that model's fields.

## Do

1. Create `Sources/Persistence/CardSRSStore.swift`:

   ```swift
   final class CardSRSStore {
     init(context: ModelContext)

     /// The row for (deckId, cardId), or nil if the card has never been tracked.
     func record(deckId: String, cardId: String) throws -> CardSRSRecord?

     /// Updates the existing (deckId, cardId) row's mutable fields (statusRaw, easeFactor,
     /// intervalDays, repetitions, lapses, dueAt, lastReviewedAt) or inserts a new row when
     /// none exists. Saves. Returns the stored record. Exactly one row may ever exist per
     /// (deckId, cardId) pair.
     @discardableResult
     func upsert(
       deckId: String, cardId: String, statusRaw: String, easeFactor: Double,
       intervalDays: Double, repetitions: Int, lapses: Int, dueAt: Date?,
       lastReviewedAt: Date?
     ) throws -> CardSRSRecord

     func records(forDeck deckId: String) throws -> [CardSRSRecord]

     /// Rows that are DUE: statusRaw != "new" AND dueAt != nil AND dueAt <= now AND
     /// deckId ∈ ownedDeckIds. Sorted by dueAt ascending (most overdue first).
     func dueRecords(now: Date, ownedDeckIds: Set<String>) throws -> [CardSRSRecord]

     /// Deletes ALL CardSRSRecord rows and saves.
     func reset() throws
   }
   ```

   Implementation notes: fetch with `FetchDescriptor<CardSRSRecord>` + `#Predicate` where
   possible; the owned-deck filter may be applied in memory after fetching (a `Set.contains`
   inside `#Predicate` can be brittle — in-memory filtering is acceptable and must be
   documented with a one-line comment).

2. Create `Tests/CardSRSStoreTests.swift` (XCTest, class `CardSRSStoreTests`). Every test uses
   `try PersistenceStack.container(inMemory: true)` + a fresh `ModelContext`. Use fixed `Date`
   values (e.g. a base date ± hours) — never `Date()` comparisons without tolerance.

Conventions: single app target (`@testable import Enough`), Swift 5 / iOS 18, 2-space indent,
SwiftLint strict (lines ≤ 130).

## Done when

  - `record(deckId:cardId:)` nil for an untracked pair.
  - `upsert` inserts: afterwards `record(...)` returns the row with every field as passed.
  - Upserting the SAME (deckId, cardId) with different values MUTATES the row (fields update)

    with `ownedDeckIds == ["A"]` returns exactly [(a)]; with `ownedDeckIds == ["A", "B"]`
    returns [(d), (a)] in that order (dueAt ascending).
  - After `reset()`: `records(forDeck:)` empty for both decks.
