## Overview
The owner has decided to remove the recurring 'New trip' flow entirely (idea #5, which supersedes idea #3's 'make it lighter'). Onboarding should be a one-time experience; getting more content is done by buying packs. Remove the button, its confirmation dialog, and the now-orphaned action, while carefully PRESERVING the shared re-onboarding machinery that the debug 'Reset all data' path still depends on.
