## Overview

Create the app's dependency container (`AppServices`), the top-level observable `AppState`
(onboarding vs main phase + active accent), and `RootView`, and rewire `EnoughApp` to use them.
This is the root everything later hangs off; the main phase shows a temporary placeholder that
T030 replaces.

## Do

1. Read .harness/docs/designs/architecture.md (§ App shell) and design-spec.md §0/§3 first.
   Use the exact type names below. All code goes in the single `Enough` app target.
2. Create `Sources/App/AppServices.swift` — `final class AppServices` holding, as `let`
   properties, every service the app needs that EXISTS at this task's build time:
   `dateProvider: DateProvider`, `contentStore: ContentStore`, `container: ModelContainer`,
   `tripStore: TripStore`, `entitlementStore: EntitlementStore`, `cardSRSStore: CardSRSStore`,
   `activityStore: ActivityStore`, `purchase: PurchaseProviding`, `audio: AudioService`,
   `study: StudyService`, `deckProgress: DeckProgressService`, `stats: StatsService`.
   (These types were all built by earlier tasks — T012, T013, T014–T018, T022, T024, T026–T028.
   If one is genuinely missing from the codebase, flag it in the worklog and stop.) Provide:
   - `static func live() -> AppServices` — on-disk `PersistenceStack.container()`,
     `SystemDateProvider()` (or `AdjustableDateProvider()` if that is what T013 shipped as the
     debug-capable default — prefer `AdjustableDateProvider` so time travel can work later),
     `StubPurchaseService`.
   - `static func preview() -> AppServices` — same graph but
     `PersistenceStack.container(inMemory: true)`.
   - A designated `init` taking every dependency so tests can inject.
3. In the same file (or `Sources/App/ServicesEnvironment.swift`), expose the container via
   SwiftUI environment: `extension EnvironmentValues { @Entry var services: AppServices =
   .preview() }`.
4. Create `Sources/App/AppState.swift` — `@Observable final class AppState`:
   - `enum Phase { case onboarding, main }`, `var phase: Phase`,
     `var activeAccent: AccentTheme`.
   - `init(services: AppServices)` — reads `services.tripStore.activeTrip()`; a non-nil active
     trip ⇒ `.main` and `activeAccent = AccentTheme(rawValue: trip.accentRawValue) ?? .japan`;
     nil ⇒ `.onboarding` with `.japan`.
   - `func completeOnboarding(accent: AccentTheme)` — sets `activeAccent = accent`,
     `phase = .main`. (Writing the TripProfileRecord is the ONBOARDING flow's job, not
     AppState's — this just flips UI phase.)
   - `func startNewTrip()` — sets `phase = .onboarding` (entitlements and SRS data untouched).
5. Create `Sources/App/RootView.swift` — `struct RootView: View`; takes the `AppState` (via
   `@Environment` or init — pin: `init(appState: AppState)` stored with `@Bindable`/plain let).
   `switch appState.phase`: `.onboarding` → a temporary `Text("Onboarding")` placeholder
   (replaced by T031); `.main` → a temporary `Text("Main")` placeholder (replaced by T030).
   Apply `.environment(\.accentTheme, appState.activeAccent)` to the switched content.
6. Rewrite `Sources/EnoughApp.swift`: build `let services = AppServices.live()` and
   `let appState = AppState(services: services)` once (as `@State` properties of the App
   struct), show `RootView(appState: appState)` inside `WindowGroup`, and apply
   `.environment(\.services, services)` and `.modelContainer(services.container)`.
7. DELETE `Sources/ContentView.swift` (the T001 placeholder view). Nothing may reference it
   afterwards.
8. Tests in `Tests/AppStateTests.swift` (XCTest, in-memory container via
   `AppServices.preview()` or a hand-built graph):
   - empty trip store ⇒ `AppState.phase == .onboarding` and `activeAccent == .japan`;
   - after saving an active `TripProfileRecord` with `accentRawValue: "france"` ⇒ a freshly
     constructed `AppState.phase == .main` and `activeAccent == .france`;
   - `startNewTrip()` flips phase back to `.onboarding` without touching the entitlement store
     (grant one entitlement first, assert it is still owned after).

## Done when

- `AppServices.live()` / `.preview()` / the injectable init exist with the property set above;
  `\.services` environment key resolves.
- App launches to `RootView` and shows the "Onboarding" placeholder on a fresh install
  (no trip yet).
- `Sources/ContentView.swift` no longer exists and the project still builds.
- `xcodebuild test` passes with the three `AppStateTests` assertions above (they must exist and
  run against an IN-MEMORY container — never the on-disk store).
