## Overview

Lay the minimal iOS project skeleton and turn the Definition-of-Done tooling on. This is the ONE
task that defines the project-wide conventions (XcodeGen manifest, SwiftLint, swift-format, test
target, CI) that gate every later build — and its whole-repo tooling sweeps a tree that already
vendors files the loop must never touch — so a human sets it up once, by hand, rather than the
loop. (`gate: needs-human`: the loop skips this task and records `failed:blocked` until you mark
it done.)

## Do

1. **Create the XcodeGen manifest + a trivial app target.** Add `project.yml` (app name `Enough`,
   `bundleIdPrefix: com.ryankrol`, `deploymentTarget.iOS` set to a current iOS version) and a
   minimal `Sources/` tree (an `EnoughApp.swift` App entrypoint + a placeholder `ContentView.swift`)
   so `xcodegen generate` produces a buildable, testable `Enough.xcodeproj`/`Enough` scheme. Add a
   `Tests/` target with at least one trivial passing test.
2. **Wire up SwiftLint + swift-format.** Add a `.swiftlint.yml` (excluding `.harness/`,
   `*.xcodeproj`, and any generated build output) and confirm `swift-format lint --recursive
   --strict Sources Tests` and `swiftlint lint --strict` both pass cleanly on the skeleton.
3. **Wire the real DoD into CI.** Confirm `.github/workflows/ci.yml`'s steps (XcodeGen generate,
   swift-format, swiftlint, `xcodebuild test`) actually pass against the new project, and that they
   match `.harness/config/harness.env`'s `LOCAL_DOD`. Adjust either side if the exact tool
   invocations need tweaking once real tooling versions/configs are known — keep them identical.
4. **Add `build_run.sh`.** A CLI script (see basket's `~/Development/basket/build_run.sh` for a
   worked reference) that runs `xcodegen generate`, builds the app for the simulator, boots a
   simulator, installs + launches the app, and saves a screenshot to `screenshots/latest.png`. This
   is the command `VISUAL_VERIFY_HOOK` in `.harness/config/harness.env` already points at
   (`./build_run.sh`) — it must exist and work before any visual-verification task runs.
5. **Exclude the harness from your own tooling (critical).** SwiftLint's `excluded:` list must
   cover `.harness/`, and no Swift source-discovery glob should pick up `.harness/dashboard/*.js`
   (not Swift, so this is likely a non-issue, but double-check the lint/format globs are
   Sources/Tests-scoped, not whole-repo).
6. **Get it green** — format / lint / test pass locally, then CI passes on `main`.

## Done when

CI is green on `main`; the Definition-of-Done commands in `.harness/config/harness.env`
(`LOCAL_DOD`) match exactly what `.github/workflows/ci.yml` runs; `build_run.sh` successfully
builds, launches, and screenshots the app on a simulator; and a whole-repo lint/format check is
clean (i.e. the tooling ignores `.harness/**` and the harness-authored root prose). Then mark this
task done so its dependents unblock: `.harness/scripts/mark-done.sh T001` (or the dashboard's "Mark
done").
