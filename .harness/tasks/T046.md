# T046 — MC question components: question header + four-state answer row

## Overview

Build the two presentational pieces of the multiple-choice session screen: the audio + phrase
question header, and the answer row with its five visual states (idle, correct, wrong-shake,
faded, correct-outline). Pure views — all state injected; no engine, no services.

## Do

Design reference: .harness/docs/designs/design-spec.md (read-only reference) §6a. Uses DesignSystem:
`PulsingAudioButton`, `CheckPopView`, `ShakeEffect`, `EnoughColor`; accent via
`\.accentTheme`. Both files in `Sources/Features/Session/`.

1. `Sources/Features/Session/MCQuestionHeader.swift` — `struct MCQuestionHeader: View` with
   `init(target: String, pronunciation: String, onPlayAudio: @escaping () -> Void)`.
   Centered VStack(spacing 14):
   - `PulsingAudioButton(size: 60, isPulsing: true, action: onPlayAudio)` — accent-tinted,
     pulsing ring while idle (behavior lives in the component).
   - `target` — 34 Bold, `EnoughColor.label`, multiline center.
   - `pronunciation` — 15 Regular, `#6b6b70`.
   - Prompt "What does this mean?" — 13 Semibold, uppercase-free, `#8e8e93`, top padding 6.
2. `Sources/Features/Session/AnswerRow.swift` — declare in this file:
   `enum AnswerRowState: Equatable { case idle, correct, wrongShake, faded, correctOutline }`
   and `struct AnswerRow: View` with
   `init(text: String, state: AnswerRowState, action: @escaping () -> Void)`.
   Base: full-width Button, min height 56, radius 15, text 17 Medium leading-aligned,
   padding 16×14. Per state:
   - `.idle` — fill `#f7f7fa` (`EnoughColor.insetSurface`), text `EnoughColor.label`.
   - `.correct` — fill `EnoughColor.successGreen`, text white, trailing
     `CheckPopView(size: 22, color: .white)` (pops on appear); apply a small spring scale
     (1 → 1.03 → 1) when entering this state.
   - `.wrongShake` — fill `#ffe9e6`, text `#b71824`, trailing `Image(systemName:
     "xmark.circle.fill")` in `#b71824`; apply `ShakeEffect` once on entering the state
     (~0.5s, `Motion.shake`).
   - `.faded` — fill `#f7f7fa`, text `#c0c0c6` (literal hex; not an EnoughColor token).
   - `.correctOutline` — fill white, 2pt `EnoughColor.successGreen` border (strokeBorder on
     the same radius-15 shape), text `EnoughColor.label`, trailing
     `Image(systemName: "checkmark.circle.fill")` in `EnoughColor.successGreen`.
   The button disables itself (`.allowsHitTesting(false)`) in every state except `.idle`.
3. State transitions animate with `Motion.popSpring` where a fill change occurs; the shake
   triggers exactly once per entry into `.wrongShake` (drive `ShakeEffect`'s `shakes` input
   from a local counter incremented in `onChange(of: state)`).
4. `#Preview` blocks: the header (use ありがとうございます / "arigatō gozaimasu"), and one
   stack showing all five `AnswerRow` states at once on a white canvas.

Copy string EXACT: "What does this mean?".
