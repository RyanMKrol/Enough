## Overview
There is no deep-link / shared-route mechanism today (MainShellView.selection is local @State; SessionRoute only has .complete). Add a minimal shared 'pending action' so an external caller (the App Intent) can request 'switch to Reviews and start a review session'.

## Do
Add a pending-route signal to AppState (e.g. var pendingAction: AppAction?, with a case .startReview). Have MainShellView observe it to switch its EnoughTab selection to .reviews, and have ReviewsTabView act on it to invoke its existing startReviewSession() (which calls services.study.makeReviewSession()). Expose a single entry point (e.g. AppState.requestStartReview()) callable synchronously from the main actor. Do not add a URL scheme unless needed by the intent; keep the hook internal so the App Intent unit can call it.

## Done when
Calling AppState.requestStartReview() switches the shell to the Reviews tab and triggers a review session via the existing StudyService path; a unit/logic test asserts the pending action flips tab selection and requests the session (using preview services / an in-memory graph); existing navigation stays green; xcodebuild test passes.
