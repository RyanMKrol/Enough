## Overview

Add the schema-versioning tripwire: a test proving a store created with `EnoughSchemaV1` +
`EnoughMigrationPlan` re-opens cleanly with the same plan. The moment a future change edits a
V1 model in place (instead of adding a V2 + migration stage), this test breaks loudly in CI
instead of corrupting user installs at launch.

## Do

   - Create an ON-DISK container at a unique temp URL under `FileManager.default.temporaryDirectory`
     (this test is the one sanctioned disk-touching persistence test — a scratch file, never the
     app's real store location; delete the temp store directory in `tearDown`). Use a dedicated
     `ModelConfiguration(url:)` pointing at the temp file, built from
     `Schema(versionedSchema: EnoughSchemaV1.self)` with `migrationPlan: EnoughMigrationPlan.self`.
   - Seed one record of each of the four model types; save; release the container (set to nil).
   - Open a SECOND container against the SAME url with the same schema + migration plan; fetch
     and assert all four seeded records are present and intact.

   `EnoughSchemaV2`: never edit V1's models; add the new schema + a MigrationStage to
   EnoughMigrationPlan, then EXTEND this test to seed with V1 and open with the full plan —

3. If `PersistenceStack` needs a small extension to build a container at an explicit URL
