## Overview

Build the onboarding flow container: a `NavigationStack` with typed steps, an `@Observable`
draft object that survives back-navigation (answers are never lost), and the completion handoff
that writes the trip and flips `AppState` to the main shell. Step screens themselves arrive in
T032–T036; this task wires the skeleton with placeholder step views.

## Do

1. Read design-spec.md §2 (intro paragraph — push/pop nav, back preserves answers) and
   architecture.md (§ Feature screens). Exact names below.
2. Create `Sources/Features/Onboarding/OnboardingDraft.swift` —
   `@Observable final class OnboardingDraft`:
   - `var selectedCountryId: String?`
   - `enum TripDuration: String { case weekend, week }` (nest or top-level in the same file;
     pin: top-level `TripDuration` so Persistence's `"weekend"|"week"` raw values match),
     `var duration: TripDuration = .weekend`
   - `var scenarioIds: Set<String> = []`
   - `var selectedBundleId: String?`
   - `var extraDeckIds: Set<String> = []`
   - computed `var accent: AccentTheme` — from the selected country's catalog `accent` string
     via `AccentTheme(rawValue:)`, defaulting `.japan`. Pass the resolved `CountryInfo?` in via
     a stored `var selectedCountry: CountryInfo?` kept in sync when the country id is set
     (single source: a `func selectCountry(_ country: CountryInfo)` that sets both).
3. Create `Sources/Features/Onboarding/OnboardingFlowView.swift`:
   - `enum OnboardingStep: Hashable { case country, tripShape, plan }`.
   - `struct OnboardingFlowView: View` — owns `@State private var draft = OnboardingDraft()`
     and `@State private var path: [OnboardingStep] = []`; reads `\.services` and the
     `AppState` (pin: `init(appState: AppState)`).
   - `NavigationStack(path: $path)`: root = `WelcomeView` placeholder for now
     (`Text("Welcome")` with a "Get started" `Button` pushing `.country`) — T032 replaces the
     root; `.navigationDestination(for: OnboardingStep.self)` switches to placeholder step
     views (`Text("Country")`, `Text("Trip shape")`, `Text("Plan")`) that T033/T034/T036
     replace. Because the SAME `draft` instance is passed down (environment:
     `.environment(draft)`), popping back never resets answers — do NOT recreate the draft on
     navigation.
   - Apply `.environment(\.accentTheme, draft.accent)` around the stack so the flow re-tints
     live as the country changes.
   - `func finish()` (internal, called by the plan step in T036): creates the
     `TripProfileRecord` via `services.tripStore` — countryId, `draft.duration.rawValue`,
     `Array(draft.scenarioIds)`, `startDate: services.dateProvider.now`,
     `accentRawValue: draft.accent.rawValue`, `isActive: true` (replacing any previous active
     trip via the store's replace/save API) — then calls
     `appState.completeOnboarding(accent: draft.accent)`. Purchases are ALREADY granted by the
     plan screen before it calls `finish()`; `finish()` does not touch entitlements.
4. In `Sources/App/RootView.swift`, replace the `.onboarding` placeholder with
   `OnboardingFlowView(appState: appState)`.
5. Nav chrome: steps use a back chevron in a small circular pill (`chevron.left`, frosted
   `#f2f2f7` circle ~34pt) via toolbar — provide a reusable
   `OnboardingBackButton` view in this directory the step screens can share; hide the default
   back button (`.navigationBarBackButtonHidden(true)`).

## Done when

- Launching fresh shows the flow's root; "Get started" pushes the country placeholder; pushing
  deeper and popping back preserves every `OnboardingDraft` value (state owned above the
  destinations).
- `finish()` writes exactly one active `TripProfileRecord` and flips `AppState.phase` to
  `.main` — proven by a unit test in `Tests/OnboardingFlowTests.swift` that builds an in-memory
  `AppServices`, sets draft values (japan, weekend, two scenarios), calls the finish logic, and
  asserts the record's fields + phase. (Extract the finish logic into a testable
  `OnboardingCompleter` helper or an `OnboardingDraft`+services function if calling the view's
  method is impractical.)
- The accent environment around the stack follows `draft.accent`.
