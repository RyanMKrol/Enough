# T040 — Debug setting: force all learned cards due now

## Overview

Add a debug-menu action that makes every card the user has ever studied due immediately, so the
review flow (Reviews tab, Home banner, review sessions) can be exercised on demand.

## Do

1. Create `Sources/Services/DebugSRSTools.swift`:

   ```swift
   enum DebugSRSTools {
     /// Sets dueAt = now on every CardSRSRecord whose statusRaw != "new"
     /// (i.e. learning + review cards; untouched cards have no row and stay New).
     /// Saves the context and returns the number of records updated.
     static func forceAllDue(store: CardSRSStore, now: Date) throws -> Int
   }
   ```

   Fetch the records with a `FetchDescriptor<CardSRSRecord>` executed from within
   `DebugSRSTools` on the store's `ModelContext` (exposed per the T017 contract). The
   persistence directory is OUT of this task's scope — do not add helpers to `CardSRSStore`
   itself; keep all new code in the two files this spec names.
2. Create `Sources/Features/Debug/DebugForceDue.swift`: `enum DebugForceDue` with
   `static let section = DebugSection(id: "srs", title: "Spaced repetition", rows: [...])`
   containing one `.action` row (id "force-due", title "Make everything due now", subtitle
   "Every studied card becomes reviewable immediately") whose closure calls
   `DebugSRSTools.forceAllDue(store:now:)` using `services.cardSRSStore` and
   `services.dateProvider.now`, returning `"N cards due"`.
3. Append `DebugForceDue.section` to the `DebugProviders.all` array literal in
   `Sources/Features/Debug/DebugRegistry.swift` (one-line edit; keep existing entries).
4. Add `Tests/DebugSRSToolsTests.swift` (in-memory ModelContainer): insert three
   `CardSRSRecord`s — one `statusRaw == "new"`, one "learning" due tomorrow, one "review" due in
   6 days — run `forceAllDue`, assert it returns 2, and that
   `CardSRSStore.dueRecords(now:ownedDeckIds:)` (with the records' deck ids owned) now returns
   exactly the learning + review records.

## Done when

- On a device state with studied-but-not-due cards, tapping "Make everything due now" toasts
  "N cards due" and the total-due count reported by `DeckProgressService.totalDue()` equals N
  immediately afterwards.
- New/unstudied cards remain New (no SRS row is created for them).
- `Tests/DebugSRSToolsTests.swift` passes with the assertions in step 4.
