# T043 — Home components: continue card, reviews banner, deck list row

## Overview

Build the three presentational components the Home (Learn) screen is assembled from — the
accent-tinted Continue hero card, the reviews-due banner, and the deck list row. Pure views:
all state is injected via init parameters; no services, no persistence, no navigation.

## Do

Design reference: .harness/docs/designs/design-spec.md (read-only reference) §4 (Learn tab) and §1 (tokens).
All three files go in `Sources/Features/Home/`. Use existing DesignSystem types
(`EnoughColor`, `EnoughFont`, `ProgressRing`, `AnimatedProgressBar`, `EyebrowLabel`) and read
the accent via `@Environment(\.accentTheme)`.

1. `Sources/Features/Home/ContinueCard.swift` — `struct ContinueCard: View` with
   `init(deckName: String, detailLine: String, progress: Double, onPlay: @escaping () -> Void)`.
   Layout: rounded rect radius 24, background = a subtle linear gradient of the accent tint
   (`accentTheme.tint` → `accentTheme.tint.opacity(0.55)`, topLeading → bottomTrailing).
   Interior padding 18. HStack(spacing: 14):
   - Left: `ProgressRing(progress: progress, size: 64, tint: accentTheme.accent)` — the ring
     animates on appear (that behavior lives inside `ProgressRing`; do not re-implement).
   - Middle (VStack, leading, spacing 4): `EyebrowLabel("Continue")`; deck name — 17 Semibold,
     `EnoughColor.label`; `detailLine` (e.g. "8 of 30 cards · about 6 min left") — 13 Regular,
     hex `#6b6b70`.
   - Right: a `Button(action: onPlay)` — 44×44 Circle filled `accentTheme.accent`, white
     `Image(systemName: "play.fill")` (~17pt), accessibility label "Continue learning".
   The WHOLE card is also tappable (`.onTapGesture` → `onPlay()`); keep the explicit play
   button too.
2. `Sources/Features/Home/ReviewsBanner.swift` — `struct ReviewsBanner: View` with
   `init(dueCount: Int, onReview: @escaping () -> Void)`. White surface card, radius 20,
   resting shadow `0 1 3 black@5%`, interior padding 16. HStack(spacing: 12):
   - 34×34 Circle filled `EnoughColor.easyTint` containing
     `Image(systemName: "arrow.triangle.2.circlepath")` in `EnoughColor.linkBlue` (~15pt).
   - VStack(leading, spacing 2): "\(dueCount) cards ready to review" — 15 Semibold,
     `EnoughColor.label`; "Spaced repetition · keeps them stuck" — 13 Regular, `#6b6b70`.
   - Spacer, then trailing "Review" pill: `Button(action: onReview)` — 13 Semibold text in
     `EnoughColor.linkBlue` on `EnoughColor.easyTint`, padding 8×14, fully rounded (Capsule).
3. `Sources/Features/Home/DeckListRow.swift` — declare
   `enum DeckRowStatus: Equatable { case learned, progress(Int, Int), new }` in this file, and
   `struct DeckListRow: View` with
   `init(iconSystemName: String, title: String, progress: Double, status: DeckRowStatus)`.
   HStack(spacing: 12), row vertical padding 12:
   - 34×34 RoundedRectangle(cornerRadius: 10) filled `accentTheme.tint` with
     `Image(systemName: iconSystemName)` in `accentTheme.accent` (~15pt).
   - VStack(leading, spacing 5): title — 17 Regular `EnoughColor.label`;
     `AnimatedProgressBar(progress: progress, tint:)` 4pt tall, tint = `EnoughColor.successGreen`
     when status == .learned, else `accentTheme.accent`.
   - Spacer, then trailing status view:
     - `.learned` → HStack(4): `Image(systemName: "checkmark.circle.fill")` in
       `EnoughColor.successGreen` (15pt) + "Learned" 13 Semibold `EnoughColor.successDeep`.
     - `.progress(a, b)` → "\(a)/\(b)" — 15 Regular, `#6b6b70`.
     - `.new` → "New" — 13 Semibold, `#6b6b70`, on `EnoughColor.insetSurface` (`#f7f7fa`)
       Capsule, padding 5×10.
4. Add `#Preview` blocks covering every state: ContinueCard (27% progress), ReviewsBanner
   (dueCount 12), and DeckListRow ×3 (learned / progress(8, 30) / new), each inside a
   `#f2f2f7` canvas so they read like the mock.

Copy strings are EXACT: "Continue", "Review", "Learned", "New",
"Spaced repetition · keeps them stuck", "\(dueCount) cards ready to review".

## Done when

- The three files above exist with the pinned type names, initializers, and `DeckRowStatus`
  cases — later tasks (T044) construct them with exactly those signatures.
- No component reads services, SwiftData, or global state; everything arrives via init or
  `\.accentTheme`.
- Every listed `#Preview` renders (compiles) for every state including all three
  `DeckRowStatus` variants.
- Colors/sizes match the values above (radius 24 hero / 20 banner, 64pt ring, 44pt play
  circle, 34pt icon tiles, exact hex values via `EnoughColor`).
