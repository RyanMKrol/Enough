# T027 — AudioService: per-card clip playback with auto-play-once semantics

## Overview

Every card has target-language audio that auto-plays ONCE when the card appears and can be
replayed via the speaker button (design-spec.md §6: "audio is core, not optional"). This task
builds the playback service with that bookkeeping; the UI components already exist
(`PulsingAudioButton`, T006) and sessions wire it later.

## Do

1. Create `Sources/Services/AudioService.swift` (`import AVFoundation`):

   ```swift
   final class AudioService {
     init(content: ContentStore)
     var playHandler: ((URL) -> Void)?   // test seam — when set, replaces real playback
     func play(fileName: String)
     func autoPlayOnce(cardId: String, fileName: String)
     func resetAutoPlay()
   }
   ```

2. `play(fileName:)`: resolve `content.audioURL(forFile: fileName)`; if nil, return silently
   (placeholder content may lack clips — never crash on missing audio). If `playHandler` is set,
   call it with the URL and return — this is the pinned test seam. Otherwise: on FIRST real
   playback only, configure the shared `AVAudioSession` — `setCategory(.playback, mode:
   .default)` + `setActive(true)` — inside `try?` (errors ignored; e.g. simulator quirks must
   not crash). Stop any currently playing clip, then create/retain an `AVAudioPlayer(contentsOf:
   url)` and `play()`.

3. `autoPlayOnce(cardId:fileName:)`: keep a private `var playedCardIds: Set<String>`; if
   `cardId` is already in the set do nothing; else insert it and call `play(fileName:)`.
   `resetAutoPlay()` empties the set — the session screen calls it whenever a NEW session
   starts, so each session auto-plays each card once (a re-queued missed card does NOT re-auto-
   play within the same session; the manual button covers replays).

4. Create `Tests/AudioServiceTests.swift` (XCTest). Use the real bundled content
   (`ContentStore()` — the hosted test bundle's `Bundle.main` contains the committed `Content/`
   fixture) and take a real audio file name from
   `ContentStore().cards(forDeck: "jp-greetings").first!.audio`. Always set `playHandler` in
   tests (append received URLs to an array) so no real audio plays on CI.

## Done when

Tests pin, at minimum:
- `play(fileName:)` with a real fixture file name invokes `playHandler` exactly once with a URL
  whose `lastPathComponent` equals that file name.
- `play(fileName: "does-not-exist.mp3")` does NOT invoke `playHandler` and does not crash.
- `autoPlayOnce(cardId: "c1", ...)` twice in a row → `playHandler` invoked exactly once;
  a different `cardId` with the same file → invoked again (keyed by card, not file).
- After `resetAutoPlay()`, `autoPlayOnce(cardId: "c1", ...)` plays again.
