## Overview

🔒 needs-human. One-time toolchain install so harness build agents can DRIVE the simulator UI
(navigate tabs, walk onboarding, assert on the accessibility tree) instead of only screenshotting
the launch screen. The Axiom plugin bundles `xcui`, but its input/tree engine (AXe) is a brew
install.

## Do

1. Run: `brew install cameroncooke/axe/axe`
2. Verify: `~/.claude/plugins/cache/axiom-marketplace/axiom/*/bin/xcui doctor` reports AXe found
   and lists your booted simulator(s).
3. Smoke-test against any booted simulator (ids exist once T065 has landed):
   `axe describe-ui --udid <UDID> | head` and one
   `xcui wait --for-element screen-home --timeout 10s --udid <UDID>` while the app shows Home.
4. The visual-verify overlay (`.harness/custom/visual-verify-build.md`) already documents the
   xcui recipes gated on "if axe is installed" — after this install they become live. Skim it
   once to confirm the recipes match your machine.

## Done when

`xcui doctor` is clean (AXe found), one tap/wait/assert round-trip works against the app on a
booted simulator, and the owner marks this done via `.harness/scripts/mark-done.sh T070`.
