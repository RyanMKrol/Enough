# T002 — Design tokens: colors, typography, layout & motion constants + accent theming

## Overview

Create the DesignSystem token layer every screen builds on: named colors, font helpers, layout
constants, motion constants, and the per-country `AccentTheme` with a SwiftUI environment key.
Values are transcribed from design-spec.md §1 (the design doc this task's `design` field points
at) and restated below — the numbers here are the contract.

## Do

1. Create `Sources/DesignSystem/Colors.swift`:
   - An internal `Color` extension initializer `init(hex: UInt32)` (e.g. `Color(hex: 0x3d7ede)`)
     that maps 0xRRGGBB to sRGB `Color`.
   - `enum EnoughColor` with `static let` SwiftUI `Color` constants (no other logic):
     `canvas` `#f2f2f7` · `surface` `#ffffff` · `insetSurface` `#f7f7fa` · `label` `#1c1c1e` ·
     `secondaryText` `#6b6b70` · `tertiaryText` `#8e8e93` · `faintText` `#a0a0a6` ·
     `inactiveTab` `#b0b0b6` · `hairline` = black at 8% opacity · `graphite` `#1c1c1e` ·
     `linkBlue` `#3d7ede` · `successGreen` `#1c8742` · `successTint` `#d8f9dd` ·
     `successDeep` `#00722e` · `easyBlue` `#1666aa` · `easyTint` `#caf7ff` ·
     `streakAmber` `#de6907` · `streakAmberTint` `#ffecdd`.
2. Create `Sources/DesignSystem/AccentTheme.swift`:
   - `enum AccentTheme: String, CaseIterable` with cases `japan`, `france`, `germany` (raw values
     `"japan"`/`"france"`/`"germany"` — these match the `accent` strings in `Content/catalog.json`).
   - Computed `var accent: Color`, `var tint: Color`, `var deep: Color`:
     japan `#e24947` / `#ffe9e6` / `#b71824`; france `#3d7ede` / `#e6f3ff` / `#1659b5`;
     germany `#d07b00` / `#ffeed9` / `#a95600`.
   - A SwiftUI environment key using the `@Entry` macro:
     `extension EnvironmentValues { @Entry var accentTheme: AccentTheme = .japan }`.
3. Create `Sources/DesignSystem/Typography.swift` — `enum EnoughFont` with static `Font` helpers
   (SF Pro = system font; sizes/weights fixed):
   `largeTitle()` 34 bold · `wordmark()` 44 bold · `screenTitle()` 31 bold ·
   `headline(_ size: CGFloat = 22)` semibold · `body()` 17 regular · `subhead()` 15 regular ·
   `footnote()` 13 regular · `eyebrow()` 12 semibold.
   Also static tracking multipliers (em units, applied by callers via `.kerning(size * multiplier)`):
   `tightTracking: CGFloat = -0.02`, `wordmarkTracking: CGFloat = -0.03`,
   `eyebrowTracking: CGFloat = 0.06`.
4. Create `Sources/DesignSystem/Layout.swift` — `enum Layout` with `static let` CGFloat constants:
   `buttonRadius = 16` · `buttonHeight = 54` · `chipRadius = 14` · `cardRadius = 20` ·
   `heroRadius = 26` · `flashcardRadius = 28` · `screenHPad = 22` · `cardPad = 18` ·
   `rowVPad = 12` · `sectionGap = 18`.
5. Create `Sources/DesignSystem/Motion.swift` — `enum Motion` with `static let` constants:
   `bobDuration: Double = 4.0` · `ringFill: Double = 1.2` · `barGrow: Double = 1.0` ·
   `checkPop: Double = 0.5` · `shake: Double = 0.5` · `pulse: Double = 2.2` ·
   `flicker: Double = 2.2` · `cardSwap: Double = 0.35` ·
   `popSpring: Animation = .spring(response: 0.5, dampingFraction: 0.55)` ·
   `swapSpring: Animation = .spring(response: 0.35, dampingFraction: 0.8)`.
6. Add `Tests/DesignTokensTests.swift` (XCTest, target EnoughTests) — see Done when.

Conventions: single app target (no new modules/imports), Swift 5 mode, iOS 18, 2-space indent,
SwiftLint strict (lines ≤ 130) and swift-format strict sweep `Sources/` and `Tests/`. No business
logic in DesignSystem. Do not touch any file outside `Sources/DesignSystem/**` and `Tests/**`.
