# T026 — PurchaseService protocol + stub implementation (buy, restore, owned stream)

## Overview

Define the purchase seam the whole app buys through — a protocol shaped like StoreKit 2 — plus
the v1 stub that grants entitlements instantly and persists them via `EntitlementStore` (T016).
Real StoreKit 2 replaces the stub in T062 behind this same protocol (design-spec.md §0:
"Purchases").

## Do

1. Create `Sources/Services/PurchaseService.swift`:

   ```swift
   enum EntitlementKind: String { case deck, bundle }
   enum PurchaseOutcome { case success, cancelled, pending }
   protocol PurchaseProviding: AnyObject {
     var ownedProductIds: Set<String> { get }
     func purchase(productId: String, kind: EntitlementKind) async throws -> PurchaseOutcome
     func restorePurchases() async throws
     func displayPrice(productId: String) -> String?
   }
   extension Notification.Name {
     /// Posted (on the main actor) whenever the owned-entitlements set may have changed.
     static let entitlementsChanged = Notification.Name("enough.entitlements.changed")
   }
   ```

   - `PurchaseOutcome` mirrors StoreKit 2's `Product.PurchaseResult`: `.cancelled` is a NORMAL
     return (user tapped Cancel on the payment sheet — never an error path), `.pending` means
     Ask to Buy / deferred approval (nothing is granted now; the entitlement arrives later via
     the store's update stream). Callers must handle all three (T036/T055 do).
   - Entitlement-change notification is **multicast by design**: implementations post
     `.entitlementsChanged` to `NotificationCenter.default` on the main actor after any grant
     change. Screens observe it independently — a single settable closure would let one screen
     silently steal another's subscription, and real StoreKit (T062) delivers external events
     (refunds, Ask to Buy approvals, Family Sharing) that ALL screens must see. There is no
     `onChange` property on the protocol.
   - `displayPrice(productId:)` is the ONLY sanctioned source of user-facing price strings.
     The stub formats from the catalog via `PricingCalculator` (below); the StoreKit
     implementation (T062) overrides with `product.displayPrice` so every storefront sees its
     localized price.

2. Create `Sources/Services/StubPurchaseService.swift`:

   ```swift
   final class StubPurchaseService: PurchaseProviding {
     init(entitlements: EntitlementStore, catalog: ContentCatalog?)
   }
   ```

   - `purchase(productId:kind:)`: `try? await Task.sleep(nanoseconds: 300_000_000)` (simulated
     store latency, so the UI's loading state is visible), then
     `entitlements.grant(productId: productId, kind: kind.rawValue)`, post
     `.entitlementsChanged`, and return `.success` (the stub ALWAYS succeeds — `.cancelled` and
     `.pending` only occur with real StoreKit). Purchasing an already-owned product returns
     `.success` without duplicating the grant (`EntitlementStore.grant` must already be
     idempotent per T016; rely on it).
   - `restorePurchases()`: the stub has nothing to restore — document that with a comment
     pointing at T062 (real StoreKit). It simply posts `.entitlementsChanged` so callers can
     refresh UI.
   - `ownedProductIds`: the set of granted product ids read from `EntitlementStore` on each
     access (no caching — the store is the truth).
   - `displayPrice(productId:)`: look the product up in the injected catalog (deck or bundle)
     and return `PricingCalculator.price(_:)` of its `priceGBP`; `nil` for an unknown id or a
     `nil` catalog.
3. Keep `EntitlementKind` string raw values aligned with `EntitlementRecord.kind`
   (`"deck"`/`"bundle"`, T016) — the store API takes the raw string.
4. Create `Tests/StubPurchaseServiceTests.swift` (XCTest): in-memory container via
   `PersistenceStack.container(inMemory: true)`, real bundled catalog via
   `ContentStore().catalog()` (hosted test bundle → `Bundle.main` has the `Content/` fixture).
   Async tests (`func test...() async throws`). Observe `.entitlementsChanged` with a local
   `NotificationCenter.default.addObserver(forName:...)` token captured into a counter;
   remove the token in the test (no shared state).

## Done when

Tests pin, at minimum:
- `purchase(productId: "jp-greetings", kind: .deck)` returns `.success`, `ownedProductIds`
  contains `"jp-greetings"`, exactly one `.entitlementsChanged` notification was posted, and
  `entitlements.isOwned(deckId: "jp-greetings", catalog:)` is true.
- Purchasing the SAME product twice leaves exactly one `EntitlementRecord` for it.
- `purchase(productId: "jp-weekend", kind: .bundle)` returns `.success`, `ownedProductIds`
  contains `"jp-weekend"`, and `entitlements.ownedDeckIds(catalog:)` now contains all three
  member decks (`"jp-greetings"`, `"jp-ordering-food"`, `"jp-at-the-bar"`) but NOT
  `"jp-getting-around"`.
- `restorePurchases()` posts `.entitlementsChanged` and does not alter `ownedProductIds`.
- `displayPrice(productId: "jp-greetings")` == `"£1.00"` and
  `displayPrice(productId: "jp-weekend")` == `"£2.49"` (stub formats from the catalog);
  unknown id → `nil`.
