# T047 — Multiple-choice learn session screen (states, sheets, haptics, transitions)

## Overview

Build the full multiple-choice session — the learn/practice mode of the app — on top of
`SessionEngine`/`StudyService`, the T046 components, and `BottomResultSheet`; wire its launch
points (Home Continue card, deck detail's Continue learning / Practice all N).

## Do

Design reference: .harness/docs/designs/design-spec.md (read-only reference) §6 + §6a. Architecture:
`SessionEngine`, `StudyService`, `SRSEngine.previewLabel` in

1. `Sources/Features/Session/SessionChrome.swift` — shared top bar for both session screens:
   `struct SessionChrome: View`, `init(progress: Double, counterText: String, onClose:

   `counterText` (e.g. "3/12") 13 Semibold `#8e8e93`, fixed trailing width.
2. `Sources/Features/Session/SessionRoute.swift` —
   `enum SessionRoute: Hashable { case complete }` (placeholder route; T050 replaces the

3. `Sources/Features/Session/MCSessionViewModel.swift` — `@Observable final class
   MCSessionViewModel`. `init(engine: SessionEngine, services: AppServices)`. Responsibilities:
   - **Options**: for the current card, 4 answer strings = the card's `english` + 3 distractor
     `english` values from OTHER cards of the same deck; if the deck has < 4 cards, top up
     from the active country's other owned decks. Selection + order are DETERMINISTIC ACROSS
     LAUNCHES — pinned rule: sort candidate cardIds by `String` order, seed a
     `SeededRandomNumberGenerator` (write a tiny SplitMix64 in this file) with a STABLE hash of
     the CURRENT card's id, pick 3, then shuffle the 4 options with the same generator. The
     stable hash is FNV-1a over the id's UTF-8 bytes — `hashValue` is BANNED here (it is
     per-process randomly seeded; it would pass a single-process test yet reshuffle every
     launch):

     ```swift
     func fnv1a(_ s: String) -> UInt64 {
       var h: UInt64 = 0xcbf29ce484222325
       for b in s.utf8 { h = (h ^ UInt64(b)) &* 0x100000001b3 }
       return h
     }
     ```

     Same card ⇒ same options in every process, every launch.
   - **Answer flow**: `select(option:)` — if correct: row → `.correct`, others → `.faded`,
     engine `submitMultipleChoice(correct: true)`; sheet = `.correct(intervalLabel:)` where
     intervalLabel = `SRSEngine.previewLabel(.good, for: currentCardState)`. If wrong: chosen
     row → `.wrongShake`, correct row → `.correctOutline`, engine
     `submitMultipleChoice(correct: false)` (engine re-queues the miss later in the session);
     sheet = `.incorrect(target:english:)`.
   - **Haptics** (canonical mechanism, view layer — never construct feedback generators):
     `.sensoryFeedback(.impact(weight: .light), trigger: vm.selectionCount)` for the tap, and
     `.sensoryFeedback(trigger: vm.lastResult) { _, new in new == .correct ? .success :
     new == .wrong ? .error : nil }` for the outcome, fired at the visual moment (check-pop /
     shake start). The system handles prepare/reuse; haptics no-op on the simulator — do not
     try to verify them in screenshots.
   - **Progress**: expose `progressText` "\(done)/\(total)" and `progressValue` from
     `engine.progress` — they advance on GRADE (sheet shown), not on card show.
   - **Advance**: `advance()` → engine.advance(), clear sheet/row states; if
     `engine.isComplete`, first `try services.study.commit(engine)` then set
     `route = SessionRoute.complete`.
   - **Close (X)**: `closeAndCommit()` → `services.study.commit(engine)` with whatever is
     graded so far, then dismiss. No confirmation dialog.
   - **Audio**: on session start `services.audio.resetAutoPlay()`; on each card appearance
     `services.audio.autoPlayOnce(cardId:fileName:)`; header's speaker replays via
     `services.audio.play(fileName:)`.
4. `Sources/Features/Session/MCSessionView.swift` — `struct MCSessionView: View`,
   `init(engine: SessionEngine)` (builds the VM with `\.services`). White canvas. Layout:
   `SessionChrome` top; centered `MCQuestionHeader`; 4 `AnswerRow`s (spacing 10). Result
   sheets via `BottomResultSheet`:
   - Correct: `EnoughColor.successTint` bg — "Nice." 20 Bold `EnoughColor.successDeep` +
     "You'll see this again in \(intervalLabel)" 15, + trailing `chevron.forward` circle.
   - Incorrect: `#ffe9e6` bg — "Not quite" 20 Bold `#b71824` + an AttributedString line
     "\(target) is the **\(english)**. We'll bring this back soon." (english bolded; if the
     card has `notes`, ignore them here — english is the key word).
   - Tap anywhere on the sheet, its chevron, or swipe down ⇒ `vm.advance()`.
   - **Card-to-card transition**: outgoing question block slides off LEFT as the next rises
     FROM BELOW — `.transition(.asymmetric(insertion: .move(edge: .bottom), removal:
     .move(edge: .leading)))` animated with `Motion.cardSwap` spring (~0.35s), keyed by
     `.id(currentCardId)`.
   - `.navigationDestination(for: SessionRoute.self)` → placeholder
     `Text("Complete") // TODO(T050): SessionCompleteView`.
5. **Launch wiring** (Home + DeckDetail scopes included for exactly these edits):
