# T023 — SessionEngine: pure session state machine (learn/review/practice)

## Overview

Build the pure state machine that runs one card session (design-spec.md §6): the queue cursor,
multiple-choice answering with miss re-queueing (learn/practice), Again/Hard/Good/Easy grading
(review), progress counting, and the summary the completion screen shows. No SwiftUI, no
SwiftData — `import Foundation` only, in `Sources/SRSKit/`.

## Do

1. Create `Sources/SRSKit/SessionEngine.swift`:

   ```swift
   struct SessionCard: Equatable {
     let deckId: String
     let cardId: String
     var state: SRSState
   }
   enum SessionMode: Equatable { case learn, review, practice }
   struct SessionSummary: Equatable {
     let cardsCompleted: Int
     let correctCount: Int
     let duration: TimeInterval
     let mode: SessionMode
   }
   final class SessionEngine {
     init(mode: SessionMode, cards: [SessionCard], now: @escaping () -> Date)
     var mode: SessionMode { get }
     var current: SessionCard? { get }          // nil when complete
     var progress: (done: Int, total: Int) { get }
     var isComplete: Bool { get }
     var gradedResults: [(card: SessionCard, grade: SRSGrade)] { get }
     func submitMultipleChoice(correct: Bool) -> SRSGrade?  // learn & practice modes only
     func submitGrade(_ grade: SRSGrade)                    // review mode only
     func advance()
   }
   ```

2. Internals: a mutable `queue: [SessionCard]` + `index: Int`; `startedAt = now()` captured in
   `init`; a `firstAttemptGrade: [String: SRSGrade]` keyed `"\(deckId)/\(cardId)"`; a
   `completedIds: Set<String>` (same key). `total` = count of UNIQUE cards passed to `init`.
   `done` = `completedIds.count` — i.e. progress advances on grade/correct answer, not on card
   show (design-spec.md §6).

3. `submitMultipleChoice(correct:)` (assert/no-op in review mode):
   - **First attempt** for this card (no entry in `firstAttemptGrade`): record
     `correct ? .good : .again` into `firstAttemptGrade`; return that grade in `.learn` mode,
     `nil` in `.practice` mode (practice never writes SRS).
   - **Any attempt, wrong**: re-insert a copy of the current card into `queue` at
     `index + 1 + SRSQueueBuilder.requeuePosition(queueLength: queue.count - index - 1)`
     (clamp to `queue.count`). The card is NOT completed.
   - **Any attempt, correct**: insert the card's key into `completedIds`.
   - A repeat attempt (key already in `firstAttemptGrade`) never changes the recorded grade and
     returns `nil`.
4. `submitGrade(_:)` (review mode; assert/no-op in learn/practice): records the grade into
   `firstAttemptGrade` (first grade wins) and marks the card completed — review cards are never
   re-queued; the SRS interval handles when they come back.
5. `advance()`: `index += 1`. `current` = `queue[index]` while in bounds AND the entry's key is
   not already in `completedIds` — skip completed duplicates automatically (advance past them).
   `isComplete` = every unique card completed. In practice mode a wrong card re-queues the same
   way, so the session still ends only when every card has been answered correctly once.
6. `gradedResults`: one entry per unique card IN FIRST-SEEN order with its first-attempt grade —
   EMPTY in practice mode. `summary()`: `cardsCompleted = done`; `correctCount` = learn/practice:
   cards whose first-attempt grade is `.good`; review: grades `.good` or `.easy`;
   `duration = now().timeIntervalSince(startedAt)`.
7. Create `Tests/SessionEngineTests.swift` (XCTest). Use a controllable clock:
   `var fake = Date(timeIntervalSince1970: 1_800_000_000)` captured by the `now` closure.

## Done when

Tests pin, at minimum (cards A, B, C in one deck):
- **Learn with one miss**: answer A correct (`returns .good`), advance; answer B WRONG (`returns
  .again`) — B re-enters 2 positions later, i.e. remaining `[C, B]`; advance; C correct; advance;
  current is B again; B correct returns `nil` (repeat attempt); `isComplete` true only now;
  `progress == (3, 3)`; `gradedResults` grades are `[A: .good, B: .again, C: .good]` in that
  order; `summary().correctCount == 2`.
- **Requeue clamp**: miss the LAST card of a session (nothing after it) — it re-enters
  immediately next (`requeuePosition(queueLength: 0) == 0`) and the session completes after it
  is answered correctly.
- **Review**: three cards graded `.easy/.again/.good` complete in three `submitGrade`+`advance`
  steps with `gradedResults` in order and `summary().correctCount == 2`; `submitMultipleChoice`
  in review mode changes nothing.
- **Practice**: a full run with one miss ends with `gradedResults` EMPTY and
  `summary().mode == .practice`, `cardsCompleted == 3`.
- **Duration**: advancing the fake clock 250 s between `init` and `summary()` yields
  `duration == 250`.
