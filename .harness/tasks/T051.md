# T051 — Reviews tab (due ring, Due/Learning/Mastered tiles, strength bars)

## Overview

Build the Reviews tab — the retention surface that shows how many cards are due, the
Due/Learning/Mastered split, and per-deck strength — and wire it into the main shell in place of
the reviews placeholder. Design reference: .harness/docs/designs/design-spec.md §7.1 (and §1
for tokens); architecture contract: .harness/docs/designs/architecture.md.

## Do

1. Create `Sources/Features/Reviews/ReviewsTabView.swift` — the screen for the `.reviews` tab.
   Read services via `@Environment(\.services)` (type `AppServices`) and the accent via
   `@Environment(\.accentTheme)`. Canvas background `EnoughColor.canvas` (`#f2f2f7`). Layout,
   top → bottom, inside a `ScrollView`:
   - Large title **"Reviews"** (`EnoughFont.largeTitle()`), subtitle **"Keeping <destination>
     fresh"** in 15 Regular `#6b6b70`. Destination comes from the active trip's country id:
     `japan` → "Tokyo", `france` → "Paris", `germany` → "Berlin" (fallback: the country's
     `name`). Get the active trip from `services.tripStore.activeTrip()` and the country via
     `services.contentStore.country(_:)`.
   - **Hero card**: white (`EnoughColor.surface`), radius 20, resting shadow. Contents centered:
     a 120pt `ProgressRing` (tint = accent) whose progress is `due / (due + learning)` (clamped
     to 0...1; show 1.0 when both are zero) with `showsPercent: false`, overlaid center stack:
     the due count in 30 Bold + **"due now"** in 13 `#6b6b70`. Below the ring: **"A quick
     <m>-minute round keeps these from slipping."** in 15 `#6b6b70`, where `m =
     max(1, Int(ceil(Double(due) * 20.0 / 60.0)))` (20 s per card). Then a **"Start review"**
     button — `PrimaryButtonStyle(background: accent)` — that presents `ReviewSessionView` via
     `.fullScreenCover`. When `due == 0` the button is disabled and the card shows the empty
     state instead of the count: **"Nothing due right now"** (17 Semibold) plus a next-due line
     ("Next review <relative time>" using `Date.RelativeFormatStyle`, from the earliest future
     `dueAt` among `services.cardSRSStore` records; omit the line if none).
   - **Three stat tiles** in an `HStack` (equal widths, white cards radius 18): count in
     22 Bold + label in 12 Semibold uppercase. Counts from
     `services.deckProgress.totals()` → `(due, learning, mastered)`. Colors: **Due** count in
     `#b71824`, **Learning** in `EnoughColor.streakAmber`, **Mastered** in
     `EnoughColor.successGreen`.
   - **Per-deck strength list**: one white grouped card; a row per OWNED deck
     (`services.entitlementStore.ownedDeckIds(catalog:)`, catalog order): a `StrengthBars`
     glyph, deck `title` (17 Regular), spacer, **"<n> due"** (13, `#8e8e93`) from
     `services.deckProgress.progress(forDeck:)`. Hairline dividers between rows.
2. Create `Sources/Features/Reviews/StrengthBars.swift`: `struct StrengthBars: View` with
   `init(strength: Int)` (0–3). Three vertical rounded bars, each 4×14pt, corner radius 2,
   3pt apart; the first `strength` bars filled `EnoughColor.successGreen`, the rest `#e5e5ea`.
   Include a `#Preview` showing strengths 0–3.
3. In `Sources/App/MainShellView.swift`, replace the `.reviews` placeholder with
   `ReviewsTabView()`. Do not change the other tabs.
4. Add `#Preview` for `ReviewsTabView` using `AppServices.preview()`.

## Done when

- The Reviews tab renders from live service data — no hardcoded counts anywhere; zero-due shows
  the empty state with a disabled button.
- "Start review" presents `ReviewSessionView` full-screen (tab bar hidden while presented).
- `StrengthBars` shows exactly `strength` green bars (0–3) and grey remainders.
- Verify on the simulator: launch with `-demo-state` (seeds 12 due, 3 owned decks, streak 6) —
  the hero reads "12 due now" with "A quick 4-minute round keeps these from slipping.", the
  three tiles show non-zero Due/Learning/Mastered counts in red/amber/green, and each owned
  deck row shows its strength bars and due count. Capture via `./build_run.sh`.
