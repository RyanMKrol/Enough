## Overview

Add `TripStore`, the persistence gateway for the user's active trip (destination country, trip
duration, chosen scenarios) including the "Day N" counter Home shows ("Weekend in Tokyo ·
Day 2"). Builds on the `TripProfileRecord` model defined in T014 — do NOT change that model's
fields.

## Do

1. Create `Sources/Persistence/TripStore.swift`:

   ```swift
   final class TripStore {
     init(context: ModelContext)

     /// The single record with isActive == true (nil before onboarding completes).
     func activeTrip() throws -> TripProfileRecord?

     /// Deactivates any existing active trip(s) (isActive = false), inserts the new record
     /// with isActive = true, and saves. Returns the inserted record.
     @discardableResult
     func saveNewTrip(
       countryId: String,
       duration: String,          // "weekend" | "week"
       scenarioIds: [String],
       accentRawValue: String,
       startDate: Date
     ) throws -> TripProfileRecord

     /// 1-based day counter: Day 1 on startDate itself, Day 2 the next calendar day, etc.
     /// Computed from whole calendar days between startOfDay(startDate) and startOfDay(now).
     /// Returns nil when there is no active trip.
     func dayNumber(now: Date) throws -> Int?

     /// Deletes ALL TripProfileRecord rows (active and inactive) and saves.
     func reset() throws
   }
   ```

   - `dayNumber` uses `Calendar.current.dateComponents([.day], from:to:)` between the two
     start-of-days, `+ 1`. A `now` earlier than `startDate` clamps to 1.
   - Old trips are kept as inactive rows by `saveNewTrip` (history is cheap); only `reset()`
     deletes.

2. Create `Tests/TripStoreTests.swift` (XCTest, class `TripStoreTests`). Every test uses
   `try PersistenceStack.container(inMemory: true)` + a fresh `ModelContext` — never on-disk.
   Build fixed dates with `DateComponents` via `Calendar.current` (do not depend on the wall
   clock for day math).

Conventions: single app target (`@testable import Enough`), Swift 5 / iOS 18, 2-space indent,
SwiftLint strict (lines ≤ 130).

## Done when

- `Tests/TripStoreTests.swift` asserts:
  - `activeTrip()` is nil on an empty store.
  - After `saveNewTrip(countryId: "japan", duration: "weekend", scenarioIds: ["greetings"],
    accentRawValue: "japan", startDate: <fixed date>)`: `activeTrip()` is non-nil with those
    exact values and `isActive == true`.
  - Calling `saveNewTrip` a SECOND time (e.g. countryId "france"): `activeTrip()!.countryId ==
    "france"`, and fetching all `TripProfileRecord`s yields 2 rows of which exactly 1 has
    `isActive == true` (the old trip was deactivated, not deleted).
  - Day counter with a trip started on a fixed date D: `dayNumber(now: D) == 1`;
    `dayNumber(now: D + 1 day) == 2`; `dayNumber(now: D + 1 day at 23:59) == 2` (calendar
    days, not 24-hour blocks); `dayNumber(now: D - 1 day) == 1` (clamped);
    `dayNumber` is nil when no trip exists.
  - After `reset()`: `activeTrip()` is nil and a fetch of all rows is empty.
- No file outside `Sources/Persistence/` and `Tests/` changes.
