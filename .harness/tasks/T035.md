## Overview

Build the two presentational rows the plan screen (T036) is assembled from: the radio-style
bundle row and the pack checklist row. Pure views — all state in via init, all interaction out
via callbacks, with `#Preview`s covering every visual state.

## Do

1. Read design-spec.md §2.4 and architecture.md (§ Feature screens). Both views live in
   `Sources/Features/Onboarding/` and depend only on DesignSystem tokens + `\.accentTheme` —
   no services, no ContentKit types (strings and numbers in).
2. Create `Sources/Features/Onboarding/BundleRowView.swift` —
   `struct BundleRowView: View` with
   `init(title: String, subtitle: String, price: String, isPopular: Bool, isSelected: Bool,
   action: () -> Void)`:
   - White card, radius 18, interior padding ~16, row layout: radio indicator (22pt circle —
     selected: accent fill with a 8pt white inner dot; unselected: 1.5pt `#c7c7cc` ring),
     then a column of title 17 semibold `#1c1c1e` (+ inline "POPULAR" tag when `isPopular`:
     11 bold white on accent, padding 3×7, radius 99, 6pt leading gap) and subtitle 13 regular
     `#6b6b70` (e.g. "3 essential packs · save £0.51" — the STRING is computed by the caller,
     never in this view), then `Spacer()`, trailing price 17 bold `#1c1c1e` (e.g. "£2.49").
   - Selected: `overlay` stroke accent 2pt + shadow `accent.opacity(0.18), radius: 12, y: 5`.
     Unselected: hairline stroke `Color.black.opacity(0.08)` 1pt, no shadow.
   - Whole row is one `Button` (light impact haptic); selection change animates with a small
     spring.
3. Create `Sources/Features/Onboarding/PackChecklistRow.swift` —
   `struct PackChecklistRow: View` with
   `init(name: String, cardCount: Int, state: PackRowState, onToggle: (() -> Void)?)` and
   `enum PackRowState { case inBundle(bundleName: String), extra(priceLabel: String),
   available(priceLabel: String) }`:
   - Row height ~52, leading 22pt indicator: `inBundle` and `extra` = filled accent circle
     with white 11pt bold `checkmark`; `available` = 1.5pt `#c7c7cc` empty ring.
   - Middle: name 16 regular `#1c1c1e` + "<cardCount> cards" 13 regular `#8e8e93` beneath.
   - Trailing: `inBundle` → the bundle name (e.g. "Weekend") 13 semibold in accent;
     `extra` → "Added · +\u{00A0}£1"-style is NOT designed — pin: show the priceLabel
     (e.g. "+ £1") 13 semibold in accent for `extra`, and the same label in `#6b6b70` for
     `available`.
   - `onToggle` non-nil ⇒ the row is a Button (à-la-carte toggle; light haptic);
     nil ⇒ static (bundle members can't be removed).
   - No card chrome — these stack inside the screen's grouped white card; provide the
     hairline divider OUTSIDE (T036 insets dividers 49pt), not in this view.
4. `#Preview`s: BundleRowView selected+popular, unselected; PackChecklistRow in all three
   states (use Japan accent environment).

## Done when

- Both views compile with exactly the pinned initializers (T036 binds to them verbatim) and
  contain zero business logic — no price math, no catalog access.
- Previews render: selected bundle row shows accent ring + tinted shadow + filled radio +
  POPULAR tag; unselected shows hairline + empty radio; checklist rows show filled-check vs
  empty-ring indicators with the correct trailing treatments.
- All visual constants match design-spec §2.4 (22pt indicators, 17 semibold titles, POPULAR
  pill, accent ring 2pt).
