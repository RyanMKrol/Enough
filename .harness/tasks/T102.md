## Overview
Re-attempt of T040 (a debug-menu action that makes every studied card due now). The first attempt blocked on scope-creep: the original spec claimed CardSRSStore's ModelContext was 'exposed per the T017 contract' and forbade adding helpers to CardSRSStore, but T017's store keeps `private let context: ModelContext` with NO accessor — so the only way to bulk-fetch/mutate records was to edit CardSRSStore.swift (out of scope) or build a throwaway ModelContext. This re-attempt resolves the contradiction: it explicitly puts the bulk-mutation on CardSRSStore (mirroring the existing `reset()` method) and widens scope to include Sources/Persistence/**.

## Do
1. Add a bulk-mutation method to `Sources/Persistence/CardSRSStore.swift`, mirroring the existing `reset()` method's shape (fetch all records on the store's own private `context`, mutate, `try context.save()`). Signature:

   ```swift
   /// Sets dueAt = now on every CardSRSRecord whose statusRaw != "new"
   /// (learning + review cards; untouched cards have no row and stay New).
   /// Saves the context and returns the number of records updated.
   @discardableResult
   func forceAllDue(now: Date) throws -> Int
   ```
   Fetch with `FetchDescriptor<CardSRSRecord>(predicate: #Predicate { $0.statusRaw != "new" })`, set `dueAt = now` on each, save, return `count`. Do NOT filter by owned decks — the debug tool makes EVERY studied card due.

2. Create `Sources/Services/DebugSRSTools.swift`: `enum DebugSRSTools` with `static func forceAllDue(store: CardSRSStore, now: Date) throws -> Int` that simply delegates to `store.forceAllDue(now:)`. (Keeping the enum wrapper preserves the debug-tooling-backend layering the original spec intended, and gives the test a stable entry point.)

3. Create `Sources/Features/Debug/DebugForceDue.swift`: `enum DebugForceDue` with `static let section = DebugSection(id: "srs", title: "Spaced repetition", rows: [...])` containing ONE `.action` `DebugRow` (id "force-due", title "Make everything due now", subtitle "Every studied card becomes reviewable immediately"). Follow the exact `.action { @MainActor services in ... }` closure shape used by `DebugReset`/`DebugDemoSeed`. Inside the closure call `DebugSRSTools.forceAllDue(store: services.cardSRSStore, now: services.dateProvider.now)` and return `"\(count) cards due"`.

4. Append `DebugForceDue.section` to the `DebugProviders.all` array literal in `Sources/Features/Debug/DebugRegistry.swift` (one-line edit; keep the existing entries and the trailing-comma swiftlint disable on the last line).

5. Add `Tests/DebugSRSToolsTests.swift` using an in-memory `ModelContainer` for `CardSRSRecord`. Insert three records via a `CardSRSStore` built on that context: one `statusRaw == "new"`, one `"learning"` with `dueAt` = tomorrow, one `"review"` with `dueAt` = 6 days out. Call `DebugSRSTools.forceAllDue(store:now:)` and assert it returns 2. Then assert `store.dueRecords(now: now, ownedDeckIds: <both records' deckIds>)` returns exactly the learning + review records (the new one is excluded).

## Done when
- `CardSRSStore.forceAllDue(now:)` exists and sets `dueAt = now` on every non-"new" record, saving the context, returning the count; the New/unstudied record keeps no row (none is created).
- The Debug menu shows a "Spaced repetition" section with a "Make everything due now" action that toasts "N cards due"; after tapping it, `DeckProgressService.totalDue()` equals N.
- `Tests/DebugSRSToolsTests.swift` passes: `forceAllDue` returns 2 for the three-record fixture, and `dueRecords(now:ownedDeckIds:)` afterwards returns exactly the learning + review records.
- No file outside the declared scope (`Sources/Features/Debug/**`, `Sources/Services/**`, `Sources/Persistence/**`, `Tests/**`) is modified.
