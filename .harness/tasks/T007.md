# T007 — Feedback motion: check pop, shake modifier, bottom result sheet

## Overview

Add the three feedback primitives sessions are built from: the springy check-pop, the
wrong-answer shake, and the bottom result sheet that slides up after an answer.
Motion values from design-spec.md §1.4 and §6a.

## Do

1. Create `Sources/DesignSystem/FeedbackViews.swift` with exactly these three types:
   - `struct CheckPopView: View` with `init(size: CGFloat, color: Color)`. A filled `Circle` in
     `color` with a white bold `checkmark` SF Symbol (symbol ~`size * 0.5`), frame
     `size × size`. On appear it "pops": scale animates 0 → 1 using `Motion.popSpring`
     (`.spring(response: 0.5, dampingFraction: 0.55)` — the overshoot approximates the design's
     0 → 1.18 → 1 cubic-bezier(.2,1.5,.4,1)). Starts invisible/scale-0 and stays at 1 after the
     spring settles.
   - `struct ShakeEffect: GeometryEffect` with `var shakes: CGFloat` as `animatableData`.
     `effectValue` translates X by `8 * sin(shakes * .pi * 6)` — 3 oscillations of ±8pt across
     one unit of `shakes`. Callers apply `.modifier(ShakeEffect(shakes: attemptCount))` and
     animate `attemptCount += 1` with `.easeInOut(duration: Motion.shake)` (0.5s).
   - `struct BottomResultSheet<Content: View>: View` with `init(tint: Color,
     onContinue: (() -> Void)? = nil, @ViewBuilder content: () -> Content)`. A bottom-pinned
     panel for use inside a `ZStack(alignment: .bottom)`: background `tint`, clipped to
     `UnevenRoundedRectangle(topLeadingRadius: 24, topTrailingRadius: 24)`, content padded 20
     with bottom safe-area respected, laid out as `HStack { content; Spacer(); chevron }` where
     the chevron (shown only when `onContinue != nil`) is a 44pt circular button with a white
     `chevron.right` on `Color.black.opacity(0.15)`, calling `onContinue`. Give the sheet a
     `.transition(.move(edge: .bottom))` so callers get the slide-up for free when they insert
     it with an animation.
2. Add a `#Preview` showing: a 56pt CheckPopView in `EnoughColor.successGreen`; a rectangle you
   can visually confirm shakes (static preview is fine); and a BottomResultSheet in
   `EnoughColor.successTint` containing a "Nice." headline + "You'll see this again in 2 days"
   subline with a continue chevron.

Conventions: single app target, SwiftUI only, 2-space indent, SwiftLint strict (≤130 cols) +
swift-format strict. These are dumb presentational pieces — no haptics, no timers, no session
logic (that lands in the session tasks). Touch only `Sources/DesignSystem/**` (and `Tests/**`
if adding optional tests).

## Done when

- `Sources/DesignSystem/FeedbackViews.swift` compiles with the exact three type names and
  signatures above (session tasks reference them verbatim).
- CheckPopView pops on appear via `Motion.popSpring` and renders `size`-scaled; ShakeEffect is a
  `GeometryEffect` whose animatable `shakes` yields 3 horizontal oscillations of ±8pt per unit;
  BottomResultSheet has top-only 24pt corners, the tint background parameter, and shows its
  chevron continue button only when a handler is provided.
- The `#Preview` block exists with the three demonstrations described.
