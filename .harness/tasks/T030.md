## Overview

Build the floating "liquid-glass" tab bar component and the `MainShellView` that hosts the four
top-level tabs (Learn, Reviews, Progress, Browse) as titled placeholders, replacing T029's
`Text("Main")` placeholder in `RootView`.

## Do

1. Read design-spec.md §3 and architecture.md (§ App shell). Exact names below.
2. Create `Sources/App/EnoughTab.swift` — `enum EnoughTab: String, CaseIterable, Identifiable`:
   cases `learn, reviews, progress, browse`. Give each: `var title: String` ("Learn",
   "Reviews", "Progress", "Browse") and `var symbol: String` — pin these SF Symbols:
   learn `"rectangle.stack.fill"`, reviews `"arrow.trianglehead.2.clockwise.rotate.90"`,
   progress `"chart.bar.fill"`, browse `"bag.fill"`. (If `arrow.trianglehead.2.clockwise.rotate.90`
   is unavailable on the iOS 18 SDK, fall back to `"arrow.2.circlepath"` and note it in the
   worklog.)
3. Create `Sources/DesignSystem/LiquidGlassTabBar.swift` —
   `struct LiquidGlassTabBar: View { init(selection: Binding<EnoughTab>) }`:
   - Horizontal pill of 4 equal tab items; each item = symbol (17–20pt) above its 10pt
     semibold title label, vertical padding ~10, min height ~58.
   - Background: capsule/rounded-rect radius 26 — `.ultraThinMaterial` overlaid with
     `Color.white.opacity(0.82)` blended to read as `rgba(255,255,255,.82)` glass (material
     provides the blur; do NOT try to replicate saturate(180%) beyond the material).
   - Shadow `color: .black.opacity(0.1), radius: 22, y: 6`; overlay stroke
     `Color.black.opacity(0.06)`, lineWidth 0.5 (the inset hairline).
   - Active tab: icon + label in `accentTheme.accent` (from `\.accentTheme`); inactive:
     `#b0b0b6` (`EnoughColor.faintText` if that token maps to `#b0b0b6`, otherwise
     `Color(red:)` from hex b0b0b6 locally).
   - Tap switches `selection` with a light impact haptic
     (`UIImpactFeedbackGenerator(style: .light)`).
4. Create `Sources/App/MainShellView.swift` — `struct MainShellView: View`:
   - `@State private var selection: EnoughTab = .learn`.
   - `ZStack(alignment: .bottom)`: the selected tab's screen, then the `LiquidGlassTabBar`
     floating with horizontal padding ~44 and bottom padding 18 above the safe area.
   - Per-tab content for now = placeholder screens each later task replaces: a full-screen
     `EnoughColor.canvas` background with the tab's title as a large-title Text. Pin the
     placeholder factory as a `@ViewBuilder private func screen(for tab: EnoughTab)` switch so
     each later tab task swaps exactly one case.
5. In `Sources/App/RootView.swift`, replace the `.main` placeholder `Text("Main")` with
   `MainShellView()`.
6. Add `#Preview` for `MainShellView` (Japan accent default).

## Done when

- `LiquidGlassTabBar` renders 4 tabs; tapping changes selection and restyles active/inactive
  colors; component compiles standalone with only DesignSystem dependencies (Binding in, no
  services).
- `MainShellView` shows the Learn placeholder by default with the bar floating at the bottom;
  `RootView` in phase `.main` shows it.
- Verify on the simulator (`./build_run.sh`): with no trip the app still opens on onboarding, so
  TEMPORARILY drive the screenshot by previewing/launching with a seeded trip if available, or
  confirm via the `#Preview` + attach a simulator screenshot after completing onboarding is NOT
  yet possible — acceptable evidence: the screenshot shows the app running; plus the Xcode
  preview (or a UI-less unit test constructing `MainShellView`) proving the shell builds. If the
  screenshot cannot show the tab bar yet (onboarding not built), say so explicitly in the
  worklog rather than faking it.
