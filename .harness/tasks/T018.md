## Overview

Add `ActivityStore`, the persistence gateway for the per-day study log (cards reviewed, cards
learned, seconds studied) that later powers streaks, week dots, and the Progress tab's totals.
Builds on `DailyActivityRecord` from T014 — do NOT change that model's fields.

## Do

1. Create `Sources/Persistence/ActivityStore.swift`:

   ```swift
   final class ActivityStore {
     init(context: ModelContext)

     /// The row whose `day` equals startOfDay(day), or nil.
     func record(for day: Date) throws -> DailyActivityRecord?

     /// Accumulates activity into TODAY's row (today = startOfDay of `now`): creates the row
     /// if absent, otherwise ADDS the deltas to the existing counters. Saves.
     @discardableResult
     func addActivity(
       cardsReviewed: Int, cardsLearned: Int, seconds: Int, now: Date
     ) throws -> DailyActivityRecord

     /// All rows sorted by day ASCENDING.
     func all() throws -> [DailyActivityRecord]

     /// Deletes all DailyActivityRecord rows and saves.
     func reset() throws
   }
   ```

   - Day normalization: ALWAYS `Calendar.current.startOfDay(for:)` — both when writing
     (`addActivity` stores `day = startOfDay(now)`) and when reading (`record(for:)` compares
     against `startOfDay(day)`). This is the single invariant that keeps one row per calendar
     day.
   - Negative deltas are not expected; do not guard for them.

2. Create `Tests/ActivityStoreTests.swift` (XCTest, class `ActivityStoreTests`). Every test
   uses `try PersistenceStack.container(inMemory: true)` + a fresh `ModelContext`. Build fixed
   dates from `DateComponents` (e.g. 2026-07-01 10:00 and 2026-07-01 22:30 for "same day",
   2026-07-02 for "next day") so results don't depend on when the tests run.

Conventions: single app target (`@testable import Enough`), Swift 5 / iOS 18, 2-space indent,
SwiftLint strict (lines ≤ 130).

## Done when

- `Tests/ActivityStoreTests.swift` asserts:
  - `record(for:)` nil on an empty store; `all()` empty.
  - One `addActivity(cardsReviewed: 5, cardsLearned: 2, seconds: 60, now: <day1 10:00>)`
    creates a row with exactly those values and `day == startOfDay(day1)`.
  - A second `addActivity(cardsReviewed: 3, cardsLearned: 0, seconds: 30, now: <day1 22:30>)`
    ACCUMULATES: still exactly 1 row total, now `cardsReviewed == 8`, `cardsLearned == 2`,
    `secondsStudied == 90`.
  - `addActivity(..., now: <day2>)` creates a second row; `all()` returns 2 rows in ascending
    day order (day1 first).
  - `record(for: <day1 at 23:59>)` finds day1's row (normalization on the read path).
  - After `reset()`: `all()` is empty.
- No file outside `Sources/Persistence/` and `Tests/` changes.
