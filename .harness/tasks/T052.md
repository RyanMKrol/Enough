# T052 — Progress components: streak card + week-dot row

## Overview

Build the two presentational components for the Progress tab — the amber streak card and the
7-dot week row — as pure views with injected state (no service access; assembly happens in a
later task). Design reference: .harness/docs/designs/design-spec.md §7.2 and §1;
architecture contract: .harness/docs/designs/architecture.md.

## Do

1. Create `Sources/Features/Progress/WeekDotsRow.swift`: `struct WeekDotsRow: View` with
   `init(dots: [DayDot])`. `DayDot` is the enum declared by the StatsService task in
   Sources/Services/ with cases `.done`, `.today`, `.upcoming`, `.missed` — consume it, do
   NOT redeclare it (if it is missing, the prerequisite isn't merged: record `failed:blocked`
   in the worklog rather than working around it). Render an `HStack` of up to 7 entries, each a
   `VStack`: a 24pt circle + a day letter below (11 Medium, `#8e8e93`). Day letters are fixed
   Monday-first: `["M", "T", "W", "T", "F", "S", "S"]`, indexed by position. Circle styles:
   - `.done` — filled `EnoughColor.streakAmber` with a white `checkmark` SF Symbol at 11pt bold.
   - `.today` — white fill, 2pt stroke in the environment accent (`@Environment(\.accentTheme)`,
     use `.accent`), no glyph.
   - `.upcoming` / `.missed` — filled `#e5e5ea`, no glyph.
2. Create `Sources/Features/Progress/StreakCard.swift`: `struct StreakCard: View` with
   `init(streak: Int, dots: [DayDot])`. A card with radius 20 and a `LinearGradient` from
   `EnoughColor.streakAmberTint` (`#ffecdd`, top-leading) to white (bottom-trailing), interior
   padding 18, resting shadow `0 1px 3px rgba(0,0,0,.05)`. Contents:
   - Top row: a `flame.fill` SF Symbol, 26pt, `EnoughColor.streakAmber`, animated with the SAME
     flicker treatment `StreakPill` uses (subtle scale + rotate loop, 2.2 s ease-in-out,
     `Motion.flicker` — mirror Sources/DesignSystem/StreakPill.swift's implementation); next
     to it **"<streak>-day streak"** in 22 Bold `EnoughColor.label`.
   - Below: **"Keep it alive — 3 min today does it"** in 13 Regular `#6b6b70`.
   - Bottom: `WeekDotsRow(dots: dots)`.
3. Both files get `#Preview`s: `StreakCard(streak: 6, dots:)` with 5 `.done`, 1 `.today`,
   1 `.upcoming` (the design's mock state), and a `WeekDotsRow` preview covering all four dot
   styles.

## Done when

- Both views compile as pure presentational components: state arrives via `init`, no
  `@Environment(\.services)` anywhere in this task's files.
- `WeekDotsRow` renders the four dot styles per the spec above (amber check / accent ring /
  faint fill) with Monday-first letters.
- `StreakCard` shows the flickering flame, "<n>-day streak", the exact keep-alive line, and the
  dot row on the amber→white gradient.
- The two `#Preview`s render the design's mock state (6-day streak, 5 done + today + 1
  upcoming) without a running app.
