# T006 — Pulsing audio button with ripple-once replay

## Overview

Add the audio affordance used on every card and sample row: an accent-tinted circular speaker
button with an idle pulsing ring, which fires its action and ripples once when tapped.
Design values from design-spec.md §1.4 (audio pulse) and §5/§6 (usage).

## Do

1. Create `Sources/DesignSystem/PulsingAudioButton.swift` with exactly this view:
   - `struct PulsingAudioButton: View` with `init(size: CGFloat = 60, isPulsing: Bool = true,
     action: @escaping () -> Void)`.
   - Body: a `Button` calling `action`. Label is a ZStack sized `size × size`:
     - Fill: `Circle()` in `accentTheme.tint` (read `@Environment(\.accentTheme)`).
     - Icon: `Image(systemName: "speaker.wave.2.fill")` in `accentTheme.accent`, font size
       `size * 0.4`, weight semibold.
     - Idle pulse ring (behind the fill, only when `isPulsing`): `Circle().stroke(
       accentTheme.accent, lineWidth: 2)` animating scale 0.85 → 1.7 and opacity 0.6 → 0 with
       `.easeOut(duration: Motion.pulse)` (2.2s) repeating forever WITHOUT autoreverse — a
       one-way expanding ripple loop.
   - Tap behavior: invoke `action()` AND emit exactly ONE extra ripple ring (same scale/opacity
     animation, single pass ~0.8s, then removed). Implement with a monotonically increasing
     trigger (e.g. an `Int` state incremented per tap keyed into `.id(...)` or a
     `phaseAnimator`) — the requirement is one ripple per tap, no residual looping.
   - The button must not manage audio itself — playback is the caller's job via `action`.
2. Add a `#Preview` showing a 60pt pulsing instance and a 34pt `isPulsing: false` instance
   (the deck-detail sample-row size) side by side.

Conventions: single app target, SwiftUI only, 2-space indent, SwiftLint strict (≤130 cols) +
swift-format strict. Touch only `Sources/DesignSystem/**` (and `Tests/**` if adding optional
tests).

## Done when

- `Sources/DesignSystem/PulsingAudioButton.swift` compiles with the exact view name and
  initializer signature above (later tasks use `PulsingAudioButton(size: 34, isPulsing: false,
  action: ...)` for sample rows and the 60pt default inside sessions).
- With `isPulsing: true` the ring loops continuously (2.2s expanding fade); with
  `isPulsing: false` no idle ring renders; every tap calls the action exactly once and shows
  exactly one transient ripple.
- Colors come from the `\.accentTheme` environment (tint fill, accent icon/ring) — nothing
  hardcoded to Japan red.
- The `#Preview` block exists with both size variants.
