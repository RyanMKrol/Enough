# T049 ‚Äî Flashcard review session screen (grades, live interval previews)

## Overview

Build the review session ‚Äî the flashcard mode that drives the SRS schedule: session chrome,
the T048 flashcard, and the Again/Hard/Good/Easy grade row whose interval previews come live
from `SRSEngine` for the current card; wire the Home reviews banner to launch it.

## Do

Design reference: .harness/docs/designs/design-spec.md (read-only reference) ¬ß6 + ¬ß6b. Architecture:
`SessionEngine` (review mode), `StudyService.makeReviewSession()`,
`SRSEngine.previewLabel(_:for:)`, `GradeChoice`/`GradeButtonRow` (DesignSystem), `SRSGrade`
(SRSKit).

1. `Sources/Features/Session/ReviewSessionViewModel.swift` ‚Äî `@Observable final class
   ReviewSessionViewModel`, `init(engine: SessionEngine, services: AppServices)`:
   - `isRevealed: Bool` (reset to `false` on every advance).
   - `previews: [GradeChoice: String]` for the CURRENT card's live `SRSState` ‚Äî pinned
     mapping GradeChoice‚ÜîSRSGrade: `.again`‚Üí`.again`, `.hard`‚Üí`.hard`, `.good`‚Üí`.good`,
     `.easy`‚Üí`.easy`; each label = `SRSEngine.previewLabel(srsGrade, for:
     currentCard.state)`. NEVER hardcode label strings.
   - `grade(_ choice: GradeChoice)` ‚Üí `UIImpactFeedbackGenerator(style: .light)`,
     `engine.submitGrade(srsGrade)`, then `advance()`.
   - `advance()` ‚Üí `engine.advance()`, `isRevealed = false`; when `engine.isComplete`:
     `try services.study.commit(engine)` then `route = SessionRoute.complete`.
   - `closeAndCommit()` ‚Üí commit graded-so-far via `services.study.commit(engine)`, dismiss.
   - Progress mirrors T047: `progressText` "\(done)/\(total)", value from
     `engine.progress`, advancing on grade only.
   - Audio: `services.audio.resetAutoPlay()` on start; `autoPlayOnce(cardId:fileName:)` per
     card appearance; flashcard's speaker ‚Üí `services.audio.play(fileName:)`.
2. `Sources/Features/Session/ReviewSessionView.swift` ‚Äî `struct ReviewSessionView: View`,
   `init(engine: SessionEngine)`. Canvas `#f2f2f7` (`EnoughColor.canvas`) ‚Äî NOT white; this
   distinguishes review from MC (design ¬ß6b). Layout: `SessionChrome(progress:counterText:
   onClose:)` (reuse from T047 ‚Äî same file, Session scope); centered `FlashcardView(card:
   isRevealed: $vm.isRevealed, onPlayAudio:)` for the current card's `CardContent` (resolve
   via `services.contentStore.cards(forDeck:)` lookup of `engine.current`); below, ONLY when
   revealed: "How well did you know it?" ‚Äî 13 Regular `#8e8e93`, centered ‚Äî then
   `GradeButtonRow(previews: vm.previews, onGrade: vm.grade)` pinned near the bottom
   (padding 22). Before reveal, no grade UI at all.
   - Card-to-card transition identical to T047: `.id(currentCardId)` +
     `.transition(.asymmetric(insertion: .move(edge: .bottom), removal: .move(edge:
     .leading)))` with `Motion.cardSwap`.
   - `.navigationDestination(for: SessionRoute.self)` ‚Üí the same placeholder T047 added
     (T050 replaces it once).
3. **Launch wiring** (Home scope included for exactly this): in `HomeView`, replace the T044
   `// TODO(T049)` ‚Äî ReviewsBanner's `onReview` presents a `fullScreenCover` with
   `ReviewSessionView(engine: try services.study.makeReviewSession())`.
4. Tests in `Tests/ReviewSessionViewModelTests.swift` (in-memory container, fixture catalog,
   fixed date provider; build an engine via `StudyService` after seeding due
   `CardSRSRecord`s).
