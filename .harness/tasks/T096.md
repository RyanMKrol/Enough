## Overview
A single pure, dependency-injected reader that returns (dueCount, nextDueDate) from the SRS schedule for the owned decks, so both the Reviews tab and a widget timeline provider read the same numbers the same way, without pulling in SwiftUI or the app-only service graph.

## Do
Add a small type (e.g. ReviewsSummaryProvider) that, given a CardSRSStore + EntitlementStore + ContentStore + DateProvider (all already existing), computes struct ReviewsSummary { let dueCount: Int; let nextDueDate: Date? }. dueCount reuses the existing due logic (DeckProgressService.totals().due / SRSEngine.isDue); nextDueDate is the earliest future dueAt across owned decks (the logic currently inline in ReviewsTabView.getNextDueDate()). Refactor ReviewsTabView to call it so there is one source of truth. Keep it free of SwiftUI and of AppServices so a widget extension can construct it from its own ModelContext + a widget-target ContentStore.

## Done when
ReviewsSummaryProvider returns (dueCount, nextDueDate) matching the values ReviewsTabView showed before the refactor; a unit test seeds SRS records (due now, due-later, mastered, new) in an in-memory store and asserts both the count and the earliest next-due date; ReviewsTabView compiles using the new provider; xcodebuild test passes.
