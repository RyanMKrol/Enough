# T054 — Progress tab assembly (streak card, stat tiles, readiness ring)

## Overview

Assemble the Progress tab from the already-built pieces — `StreakCard`/`WeekDotsRow`,
`StatsService`, `DeckProgressService`, `ReadinessCalculator`, `ProgressRing` — and wire it into
the main shell in place of the progress placeholder. Design reference:
.harness/docs/designs/design-spec.md §7.2 (+ §1 tokens); architecture:
.harness/docs/designs/architecture.md.

## Do

1. Create `Sources/Features/Progress/ProgressViewModel.swift`: `@Observable final class
   ProgressViewModel` with `init(services: AppServices)` and a `func refresh()` that loads:
   - `streak: Int` ← `services.stats.currentStreak()`
   - `dots: [DayDot]` ← `services.stats.weekDots()`
   - `wordsLearned: Int` ← `services.deckProgress.wordsLearned()`
   - `minutes: Int` ← `services.stats.totalMinutes()`
   - `decksGoing: Int` ← count of owned decks (via
     `services.entitlementStore.ownedDeckIds(catalog:)`) whose
     `services.deckProgress.progress(forDeck:).learned > 0`
   - `readinessPercent: Int` + `readinessLine: String` ← `ReadinessCalculator.readiness(
     tripScenarios:decks:progress:)` fed from the active trip
     (`services.tripStore.activeTrip()` → its `scenarioIds` resolved against
     `catalog.scenarios`, in the trip's stored order) and the trip country's `decks`.
   - `destination: String` from the trip country id: `japan` → "Tokyo", `france` → "Paris",
     `germany` → "Berlin" (fallback: country `name`).
   Swallow-and-default on thrown errors (zeroes/empty line) — never crash the tab.
2. Create `Sources/Features/Progress/ProgressTabView.swift` — the `.progress` tab screen.
   `@Environment(\.services)`, `@Environment(\.accentTheme)`; canvas `EnoughColor.canvas`;
   `refresh()` on `.onAppear`. `ScrollView`, top → bottom:
   - Large title **"Progress"** (`EnoughFont.largeTitle()`).
   - `StreakCard(streak: vm.streak, dots: vm.dots)`.
   - Three equal-width stat tiles (white, radius 18, count 22 Bold + caption 12 Semibold
     uppercase `#8e8e93`): **"<n> / words learned"**, **"<n> / minutes"**, **"<n> / decks
     going"** (count above caption).
   - **Readiness card** (white, radius 20, padding 18): header **"<destination> survival
     readiness"** in 15 Semibold; an `HStack` of a 72pt `ProgressRing(progress:
     Double(vm.readinessPercent)/100, size: 72, tint: accent)` (percent label shown) and
     `vm.readinessLine` in 15 Regular `#6b6b70`, multiline.
3. In `Sources/App/MainShellView.swift`, replace the `.progress` placeholder with
   `ProgressTabView()`. Touch nothing else there.
4. `#Preview` with `AppServices.preview()`.

## Done when

- Every number on the screen comes from the view model (services/calculator) — zero hardcoded
  stats; a fresh install renders zeroes and "Start with …" without crashing.
- The readiness ring animates to the calculated percent and the line is exactly
  `ReadinessCalculator`'s output.
- Verify on the simulator: launch with `-demo-state` (streak 6, 12 due, 3 owned decks) — the
  amber card reads "6-day streak" with "Keep it alive — 3 min today does it" and a week row of
  filled amber checks + an outlined today ring; the three tiles show non-zero words learned /
  minutes / decks going; the "<destination> survival readiness" card shows a partial accent
  ring and a "You can handle …" line. Capture via `./build_run.sh`.
