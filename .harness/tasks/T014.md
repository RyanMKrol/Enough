## Overview

Lay the SwiftData foundation: define ALL FOUR persistent record models in one place plus a
container factory with an in-memory mode for tests. This task defines the complete schema; the
follow-up tasks (T015–T018) add store classes AROUND these records without altering their
fields.

## Do

1. Create `Sources/Persistence/Records.swift` with exactly these four `@Model` classes (import
   `SwiftData` + `Foundation`). Every property below is required, non-optional unless marked;
   give each class a memberwise-style `init` that takes every property (with the noted
   defaults):

   ```swift
   @Model final class TripProfileRecord {
     var countryId: String
     var duration: String          // "weekend" | "week"
     var scenarioIds: [String]
     var startDate: Date
     var accentRawValue: String    // matches AccentTheme raw values, e.g. "japan"
     var isActive: Bool            // default true
   }

   @Model final class EntitlementRecord {
     var productId: String         // a deck id or bundle id from the content catalog
     var kind: String              // "deck" | "bundle"
     var grantedAt: Date
   }

   @Model final class CardSRSRecord {
     var deckId: String
     var cardId: String
     var statusRaw: String         // "new" | "learning" | "review"
     var easeFactor: Double        // default 2.5
     var intervalDays: Double      // default 0
     var repetitions: Int          // default 0
     var lapses: Int               // default 0
     var dueAt: Date?              // nil until first graded
     var lastReviewedAt: Date?     // nil until first graded
   }

   @Model final class DailyActivityRecord {
     var day: Date                 // normalized start-of-day
     var cardsReviewed: Int
     var cardsLearned: Int
     var secondsStudied: Int
   }
   ```

2. Create `Sources/Persistence/EnoughSchema.swift` — the schema is VERSIONED from day one so
   every future model change is an explicit migration, never an implicit-schema retrofit:

   ```swift
   enum EnoughSchemaV1: VersionedSchema {
     static var versionIdentifier: Schema.Version { Schema.Version(1, 0, 0) }
     static var models: [any PersistentModel.Type] {
       [TripProfileRecord.self, EntitlementRecord.self,
        CardSRSRecord.self, DailyActivityRecord.self]
     }
   }

   enum EnoughMigrationPlan: SchemaMigrationPlan {
     static var schemas: [any VersionedSchema.Type] { [EnoughSchemaV1.self] }
     static var stages: [MigrationStage] { [] }
   }
   ```

   (The record classes stay top-level with plain names — registration in `EnoughSchemaV1` is
   what matters. A nested-classes-with-typealiases refactor can come at the first real V2
   migration.) Every future model/field change adds a `SchemaV2` + `MigrationStage` — it must
   NEVER edit `EnoughSchemaV1`'s models.

3. Create `Sources/Persistence/PersistenceStack.swift`:

   ```swift
   enum PersistenceStack {
     static var models: [any PersistentModel.Type] { EnoughSchemaV1.models }
     static func container(inMemory: Bool = false) throws -> ModelContainer
   }
   ```

   `container(inMemory:)` builds `Schema(versionedSchema: EnoughSchemaV1.self)` and a
   `ModelConfiguration` — named `UUID().uuidString` when `inMemory` is true (two in-memory
   containers must provably not share a store) and a stable `"Enough"` on disk — with
   `isStoredInMemoryOnly: inMemory`, returning
   `try ModelContainer(for: schema, migrationPlan: EnoughMigrationPlan.self,
   configurations: [config])`.

4. Create `Tests/PersistenceStackTests.swift` (XCTest, class `PersistenceStackTests`).
   HERMETIC-TESTS RULE: every test builds `try PersistenceStack.container(inMemory: true)` and a
   `ModelContext(container)` — never the on-disk store, never the `ModelContainer(for:)`
   convenience initializer (it hits disk).

Conventions: single app target (`@testable import Enough`), Swift 6 mode / iOS 18 (the test
target uses nonisolated default isolation — mark test methods that touch app types `@MainActor`,
never the XCTestCase class), 2-space indent, SwiftLint strict (lines ≤ 130).

## Done when

- `Tests/PersistenceStackTests.swift` asserts:
  - `PersistenceStack.container(inMemory: true)` does not throw and
    `PersistenceStack.models.count == 4`.
  - `EnoughSchemaV1.models` contains all four record types and
    `EnoughSchemaV1.versionIdentifier == Schema.Version(1, 0, 0)`; the container is built with
    `EnoughMigrationPlan` (constructing it via the migration-plan initializer not throwing IS
    the assertion).
  - For EACH of the four record types: insert one fully-populated instance into a fresh
    in-memory context, `try context.save()`, fetch it back with `FetchDescriptor`, and assert
    every stored property round-trips (e.g. a `TripProfileRecord` with
    `countryId == "japan"`, `scenarioIds == ["greetings", "eating-out"]`,
    `isActive == true`; a `CardSRSRecord` with `dueAt == nil` stays nil).
  - Two separate in-memory containers are isolated: a record inserted into container A is not
    fetchable from container B (proves `inMemory` isn't sharing an on-disk store).
- No file outside `Sources/Persistence/` and `Tests/` changes.
