## Overview

Accessibility conformance sweep across every screen: VoiceOver labels/values/traits on custom
controls, Dynamic Type survival, Reduce Motion fallbacks, and 44pt hit targets. The app must be
genuinely usable with assistive settings on — this is also App Review surface.

## Do

1. **VoiceOver** — add labels/values/traits to every custom control (custom controls expose
   nothing by default):
   - `PulsingAudioButton`: `.accessibilityLabel("Play audio")`.
   - `StreakPill`: combined element, label "\(count)-day streak".
   - `ProgressRing`/`AnimatedProgressBar`: `.accessibilityValue("\(percent) percent")`.
   - `GradeButtonRow`: each button labeled with its title AND interval preview as value.
   - Tab bar items: label = tab name, `.accessibilityAddTraits(.isSelected)` when active.
   - `ScenarioChip`, country cards, bundle rows, pack rows: button trait + `.isSelected` when
     selected.
   - List rows (deck rows, sample-card rows): `.accessibilityElement(children: .combine)` so a
     row reads as one sentence, with the row's action as the element's action.
   - Decorative motion (card-stack hero, flame) marked `.accessibilityHidden(true)` where it
     duplicates adjacent text.
2. **Dynamic Type** — launch with `-demo-state` at AX2/AX3 (set via
   `xcrun simctl ui booted content-size accessibility-extra-large` before screenshotting) and
   walk every screen: no clipped text, no overlapping controls. Fix with `minHeight`,
   `.fixedSize(horizontal: false, vertical: true)`, and `AnyLayout(HStackLayout/VStackLayout)`
   reflow where a row genuinely can't fit horizontally.
3. **Reduce Motion** — every infinite animation (card bob, audio pulse, flame flicker) gates on
   `@Environment(\.accessibilityReduceMotion)`: static fallback (one-shot fills stay).
4. **Hit targets** — every tappable element ≥44×44pt effective area:
   `.frame(minWidth: 44, minHeight: 44).contentShape(Rectangle())` on the small ones (34pt audio
   buttons, 30pt close X, 22pt checks).

## Done when

- The worklog contains a per-screen checklist table: VoiceOver reading order sensible, every
  interactive element labeled, Dynamic Type AX3 un-clipped, Reduce Motion honored, hit targets
  ≥44pt — each row checked or fixed-and-checked.
- Screenshots at default size AND accessibility-extra-large for Home, Plan, MC session, and
  Reviews are captured and visually verified (no clipping/overlap).
- No `.accessibilityHidden` on anything informative; no unlabeled icon-only buttons remain
  (grep audit: every `Image(systemName:` inside a Button has a label somewhere in the chain).
