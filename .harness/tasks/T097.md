## Overview
Create the WidgetKit app-extension target in project.yml so a widget bundle exists, builds, is embedded in the Enough app, and can read the shared store + content. This is the plumbing only; the timeline provider + widget UI land in the next unit.

## Do
In project.yml add a new target EnoughWidget (type: app-extension, platform iOS 26 to match the app's iOS 26 deployment target, uses the CODE_SIGN_ENTITLEMENTS from reviews-surface-appgroup with App Group group.com.ryankrol.enough, product bundle id com.ryankrol.enough.widget), embed it in the Enough app (dependency + copy-into-appex). Give it its own Sources/Widget source dir with a minimal WidgetBundle entry point and an Info.plist declaring NSExtension WidgetKit. Embed the Content folder into the widget target's resources (so ContentStore(bundle:) can read the catalog from the widget bundle) and add the shared Sources modules it needs to compile. Add Sources/PrivacyInfo.xcprivacy if a required-reason API (e.g. file-timestamp) is used. Keep .harness/ out of any source glob.

## Done when
xcodegen generate produces an Enough.xcodeproj with an EnoughWidget app-extension embedded in the Enough app; xcodebuild build for the Enough scheme compiles both the app and the widget target with the App Group entitlement present; the widget target can resolve ContentStore + the shared SRS/persistence types.
