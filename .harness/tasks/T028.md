# T028 — StatsService: streaks, week dots & lifetime totals

## Overview

Read-only aggregation over `DailyActivityRecord` rows (T018) powering the streak pill, the
Progress tab's streak card + week-dot row, and the stat tiles (design-spec.md §4, §7.2). Writes
happen elsewhere (`ActivityStore` via `StudyService`) — this service only computes.

## Do

1. Create `Sources/Services/StatsService.swift`:

   ```swift
   enum DayDot: String, Equatable { case done, today, missed, upcoming }
   final class StatsService {
     init(activityStore: ActivityStore, dateProvider: DateProvider,
          calendar: Calendar = .current)
     func currentStreak() throws -> Int
     func weekDots() throws -> [DayDot]   // exactly 7, Monday first
     func totalMinutes() throws -> Int
   }
   ```

   A day "has activity" when a `DailyActivityRecord` exists for it with
   `cardsReviewed + cardsLearned > 0 || secondsStudied > 0`. Compare days via
   `calendar.startOfDay(for:)`.

2. `currentStreak()`: count consecutive days with activity walking BACKWARD from a start day:
   start at today if today has activity, else at yesterday (a streak survives until a full day
   is actually missed — design-spec.md §7.2 "Keep it alive — 3 min today does it"). If neither
   today nor yesterday has activity → 0. Walk back day by day while each day has activity.

3. `weekDots()`: the Monday-starting week containing `dateProvider.now` (force Monday start via
   a local copy of the calendar with `firstWeekday = 2`; derive Monday from
   `calendar.dateInterval(of: .weekOfYear, for: now)`). Per day, first match wins:
   activity → `.done`; is today (no activity) → `.today`; before today → `.missed`;
   after today → `.upcoming`.

4. `totalMinutes()`: `Int((Double(totalSeconds) / 60.0).rounded())` where `totalSeconds` sums
   `secondsStudied` over ALL rows (`activityStore.all()`).

5. Create `Tests/StatsServiceTests.swift` (XCTest): in-memory container
   (`PersistenceStack.container(inMemory: true)`), `struct FixedDateProvider: DateProvider
   { let now: Date }`, and a FIXED calendar (`Calendar(identifier: .gregorian)` with
   `timeZone = TimeZone(identifier: "Europe/London")!`) passed into the service. Pick a known
   Wednesday as `now` (e.g. 2026-07-15 12:00 UTC — construct via `DateComponents` through that
   calendar, don't hand-compute epochs). Seed activity via `ActivityStore.addActivity`.

## Done when

Tests pin, at minimum (now = a Wednesday):
- Activity on Mon+Tue+Wed (today) → `currentStreak() == 3`.
- Activity on Mon+Tue, NONE today → `currentStreak() == 2` (yesterday anchor keeps it alive).
- Activity on Mon only (gap on Tue, none today) → `currentStreak() == 0`.
- No rows at all → `currentStreak() == 0`; a row with all-zero counts does NOT count as activity.
- Week dots for activity on Mon+Tue, none Wed: `[.done, .done, .today, .upcoming, .upcoming,
  .upcoming, .upcoming]`; with activity ALSO on Wed the third dot is `.done`; a week where Tue
  was skipped but Mon+Wed done: `[.done, .missed, .done, ...]`.
- `totalMinutes`: rows of 90 s + 45 s → `2` (135 s / 60 = 2.25 → 2); 150 s → `3` (2.5 rounds up).
