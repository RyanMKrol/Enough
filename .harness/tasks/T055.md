# T055 — Browse tab store (country sections, prices, buy & restore)

## Overview

Build the Browse tab — the store where packs and bundles are bought — and wire it into the main
shell in place of the browse placeholder. Browse is unmocked: compose it strictly from the
established design vocabulary per .harness/docs/designs/design-spec.md §9 (+ §1 tokens, §2.4
pricing language); architecture: .harness/docs/designs/architecture.md.

## Do

1. Create `Sources/Features/Browse/BrowseViewModel.swift`: `@Observable final class
   BrowseViewModel`, `init(services: AppServices)`. State: `countries: [CountryInfo]` (every
   catalog country, the ACTIVE trip's country first, rest in catalog order),
   `comingSoon: [String]`, `ownedDeckIds: Set<String>`, `ownedProductIds: Set<String>`,
   `inFlightProductId: String?`, `errorMessage: String?`, `pendingProductId: String?`,
   `showRestoredToast: Bool`. Methods:
   - `refresh()` — reload catalog + owned sets from `services.entitlementStore` /
     `services.purchase.ownedProductIds`.
   - `buy(productId:kind:) async` — guard nothing in flight; set `inFlightProductId`;
     `try await services.purchase.purchase(productId:kind:)` and handle the returned
     `PurchaseOutcome`: `.success` → refresh; `.cancelled` → do NOTHING (a user Cancel is
     normal — no alert, no error text); `.pending` → set `pendingProductId` (row shows an
     inline "Pending approval" note, 13 `#6b6b70`; the entitlement arrives later via the
     change notification). On throw set `errorMessage`. Always clear in-flight.
   - `restore() async` — wrap `services.purchase.restorePurchases()` in `do/catch`:
     on success ONLY, `showRestoredToast = true`; on throw route to `errorMessage` (the same
     alert path).
   - `price(for productId: String, fallbackGBP: Double) -> String` — returns
     `services.purchase.displayPrice(productId:)` falling back to
     `PricingCalculator.price(fallbackGBP)`. ALL unit prices render through this.
   In `init`, observe the multicast entitlement change from T026:
   `NotificationCenter.default.addObserver(forName: .entitlementsChanged, ...)` → `refresh()`
   (store the token; remove in `deinit`). Never assume this VM is the only observer.
2. Create `Sources/Features/Browse/BrowseView.swift` — the `.browse` tab screen. Canvas
   `EnoughColor.canvas`; large title **"Browse"** (`EnoughFont.largeTitle()`); `ScrollView` of
   country sections; `refresh()` on `.onAppear`. Per country section:
   - Header: `flagEmoji` + country `name` (20 Semibold) and the native language name as an
     `EyebrowLabel` beneath (e.g. "日本語").
   - **Bundle cards** first — white card radius 20, per bundle: `title` (17 Semibold),
     `subtitle` + " · save <£x.xx>" (13, `#6b6b70`) where the savings figure is
     `PricingCalculator.bundleSavings(_:in:)` formatted by `PricingCalculator.price(_:)`
     (computed savings stay catalog-based in the stub era — localized savings land in T073);
     trailing: if `ownedProductIds` contains the bundle id (or all member decks owned) a
     `checkmark.circle.fill` in `EnoughColor.successGreen` + "Owned" (13 Semibold); otherwise a
     price pill (accent-tinted fill, `AccentTheme(rawValue: country.accent)`'s tint/deep
     colors, fully rounded, label via `vm.price(for: bundle.id, fallbackGBP: bundle.priceGBP)`)
     that triggers `buy(productId: bundle.id, kind: .bundle)`.
   - **Deck rows** in one grouped white card, hairline dividers: 34pt icon tile (the deck's
     `icon` SF Symbol on the country's accent tint, radius 10), `title` (17), "<cardCount>
     cards" (13 `#8e8e93`); trailing: green check + "Owned" when `ownedDeckIds` contains it,
     else a price pill labeled via `vm.price(for: deck.id, fallbackGBP: deck.priceGBP)`
     triggering `buy(productId: deck.id, kind: .deck)`. While `inFlightProductId` matches a
     row, replace its pill with a `ProgressView` and disable it; while `pendingProductId`
     matches, show the "Pending approval" note instead of the pill.
   - Purchase errors surface via `.alert("Something went wrong", …)` showing `errorMessage`
     with an "OK" button. (`.cancelled` NEVER reaches this alert.)
3. Bottom of the scroll, after all sections: a **"Restore purchases"** button
   (`TextLinkButtonStyle`) calling `restore()`; on `showRestoredToast`, show a brief overlay
   capsule **"Purchases restored"** (white on `EnoughColor.graphite`, auto-dismiss ~2 s). Then a
   footer **"More soon: Italy · Spain · Thailand"** built by joining `comingSoon` with " · "
   (13, `#8e8e93`, centered).
4. In `Sources/App/MainShellView.swift`, replace the `.browse` placeholder with `BrowseView()`.
5. `#Preview` with `AppServices.preview()`.

## Done when

- Every product shown comes from the catalog (no hardcoded titles/prices); the active trip's
  country section is first; every unit price string flows through
  `displayPrice`-with-fallback. Grep gate: `grep -rn "£" Sources/Features/` returns NOTHING —
  no currency literal is ever hardcoded in feature code.
- Tapping an unowned price pill flips it to "Owned" (stub grants instantly) and the owned state
  also refreshes when `.entitlementsChanged` is posted from elsewhere (e.g. a purchase made on
  the plan screen); a second purchase of an owned product is impossible (pill no longer
  rendered).
- A `.cancelled` outcome produces no alert and no state change (unit-test the VM with a fake
  `PurchaseProviding` returning `.cancelled`); `.pending` shows the inline note and grants
  nothing.
- "Restore purchases" shows the "Purchases restored" toast only when restore did not throw;
  a throwing restore lands in the error alert. Buying while another purchase is in flight is
  blocked.
- Verify on the simulator: launch with `-demo-state` (3 Japan decks owned) — Japan section
  first with its owned decks marked "Owned" (green check), remaining products showing price
  pills, bundle savings lines present, restore row + "More soon" footer at the bottom. Capture
  via `./build_run.sh`.
