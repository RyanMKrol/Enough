## Overview

Audit-and-fix sweep: run the Axiom **swiftdata-auditor** over the whole codebase (SwiftData safety), triage its
findings against this project's documented conventions, and fix every confirmed finding. This is
a wide-scope quality gate near the end of the backlog — the app is feature-complete by now.

## Do

1. Launch the `axiom:swiftdata-auditor` agent via the Agent tool with a prompt like: "Audit the project at
   the repo root (SwiftData safety). Report every finding with file:line, severity, and the exact
   violated rule." If that agent type is unavailable in your session, read its definition —
   `ls ~/.claude/plugins/cache/axiom-marketplace/axiom/*/agents/swiftdata-auditor.md` — and execute its
   documented checklist yourself over the same code.
2. Triage EVERY finding before touching code. These auditors are pattern-based; false positives
   are expected. Judge each against the repo's architecture contract and design spec (the task's
   design doc). Domain focus + known-intentional choices: struct models, missing VersionedSchema registration, relationship defaults, migration timing, background-context misuse, N+1 fetch patterns. Known intentional choices that are NOT findings: relationship-free models, code-level uniqueness (no @Attribute(.unique)), single main-thread context.
3. Fix confirmed findings in place with the smallest reasonable diff, preserving behavior unless
   the finding IS a behavior bug. Update or add tests where a fix changes logic.
4. Record the full triage table in your worklog entry: finding → verdict (fixed / false-positive
   with reason / deferred with reason). Deferred findings must have a concrete revisit condition.

## Done when

- The audit ran (agent or manual checklist) over all of Sources/ and Tests/ and the triage table
  in the worklog covers every reported finding — none unaddressed.
- Every confirmed finding is fixed in this branch; a re-run of the audit (or a re-walk of the
  checklist) over the changed files reports no remaining confirmed findings in this domain.
- No behavior regressions: the full existing test suite still passes unmodified except where a
  fix legitimately changed pinned behavior (justify any test edit in the worklog).
