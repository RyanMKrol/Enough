## Overview
The Reviews page (Sources/Features/Reviews/ReviewsTabView.swift) currently lists every owned deck across all owned packs in a single FLAT list (deckStrengthList(), ~line 182), so decks that share the same name across country packs (e.g. a "Greetings & basics" deck in each pack) appear as ambiguous, duplicate-looking rows once a user owns more than one pack. Restructure the flat list into per-language SECTIONS, each introduced by a header built from the deck's parent CountryInfo (flagEmoji + a language name such as nativeLanguageName), so multi-pack ownership is legible. The deck->country mapping is already at hand: reload() (~line 238) iterates catalog.countries[].decks to build deckInfoById, and DeckInfo (Models.swift) has no country field so grouping MUST be derived from the nested CountryInfo (Models.swift has name/languageName/nativeLanguageName/flagEmoji). The global Due/Learning/Mastered counters at the top (statTiles()) stay global and unchanged; this is purely the per-deck strength list.
