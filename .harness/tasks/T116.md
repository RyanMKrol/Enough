## Overview
On DeckDetailView the primary action buttons ('Continue learning' and 'Practice all' beneath it) are hidden behind the floating Liquid Glass tab bar. Root cause is in MainShellView: its ZStack(alignment: .bottom) overlays LiquidGlassTabBar ON TOP of the tab content with no reserved bottom space, so any bottom-pinned content is covered. This is systemic (every tab screen currently hand-rolls its own inconsistent bottom padding to compensate). Fix it globally by making MainShellView reserve room for the tab bar via .safeAreaInset(edge: .bottom) instead of a pure overlay, so all tab content (including DeckDetail's bottom-pinned action bar) lays out ABOVE the bar. NOTE: the fix strategy (global inset vs per-screen padding vs hiding the bar on detail screens) and the exact done bar are OPEN QUESTIONS relayed to the owner — read the resolved answers before implementing; if the owner picks a per-screen or hide-on-detail approach instead, the scope moves to Sources/Features/DeckDetail/DeckDetailView.swift.
