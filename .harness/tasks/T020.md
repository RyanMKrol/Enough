# T020 — SRS queries: dueness, mastery, interval previews & next-due formatting

## Overview

Add the read-only query layer over `SRSState` (T019): due/mastered predicates, per-grade interval
previews (the labels shown under the Again/Hard/Good/Easy buttons — see design-spec.md §6b), and
the humanized next-due string used by the Reviews tab empty state. All pure functions; the UI must
always ask these functions for previews, never hardcode them.

## Do

1. Create `Sources/SRSKit/SRSQueries.swift` (`import Foundation` only). Extend `enum SRSEngine`
   with:

   ```swift
   static func previewInterval(_ grade: SRSGrade, for state: SRSState) -> TimeInterval
   static func previewLabel(_ grade: SRSGrade, for state: SRSState) -> String
   static func isDue(_ state: SRSState, now: Date) -> Bool
   static func isMastered(_ state: SRSState) -> Bool
   static func nextDueLabel(_ dueAt: Date, now: Date, calendar: Calendar = .current) -> String
   ```

2. `previewInterval` returns the `intervalDays * 86_400` seconds that
   `SRSEngine.apply(grade, to: state, now:)` would produce (implement by calling `apply` with any
   fixed reference date and reading the new interval — do NOT duplicate the ladder maths).

3. `previewLabel` formats that interval, pinned rules:
   - interval < 1 hour → `"<1 min"`; ≥ 1 hour but < 1 day → `"<1 day"`;
   - otherwise round to whole days `n`: `n == 1` → `"1 day"`; `n < 30` → `"\(n) days"`;
   - `n >= 30` → months `m = Int((Double(n) / 30.0).rounded())`: `m == 1` → `"1 month"`,
     else `"\(m) months"`.

4. `isDue`: `state.dueAt != nil && state.dueAt! <= now` (a `.new` card with nil `dueAt` is NOT
   due — it is unseen, not overdue). `isMastered`: `state.status == .review &&
   state.intervalDays >= 21`.

5. `nextDueLabel` (for "Nothing due right now — next review <label>"), pinned rules evaluated in
   order against `dueAt` vs `now`:
   - `dueAt <= now` → `"now"`;
   - under 1 hour away → `"in \(max(1, minutes)) minutes"` (singular `"in 1 minute"`);
   - under 24 h away AND same calendar day → `"in \(hours) hours"` (singular `"in 1 hour"`,
     hours = Int(seconds/3600) rounded down, min 1);
   - `calendar.isDateInTomorrow(dueAt)` → `"tomorrow"`;
   - else → `"in \(days) days"` where `days = calendar.dateComponents([.day], from:
     calendar.startOfDay(for: now), to: calendar.startOfDay(for: dueAt)).day!`.

6. Create `Tests/SRSQueriesTests.swift` (XCTest) with a fixed
   `now = Date(timeIntervalSince1970: 1_800_000_000)` and, for calendar-dependent asserts, an
   explicit `var cal = Calendar(identifier: .gregorian); cal.timeZone = TimeZone(identifier:
   "Europe/London")!` passed to `nextDueLabel`.

## Done when

- Tests pin the design's young-card preview quartet: for
  `SRSState(status: .learning, easeFactor: 2.5, intervalDays: 1, repetitions: 1, lapses: 0,
  dueAt: now)` the four `previewLabel` results are exactly
  `again → "<1 min"`, `hard → "1 day"` (a learning card's interval HOLDS on hard — see T019),
  `good → "3 days"`, `easy → "6 days"` — exactly the quartet the design shows under the grade
  buttons.
- Tests pin: a fresh `SRSState.newCard` previews `good → "1 day"` and `easy → "6 days"`.
- Mastery boundary: `.review` at `intervalDays: 20.9` is NOT mastered; at `21.0` it IS.
- Dueness: nil `dueAt` → false; `dueAt == now` → true; `dueAt = now + 60` → false.
- `nextDueLabel` cases each asserted: past date → `"now"`; +30 min → `"in 30 minutes"`;
  +2 h (same day — pick a morning `now`) → `"in 2 hours"`; next calendar day → `"tomorrow"`;
  +3 days → `"in 3 days"`.
- Label formatting: 45-day interval previews as `"2 months"` (45/30 = 1.5 → rounds to 2).
