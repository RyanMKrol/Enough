# T010 — SRS grade button row (Again/Hard/Good/Easy with interval sublabels)

## Overview

Add the four-button grading row used on revealed flashcards, plus the display-level
`GradeChoice` enum it renders. Deliberately UI-only: SRSKit's `SRSGrade` (a later task) maps
onto `GradeChoice` at the call site — this task must NOT create or reference any SRS logic.
Design values from design-spec.md §6b and §1.1.

## Do

1. Create `Sources/DesignSystem/GradeButtons.swift` with exactly these two types:
   - `enum GradeChoice: CaseIterable` with cases `again`, `hard`, `good`, `easy`, and computed
     vars (switch per case):
     - `var title: String` → "Again" / "Hard" / "Good" / "Easy".
     - `var fill: Color` → `#ffe9e6` / `#f2f2f7` / `#d8f9dd` / `#caf7ff`
       (again red-tint, hard neutral, good `EnoughColor.successTint`, easy
       `EnoughColor.easyTint` — use the EnoughColor constants where they exist).
     - `var textColor: Color` → `#b71824` / `#6b6b70` (`EnoughColor.secondaryText`) /
       `#00722e` (`EnoughColor.successDeep`) / `#1666aa` (`EnoughColor.easyBlue`).
   - `struct GradeButtonRow: View` with `init(previews: [GradeChoice: String],
     onGrade: @escaping (GradeChoice) -> Void)`.
     An `HStack(spacing: 8)` of four equal-width buttons (`.frame(maxWidth: .infinity)`), one
     per `GradeChoice.allCases` in declaration order. Each button: a `VStack(spacing: 2)` of
     the `title` (15pt semibold, `textColor`) and the interval sublabel
     (`previews[choice] ?? ""`, 11pt regular, `textColor` at 70% opacity), padded to ~56pt
     tall, background `fill` in a `RoundedRectangle(cornerRadius: 14)` (`Layout.chipRadius`).
     Tapping calls `onGrade(choice)`. Pressed state: scale 0.97.
     No haptics here — the session screen owns feedback.
2. Add a `#Preview` rendering
   `GradeButtonRow(previews: [.again: "<1 min", .hard: "1 day", .good: "3 days",
   .easy: "6 days"], onGrade: { _ in })` on a `#f2f2f7` background.

Conventions: single app target, SwiftUI only, 2-space indent, SwiftLint strict (≤130 cols) +
swift-format strict. Touch only `Sources/DesignSystem/**` (and `Tests/**` if adding optional
tests).

## Done when

- `Sources/DesignSystem/GradeButtons.swift` compiles with the exact enum/view names and
  signatures above (the review-session task constructs `GradeButtonRow` with previews computed
  live by the SRS engine and maps `GradeChoice` → its own grade type).
- The four buttons render in order Again/Hard/Good/Easy with the pinned fill/text colors, equal
  widths, titles over interval sublabels, and each tap reports the right `GradeChoice`.
- A missing preview string renders an empty sublabel rather than crashing.
- The `#Preview` block exists with the "<1 min / 1 day / 3 days / 6 days" example.
